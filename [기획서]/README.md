# 십보강호 기획서

사용자와 새 작업자는 아래 **다섯 활성 기획서만 먼저 읽는다.** 이 문서들은 과거 대화 없이도 게임 방향, 현재 상태, 품질 기준, 최신 이미지 방향과 다음 작업을 이해할 수 있도록 유지한다.

## 활성 기획서 5개

1. [게임 기획서](01_게임기획/게임기획서.md)  
   게임 규칙, 전투, 성장, 10문파, 무공서·기술, UI/UX와 밸런스 기준.

2. [프로그래밍 기획서 — 로드맵·MVP](06_프로그래밍/프로그래밍_기획서_로드맵_MVP.md)  
   Godot 구현 구조, T0·T1·T2 범위, 데이터 계약, 구현 순서와 완료 조건.

3. [아트 기획서](04_아트디자인/아트_기획서.md)  
   그림체, 문파 캐릭터, 무공서·카드, 문양·아이콘, 전투 화면, 제작·반입 파이프라인.

4. [사운드 기획서](07_사운드/사운드_기획서.md)  
   BGM, 전투·UI SFX, 절초·진의, 문파별 음색, 오디오 이벤트와 믹싱 기준.

5. [QA·PM 기획서](08_QA_PM/QA_PM_기획서.md)  
   자동·수동 테스트, 아트·사운드 검수, 일정·범위·위험·승인과 인수인계 규칙.

## 최신 이미지

최신 이미지와 시안의 상태는 [이미지 자료 인덱스](04_아트디자인/이미지_인덱스.md) 한 곳에서 관리한다. 다른 기획서는 이미지를 복제하지 않고 이 원본을 참조한다.

## 상세 명세와 부록

`docs/`, 기존 `[기획서]/` 세부 문서, `plans/`, 코드·씬·테스트는 다섯 활성 기획서를 뒷받침하는 상세 명세와 증거다. 필요한 부록은 각 활성 기획서 하단에서 연결한다.

- 기존 문서를 별도의 활성 기획서처럼 나열하지 않는다.
- 중복 내용은 다섯 문서로 흡수한다.
- 고유 수치·데이터·계약·테스트가 없는 중복 문서는 참조 검사 후 아카이브한다.

## 문서 작성 품질

- 사용자가 빠르게 이해할 수 있도록 표·제목·짧은 문단으로 작성한다.
- 새 채팅·새 작업자가 문서만 보고 같은 방향과 품질로 결과물을 만들 수 있을 만큼 상세하게 작성한다.
- 확정·진행 중·검증 필요·보류를 구분한다.
- 모든 작업에서 영향받은 활성 기획서를 같은 PR로 갱신한다.
- 이미지 작업은 아트 기획서와 이미지 인덱스를 반드시 함께 갱신한다.
- `v2`, `final`, `latest` 복제본을 만들지 않고 현재 파일을 직접 갱신한다.

## 완료 검사

```bash
python scripts/check_planning_sync.py
python tests/static_check.py
godot --headless --path . --script res://tests/run_all.gd
```

Godot 실행 환경이 없으면 앞의 두 검사를 수행하고 런타임 미검증을 명시한다.
