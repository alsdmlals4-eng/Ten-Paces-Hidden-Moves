# 십보강호 문서·자산 수명주기

## `[현행]`

현재 프로젝트가 실제로 참조하는 책임 원본·Registry·스킬·실제 파일이다.

- 기본 읽기 대상은 현재 질문에 필요한 현행 파일만 선택한다.
- 한 질문에 Markdown 또는 JSON 책임 원본 하나를 둔다.
- PDF는 최신 사람용 파생본이며 원본이 아니다.

현재 현행:

- `docs/01~11` 활성 Markdown 본책.
- 루트·허브 운영 문서와 Registry.
- 현재 Issue·Plan.
- 설치된 프로젝트 스킬.

## `[백업]`

Git 이력만으로 충분하지 않은 외부 원본·승인 기록·감사·고정 외부 링크에만 사용한다.

- `docs/[백업]/`은 기본 읽기·수정 대상에서 제외한다.
- 고유 정보·외부 참조·보존 이유를 감사하기 전 삭제하지 않는다.
- 단순 이전 버전은 새 백업 파일을 만들지 않고 Git 이력으로 보존한다.

## `[제거 후보]`

완전히 흡수됐을 가능성은 있지만 아직 삭제 승인을 받지 않은 파일이다.

필수 조건:

- 모든 고유 정보가 현행 원본에 있음.
- 이미지·표·예외·보류·미검증이 보존됨.
- 활성·보조·코드·Issue·외부 참조가 없음.
- 새 경로와 PDF·Manifest가 검증됨.
- Git 이력 또는 백업으로 복구 가능.
- Documentation Map·콜드 스타트가 정상.
- 사용자 삭제 승인.

하나라도 불확실하면 제거하지 않는다.

## 마이그레이션 적용

- 기존 `docs` 본책: `[현행]` 유지.
- 기존 `docs/[백업]/`: `[백업]` 유지.
- 루트 `[기획서]` 허브: 신규 `[현행]` 운영 라우터.
- PDF·Manifest: 생성 전 `MIGRATION_PENDING`, 생성 후 검수 상태에 따라 현행 판정.
- 기존 문서의 이동·삭제: 사용자 승인 범위 안에서만 실행.
