# 십보강호 사운드 기획서

## 0. 한눈에 보기

사운드는 화려함보다 **판정의 원인과 무협 분위기를 동시에 전달하는 것**이 우선이다. 플레이어는 화면을 보지 않아도 이동, 공격, 합, 막기, 회피, 자원 부족과 절초 가능 여부를 구분할 수 있어야 한다.

현재는 상세 제작 전 단계이며, 이 문서는 BGM·SFX·보이스를 같은 방향과 품질로 제작하기 위한 기준이다.

## 1. 사운드 팀의 책임

- 대회·문파·전투 단계에 맞는 BGM을 설계한다.
- 이동, 무기, 합, 방어, 자원, UI, 절초·진의의 SFX를 제작한다.
- 구조화된 게임 이벤트를 받아 소리를 재생하며 전투 결과를 직접 계산하지 않는다.
- 같은 규칙은 같은 청각 언어를 사용한다.
- 문파의 개성을 악기·리듬·음색으로 표현하되 정보 가독성을 해치지 않는다.
- 음량, 반복 피로, 접근성, 폴백과 누락 상태를 QA 가능한 형태로 기록한다.
- 성우 더빙은 범위 승인 전까지 MVP 필수 항목이 아니다.

## 2. 사운드 목표

### 2.1 규칙 가독성

다음 사건은 소리만으로도 구분되어야 한다.

- 행동 잠금과 양측 공개
- 접근과 후퇴
- 공격 시도와 명중
- 사거리 실패와 자원 부족
- 막기와 회피
- 합 시작·승리·패배·동률
- 피해와 방어도 변화
- 절초 기세 획득·사용 가능·예약·발동·소모
- 라운드 종료와 승패

### 2.2 무협 분위기

- 타악기, 금속 충돌, 천과 바람, 목재, 발소리 같은 물리적 질감을 우선한다.
- 내공·심법·진의에는 공명과 호흡 계열의 신비 음색을 제한적으로 사용한다.
- 모든 기술에 동일한 폭발음이나 과도한 저음을 사용하지 않는다.
- 문파별 악기와 잔향을 달리하되 합·막기·회피 같은 공통 판정음은 유지한다.

## 3. 오디오 계층

| 버스 | 역할 |
|---|---|
| Master | 전체 출력과 리미터 |
| BGM | 메뉴·대회·전투 음악 |
| SFX_Combat | 무기, 타격, 이동, 합, 상태 |
| SFX_UI | 버튼, 카드, 잠금, 공개, 경고 |
| SFX_Ultimate | 절초·진의·기세 강조층 |
| Ambience | 도관, 산림, 군영, 비무대 환경음 |
| Voice | 향후 성우·기합·호흡 |

절초·진의 전용 버스는 존재감을 주되 일반 판정음과 로그 확인음을 가리지 않아야 한다.

## 4. 전투 SFX 언어

### 4.1 이동

- 접근: 짧고 전진감 있는 발소리와 옷자락.
- 후퇴: 가벼운 마찰음과 거리감 있는 잔향.
- 보법: 문파색 음색을 더하되 기본 발소리를 완전히 지우지 않는다.
- 강제 이동: 원인 타격음, 밀려나는 마찰음, 충돌음을 분리한다.

### 4.2 무공 분류별 기본 음색

| 분류 | 기본 음색 |
|---|---|
| 검법 | 얇고 빠른 금속성 선음 |
| 도법 | 낮고 넓은 절단음 |
| 권법 | 압축된 타격과 짧은 공기 파열 |
| 장법 | 손바닥 타격 뒤 퍼지는 저중역 파동 |
| 창법 | 길고 날카로운 직선 관통음 |
| 곤봉법 | 목재·금속의 둔중한 회전과 충격 |
| 궁술 | 시위·발사·비행·명중의 분리 |
| 암기술 | 작은 금속 고주파와 빠른 다중 궤적 |

### 4.3 합

```text
합 진입 예고
→ 양측 공격 충돌
→ 짧은 긴장 정지
→ 승패 또는 동률 결과음
```

- 승리: 힘이 상대 방향으로 밀리는 결론음.
- 패배: 깨지거나 꺾이는 하강음.
- 동률: 중심에서 멈추는 균형음.
- 화면 연출을 줄여도 합 결과를 소리로 구분할 수 있어야 한다.

### 4.4 막기·회피·무효 행동

- 막기: 재질은 다르지만 공통적인 방어 확인음을 가진다.
- 회피: 피격음 없이 짧은 바람과 발소리로 성공을 전달한다.
- 사거리 실패·자원 부족·사용 조건 미달은 서로 다른 낮은 경고음을 사용한다.

## 5. 절초와 진의

### 절초

- 충전 완료, 예약, 발동은 서로 다른 소리다.
- 발동 시작에 짧고 명확한 필살기 표식을 둔다.
- BGM을 잠시 덕킹할 수 있으나 전투 흐름을 오래 끊지 않는다.
- 무기와 문파의 물리적 음색이 중심이고 저음 폭발은 보조한다.

### 진의

- 호흡, 공명, 경맥, 자연 현상 같은 음색을 사용한다.
- 공격 폭발보다 깨달음과 완성의 지속음·잔향을 강조한다.
- 심법·보법·호신공·의술 등 계통에 따라 성격을 달리한다.
- 절초와 같은 폭발음을 재사용하지 않는다.

## 6. 문파별 음색

| 문파 | 주요 음색·리듬 | 피할 방향 |
|---|---|---|
| 무당 | 호흡, 물결, 풍경, 부드러운 현악, 원형 잔향 | 과도한 폭발·공포음 |
| 화산 | 빠른 현악, 얇은 검성, 꽃잎·바람, 상승 리듬 | 둔중한 군악 |
| 소림 | 목어, 범종, 낮은 타악, 단단한 육체 타격 | 가벼운 암살 음색 |
| 당문 | 작은 금속, 침, 기계 클릭, 약병, 긴장 고주파 | 대포형 타격 |
| 개방 | 대나무 봉, 호리병, 손뼉·북, 자유로운 리듬 | 귀족적 관현악 |
| 마교 | 역재생 질감, 낮은 맥박, 금속 마찰, 혈류 저음 | 일반 교도에 천마 전용 테마 |
| 양가장 | 군북, 창·방패 금속, 규칙적인 행진 리듬 | 취중·불규칙 리듬 |
| 백우궁 | 시위, 바람, 화살 비행, 맑은 고음과 넓은 공간 | 무거운 근접음 중심 |
| 사파 | 거친 금속, 불규칙 리듬, 숨겨진 암기, 짧은 정적 | 정파식 의식음 |
| 녹림 | 나무, 가죽, 숲바람, 호각, 원거리 화살 | 설산·궁정풍 음색 |

## 7. BGM과 UI 사운드

### 7.1 BGM 화면 구분

- 메인 메뉴: 강호와 대회의 시작.
- 문파 선택: 선택 문파의 악기 레이어 강조.
- 상대 정보: 탐색과 긴장, 전투 리듬 최소화.
- 두 수 선택: 반복 피로가 적은 저밀도 루프.
- 공개·해상: 짧은 전환 신호와 리듬 상승.
- 결과·수련: 승패와 성장의 감정 분리.
- 예선 결승·결승: 기존 테마의 상위 편곡.

음악이 AI의 숨겨진 선택이나 잠금 내용을 유출해서는 안 된다.

### 7.2 UI 사운드

- 카드 선택: 종이·목재 클릭.
- 1수·2수 배치: 서로 다른 음높이.
- 잠금: 짧은 봉인음.
- 동시 공개: 양측을 한 번에 여는 공통 신호.
- 3성·7성 신규 기술, 5성·9성 강화, 10성 대성을 서로 다른 상승음으로 표현한다.

### 7.3 보이스 범위

MVP에서는 성우 대사를 필수로 하지 않는다. 후속 후보는 문파 소개, 전투 시작·승패, 절초 명칭 외침과 결승 연출이다.

## 8. 구현접근성 기준

- 사운드는 이벤트를 구독하고 상태를 수정하지 않는다.
- 이벤트에는 원인, 행위자, 대상, 문파, 분류, 결과 강도를 전달할 수 있다.
- 파일명과 이벤트 키는 안정적인 영문 ID를 사용한다.
- 자산 누락 시 무음 또는 공용 폴백과 오류 로그를 사용한다.
- 동일 이벤트 중복 재생을 막기 위해 타이밍 ID 또는 묶음 ID를 사용한다.
- BGM·전투 SFX·UI·보이스 음량을 개별 조절한다.
- 중요한 상태는 소리만이 아니라 시각 피드백도 함께 제공한다.
- 반복 기술은 여러 변형을 사용해 피로를 줄인다.

## 9. 현재 상태

### 확정

- 사운드는 구조화 이벤트의 표현 계층이다.
- 합·일반 공격·막기·회피·무효 행동을 구분한다.
- 절초와 진의는 서로 다른 청각 성격을 가진다.
- 문파별 음색과 공통 판정음을 함께 사용한다.

### 진행 중

- 실제 BGM 작곡 방향과 길이.
- SFX 자산 목록과 파일 규격.
- Godot 오디오 버스와 이벤트 연결.
- 절초 기세 상태별 사운드.

### 보류

- 전면 성우 더빙.
- 문파별 대규모 전용 대사.
