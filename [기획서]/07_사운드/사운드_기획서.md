# 십보강호 사운드 기획서

## 0. 한눈에 보기

십보강호의 사운드는 화려함보다 **판정의 원인과 무협의 분위기를 동시에 전달하는 것**이 우선이다. 플레이어는 화면을 보지 않아도 이동, 공격, 합, 막기, 자원 부족, 절초 가능 여부를 구분할 수 있어야 한다.

현재 사운드는 상세 제작 전 단계다. 본 문서는 이후 작업자가 같은 방향으로 BGM·SFX·보이스를 설계할 수 있도록 책임 범위와 품질 기준을 고정한다.

## 1. 사운드 팀의 책임

- 대회·문파·전투 단계에 맞는 BGM을 설계한다.
- 이동, 무기, 합, 방어, 자원, UI, 절초·진의의 SFX를 제작한다.
- 구조화된 게임 이벤트를 받아 소리를 재생하고 전투 결과를 직접 계산하지 않는다.
- 같은 규칙은 같은 청각 언어를 사용한다.
- 문파의 개성을 악기, 리듬, 음색으로 표현하되 전투 정보 가독성을 해치지 않는다.
- 음량, 반복 피로, 접근성, 폴백과 누락 상태를 QA 가능한 형태로 문서화한다.
- 성우 더빙은 범위가 승인되기 전까지 핵심 MVP에 포함하지 않는다.

## 2. 사운드 목표

### 2.1 규칙 가독성

다음 사건은 소리만으로도 구분되어야 한다.

- 행동 잠금
- 양측 행동 공개
- 접근과 후퇴
- 공격 시도와 명중
- 사거리 실패
- 자원 부족
- 막기
- 회피
- 합 시작
- 합 승리·패배·동률
- 피해와 방어도 변화
- 절초 기세 획득
- 절초 사용 가능
- 절초 예약
- 절초 발동과 소모
- 라운드 종료와 승패

### 2.2 무협 분위기

- 타악기, 금속 충돌, 천과 바람, 목재, 발소리 같은 물리적 질감을 우선한다.
- 신비 효과는 내공·심법·진의·문파 특수기에서 제한적으로 사용한다.
- 모든 기술에 동일한 폭발음이나 과도한 저음을 사용하지 않는다.
- 문파별 악기와 잔향을 사용하되 공통 판정음은 유지한다.

## 3. 오디오 계층

| 버스 | 역할 |
|---|---|
| Master | 전체 출력과 리미터 |
| BGM | 전투·메뉴·대회 음악 |
| SFX_Combat | 무기, 타격, 이동, 합, 상태 |
| SFX_UI | 버튼, 카드, 잠금, 공개, 경고 |
| SFX_Ultimate | 절초·진의·기세 전용 강조층 |
| Ambience | 도관, 산림, 군영, 비무대 환경음 |
| Voice | 향후 성우·기합·호흡 |

절초·진의는 별도 버스를 사용하되 Master와 Combat SFX를 압도하지 않도록 제한한다.

## 4. 전투 SFX 언어

### 4.1 이동

- 접근: 짧고 전진감 있는 발소리와 옷자락.
- 후퇴: 가벼운 마찰음과 거리감 있는 잔향.
- 보법: 문파색 음색을 더하되 기본 발소리를 완전히 지우지 않는다.
- 강제 이동: 타격음 뒤 밀려나는 마찰과 충돌을 분리한다.

### 4.2 무기와 공격

- 검: 얇고 빠른 금속성 선음.
- 도: 낮고 넓은 절단음.
- 권: 압축된 타격과 짧은 공기 파열.
- 장: 손바닥 타격 뒤 넓게 퍼지는 저중역 파동.
- 창: 길고 날카로운 직선 관통음.
- 곤봉: 목재·금속의 둔중한 회전과 충격.
- 궁: 시위, 발사, 비행, 명중을 분리.
- 암기: 작은 금속 고주파와 빠른 다중 궤적.

### 4.3 합

합은 일반 공격과 가장 명확히 구분해야 한다.

```text
합 진입 예고
→ 양측 공격 충돌
→ 짧은 긴장 정지
→ 승패 또는 동률 결과음
```

- 승리: 충돌음 뒤 상대 방향으로 힘이 밀리는 결론음.
- 패배: 충돌음 뒤 깨지거나 꺾이는 짧은 하강음.
- 동률: 중심에서 멈추는 균형음. 승패 음색을 사용하지 않는다.
- 화면 연출을 줄여도 합의 결과를 소리로 구분할 수 있어야 한다.

### 4.4 막기와 회피

- 막기: 무기·방패·호신공에 따라 재질은 다르지만 공통적인 방어 확인음을 가진다.
- 회피: 피격음이 없어야 하며, 짧은 바람과 발소리로 성공을 전달한다.
- 무효 행동: 경고 UI음과 원인별 작은 차이를 둔다.

## 5. 절초·진의 사운드

### 5.1 절초

- 필살기다운 명확한 시작 표식을 사용한다.
- 충전 완료, 예약, 발동은 서로 다른 소리다.
- 발동 시 BGM을 짧게 덕킹할 수 있으나 긴 정지는 피한다.
- 무기와 문파의 물리적 음색이 중심이고, 광대한 저음은 보조한다.
- 일반 기술보다 길 수 있지만 전투 흐름을 지나치게 끊지 않는다.

### 5.2 진의

- 공격 폭발보다 호흡, 공명, 경맥, 자연 현상 같은 음색을 사용한다.
- 깨달음과 완성의 인상을 주는 지속음과 잔향을 활용한다.
- 심법·보법·호신공 등 계통에 따라 음색을 달리한다.
- 절초와 동일한 강한 폭발음을 사용하지 않는다.

## 6. 문파별 음색 방향

| 문파 | 주요 음색·리듬 | 금지 방향 |
|---|---|---|
| 무당 | 낮은 호흡, 물결, 풍경, 부드러운 현악, 원형 잔향 | 과도한 폭발·공포음 |
| 화산 | 빠른 현악, 얇은 검성, 매화잎·바람, 상승 리듬 | 둔중한 군악 |
| 소림 | 목어, 범종, 낮은 타악, 단단한 육체 타격 | 가벼운 암살 음색 |
| 당문 | 작은 금속, 침, 기계 클릭, 약병, 긴장된 고주파 | 큰 대포형 타격 |
| 개방 | 대나무 봉, 호리병, 손뼉·북, 자유로운 리듬 | 지나치게 귀족적인 관현악 |
| 마교 | 역재생 질감, 낮은 맥박, 금속 마찰, 혈류형 저음 | 일반 교도에 천마 전용 테마 사용 |
| 양가장 | 군북, 창·방패 금속, 규칙적인 행진 리듬 | 흐트러진 취중 리듬 |
| 백우궁 | 시위, 바람, 화살 비행, 맑은 고음과 넓은 공간감 | 무거운 근접 타격 중심 |
| 사파 | 거친 금속, 불규칙 리듬, 숨겨진 암기, 짧은 정적 | 정파식 정갈한 의식음 |
| 녹림 | 나무, 가죽, 숲바람, 호각, 멀리서 오는 화살 | 설산·궁정풍 고급 음색 |

## 7. BGM 구조

### 7.1 화면별 기본 구분

- 메인 메뉴: 강호의 넓이와 대회의 시작을 암시.
- 문파 선택: 선택한 문파의 악기 레이어를 짧게 강조.
- 상대 정보: 긴장과 탐색, 과도한 전투 리듬 금지.
- 비무 준비: 잠금과 예측을 위한 낮은 긴장.
- 전투: 두 수 선택 구간과 공개·해상 구간의 리듬 차이.
- 합·절초: 짧은 음악 반응 또는 스팅어.
- 결과·수련: 승패와 성장의 감정 분리.
- 예선 결승·결승: 기존 테마를 확장한 상위 편곡.

### 7.2 적응형 음악 원칙

- 선택 중에는 반복 피로가 적은 저밀도 루프.
- 공개 시 짧은 전환 신호.
- 체력·기세·결승 단계에 따라 레이어를 추가할 수 있다.
- 음악이 AI 잠금 시점이나 숨겨진 행동을 유출해서는 안 된다.

## 8. UI 사운드

- 카드 선택: 가벼운 종이·목재 클릭.
- 1수·2수 슬롯 배치: 서로 다른 음높이로 순서를 구분.
- 잠금: 확정감 있는 짧은 봉인음.
- 공개: 양측이 동시에 열리는 한 번의 공통 신호.
- 선택 불가: 원인 설명을 방해하지 않는 낮은 경고음.
- 성급 상승: 3·7성 신규 기술, 5·9성 강화, 10성 대성을 서로 다르게 표현.
- 절초·진의 해금: 일반 성급 상승보다 특별하지만 지나치게 길지 않게.

## 9. 보이스 범위

### MVP

- 성우 대사 없음.
- 필요하면 짧은 호흡·기합을 공용 폴백으로 사용.

### 후속 후보

- 문파 선택 소개.
- 전투 시작·승리·패배 한 줄.
- 절초 명칭 외침.
- 상대 이명과 결승 연출.

보이스를 도입할 경우 자막과 음성 언어 독립, 재생 중복, 스킵 규칙을 함께 설계한다.

## 10. 구현 계약

- 사운드는 이벤트를 구독하고 게임 상태를 수정하지 않는다.
- 이벤트에는 원인, 행위자, 대상, 무공 분류, 문파, 결과 강도 정보가 포함될 수 있다.
- 파일명과 이벤트 키는 안정적인 영문 ID를 사용한다.
- 자산 누락 시 무음 또는 공용 폴백을 사용하고 오류 로그를 남긴다.
- 동일 이벤트가 중복 재생되지 않도록 타이밍 ID 또는 묶음 ID를 사용한다.

## 11. 접근성과 믹싱

- 중요한 상태는 소리만으로 전달하지 않고 시각 피드백을 병행한다.
- BGM·전투 SFX·UI·보이스 음량을 개별 조절한다.
- 갑작스러운 고주파와 저주파 충격을 제한하는 옵션을 고려한다.
- 반복 기술의 음색을 2~4개 변형해 피로를 줄인다.
- 절초 연출 중 일반 타격·로그 확인음이 완전히 묻히지 않게 한다.

## 12. 현재 상태

### 확정

- 사운드는 구조화 이벤트의 표현 계층이다.
- 합·일반 공격·막기·회피·무효 행동을 구분한다.
- 절초와 진의는 서로 다른 청각 성격을 가진다.
- 문파별 음색을 사용하되 공통 판정음은 유지한다.

### 진행 중

- 실제 BGM 작곡 방향과 길이.
- SFX 자산 목록과 파일 규격.
- Godot 오디오 버스와 이벤트 연결.
- 절초 기세의 정확한 상태별 사운드.

### 보류

- 전면 성우 더빙.
- 문파별 대규모 전용 대사.

## 13. 다음 작업

1. T0 전투 이벤트 기준 최소 SFX 목록 작성.
2. 합·막기·회피·절초 가능의 회색박스 오디오 프로토타입 제작.
3. Godot 오디오 버스와 폴백 구조 구현.
4. 무당파 기술을 기준으로 문파 음색 차이 검증.
5. 실제 플레이테스트에서 정보 인지와 반복 피로를 측정.

BGM·SFX·보이스·오디오 이벤트·믹싱·접근성 방향이 바뀌면 이 문서를 같은 작업에서 갱신한다. 전투 이벤트와 절초 상태가 바뀌면 게임·프로그래밍·QA 기획서도 함께 확인한다.
