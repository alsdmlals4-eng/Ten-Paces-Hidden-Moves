# 십보강호 활성 컨텍스트

> 목적: 작업자가 바뀌어도 과거 대화 없이 현재 방향, 확정 규칙, 미확정 범위와 다음 작업을 복원하게 한다. 모든 작업 종료 시 이 문서를 갱신한다.

## 1. 현재 상태

- 현재 단계는 **전투·성장·대회·연출 구조의 승인 기획 정리와 구현 인수 준비**다.
- 다음 기획 작업은 **절초 기세·상단 HUD UX 설계**다.
- Godot 프로젝트·스크립트·데이터·테스트의 실제 존재와 구현 완료 여부는 아직 확인하지 않았다.
- 구현 AI는 `AGENTS.md`, 이 문서, `DOCUMENTATION_MAP.md`, 연결된 기준 문서를 읽고 Plan Mode 감사를 먼저 수행한다.
- 사용자 승인 전에는 코드·씬·Resource·저장 구조를 생성하거나 수정하지 않는다.
- `docs/[백업]/`은 과거 기록, `docs/[보류]/`는 재개 전 구현 금지 영역이다.

## 2. 제품 방향

- 장르: 1대1 무협 로그라이트 비무 대회.
- 핵심 경험: 상대의 거리·자원·성향을 읽고 두 수를 비공개로 잠근 뒤 동시에 공개하는 예측전.
- 내부 시스템은 수치와 데이터로 정확히 계산한다.
- 플레이어에게는 불필요한 개발용 분류를 직접 노출하지 않고 **무림식 이명·풍문·전적·정탐 문장**으로 암시한다.
- 같은 표현은 같은 의미를 가져 반복 플레이로 학습 가능해야 하며 강한 적을 약한 표현으로 속이지 않는다.
- 연출은 규칙을 숨기는 장식이 아니라 상대 읽기, 위험 선택, 예측, 결과 원인, 성장 보상을 이해시키는 프레젠테이션 계층이다.
- 프레젠테이션은 피해·보상·수련·저장을 결정하지 않는다.

## 3. 고정 숫자 구조

- 전장: 10칸.
- 라운드: 행동 타이밍 10개.
- 선택: 2타이밍씩 묶어 행동 두 개를 잠금·동시 공개·순차 해상, 총 5회.
- 전체 대회: 10전, 8전 8강·9전 준결승·10전 결승.
- 데모: 1~5전.
  - 1~2전: 내부전.
  - 3~4전: 예선.
  - 5전: 예선 결승이자 데모 최종전.

## 4. 전투 핵심

- AI는 각 2수 묶음을 플레이어 선택 전에 공개 정보만으로 잠근다.
- 양측 행동 두 개를 함께 공개한 뒤 1수와 2수 순으로 해상한다.
- 첫 수 해상 뒤 둘째 수를 변경할 수 없다.
- 같은 타이밍에 양측 공격이 모두 유효하면 `[합]`을 판정한다.
- 낮은 최종 합 수치 쪽이 차이만큼 피해를 받고 동률은 피해가 없다.
- 무당 태극권은 상대 공격을 이용하는 합 반격 특화다.
- 라운드 종료 시 행동력 전부, 기력·내공은 소량 회복한다.
- 공용 전투 해상과 구조화 이벤트가 결과를 소유하며 UI·VFX·애니메이션은 이를 표현한다.

## 5. 무공·심법

- 각 핵심무공 종류는 관련 문파 전용 1종과 공용 2종을 기본 콘텐츠 단위로 한다.
- 핵심무공은 여러 개 보유 가능하다.
- 심법은 보유 자체가 최대 2개다.
- 시작 문파 핵심무공과 심법은 각각 3성이다.
- 성급 `n`에 도달하는 비용은 `n`포인트다.
- 핵심무공은 3·6·9성에 기술, 10성에 절초를 얻는다.
- 심법은 3·6·9성에 패시브·기믹 생성/강화·명상 효과 등을 조합하고 10성에 진의를 얻는다.
- 핵심무공 10성은 최대 1개, 심법 10성은 최대 1개다.
- 심법은 관련 핵심무공을 단방향으로 강화한다.
- 화산의 `[연격]` 같은 핵심 기믹은 심법 3성에서 생성하고 10성 진의에서 운용을 강화한다.
- 문파 효과는 가능한 한 공용 비용·합·사거리·상태·명상·연계 수정자로 표현한다.

### 5.1 절초 해금과 전투 발동 UX

상태: **구조 승인 / 정확한 수치·명칭 미확정 / 다음 기획 작업**.

절초는 두 개의 서로 다른 조건을 가진다.

```text
핵심무공 10성 → 절초 자체 해금
전투 중 기세 충전 → 해당 전투에서 절초 발동 가능
```

승인된 방향:

- 플레이어와 상대의 절초 기세를 전투 화면 상단 HUD에 각각 표시한다.
- 짧은 칸형 게이지, 절초 아이콘과 발동 가능 상태를 사용한다.
- 라운드 진행, 합, 회피, 막기 등 전투 참여와 판단 결과가 기세 축적 후보가 된다.
- 점수 획득 시 증가량뿐 아니라 `합`, `회피` 같은 이유를 함께 표시한다.
- 조건 미달 시 절초 행동의 잠금 이유를 표시한다.
- 조건 달성 시 상단 HUD와 절초 행동에서 `절초 가능` 상태를 명확히 강조한다.
- 절초를 1수·2수 슬롯에 배치하면 발동 예약 상태를 표시하고 정상 사용 뒤 기세를 소모한다.
- UI·VFX·오디오는 절초 해금·누적·소모를 직접 계산하지 않는다.

다음 작업에서 확정할 내용:

- 자원 명칭, 최대치와 발동 요구치.
- 라운드·합·합 승리·회피·막기·피격 등의 획득량과 중복 상한.
- 시작값, 전투·라운드 간 유지 여부와 초기화 규칙.
- 사용·실패·취소 시 소모 규칙.
- 상대 게이지 공개 범위와 AI 사용 기준.
- 10성 미해금 상태의 HUD 표시 방식.
- 툴팁·로그·접근성 폴백.

세부 작업과 완료 기준은 `04_ROADMAP.md`의 `단계 0B — 다음 기획 작업`을 따른다.

## 6. 성과·행운·수련

### 고정 정산식

```text
최종 수련 포인트 = 승리 기본 5 + 등급 보너스 + 행운 앞면 수
```

### 등급 보너스

| 등급 | 보너스 |
|---|---:|
| C | +1 |
| B | +3 |
| A | +5 |
| S | +7 |

- A는 의도한 평균적 전투 수행 등급이다.
- 문파별 기믹 수행도 평가는 작업량 때문에 현재 `[보류]`다.

### 행운 결과표

행운 다이스 3개는 비공개로 굴린다. 독립 확률 3회가 아니라 등급별 `앞면 0~3개` 결과표에서 한 번 추첨한다.

| 등급 | 0개 | 1개 | 2개 | 3개 | 평균 행운 |
|---|---:|---:|---:|---:|---:|
| C | 0% | 0% | 5% | 95% | 2.95 |
| B | 5% | 5% | 5% | 85% | 2.70 |
| A | 20% | 5% | 20% | 55% | 2.10 |
| S | 60% | 10% | 15% | 15% | 0.85 |

플레이어 표시는 `B`, `B+`, `B++`, `B+++`처럼 앞면 수만 `+`로 붙인다. 개별 다이스 면은 공개하지 않는다.

### 데모 절초 목표

- 핵심무공 3성→10성에는 49포인트가 필요하다.
- 첫 4전 수련을 핵심무공에 전부 투자한다.
- 목표 확률:
  - A 4회: 약 50%.
  - A 3회 + S 1회: 약 60%.
  - A 3회 + B 1회: 약 30%.
- 절초는 데모 진행 필수가 아니라 높은 성과와 행운으로 예선 결승에서 조기 체험하는 하이라이트다.
- 정식판에서는 이후 수련 기회가 이어져 절초를 더 자주 사용한다.

## 7. 상대 성장·수련 스타일

- 상대의 기준 등급 분포는 A계열 60%, B계열 40%다.
- 상대도 등급에 맞는 행운 결과표에서 `+`를 추첨하고 실제 수련 포인트에 반영한다.
- 상대의 등급, 행운, 투자 결과, 수련 스타일은 회차에서 처음 생성할 때 확정한다.
- 재도전과 저장·불러오기에서 같은 상대의 성장 결과를 다시 굴리지 않는다.

내부 종합 강도:

1. 많이 약한 적.
2. 평균보다 약한 적.
3. 중간.
4. 강적.
5. 매우 강한 적.

내부 수련 스타일:

- 균형형: 핵심무공과 심법을 고르게 투자.
- 핵심무공 올인형: 절초와 6·9성 기술을 우선.
- 심법 연계형: 문파 기믹과 명상·자원 순환을 우선.
- 다무공형: 주력·심법·공용/외문 무공으로 분산 투자.

## 8. 상대 정보의 플레이어 표현

다음 개발용 태그는 플레이어에게 직접 보여주지 않는다.

```text
[강적] [A++] [핵심무공 올인형]
```

대신 다음 층으로 암시한다.

- 이명·수식언: 종합 강도와 대표 전법.
- 풍문·전적: 행운과 최근 성장세.
- 무공 평가 문장: 투자 성향과 성급 경계.
- 정탐: 같은 의미를 더 구체적인 무림식 문장으로 제공.

예시:

```text
[매화일섬] 낙유진
“검 한 자루만을 벼려 예선의 이름난 자들을 연달아 꺾었다.”
“최근 두 차례의 비무에서 기연을 얻은 뒤 검세가 크게 달라졌다고 한다.”
```

- 이명과 문구는 무작위 장식이 아니라 실제 내부 데이터와 연결한다.
- 같은 어휘 계열은 같은 위험도·성향을 반복적으로 암시한다.
- 정탐은 개발용 수치를 그대로 노출하기보다 `절초의 문턱`, `내외겸수`, `한 계통에 매진`처럼 표현을 더 구체화한다.
- 플레이어는 이 정보를 보고 `삼수양보` 같은 제약의 위험도를 선택한다.

## 9. 전투·대회 연출 방향

전체 책임 원본은 `10_COMBAT_PRESENTATION_PLAN.md`다.

```text
상대 읽기
→ 제약 선택
→ 전투 준비
→ 2수 잠금·동시 공개
→ 1수·2수 해상
→ 일반 공격·방어·합 결과
→ 성과·행운 정산
→ 수련·성급·절초 해금
→ 다음 상대 또는 5전 데모 종료
```

고정 원칙:

- 한 화면에는 하나의 중심 질문을 둔다.
- 카드 본체, 호버 팝업, 클릭 상세의 정보 역할을 분리한다.
- AI 사전 잠금과 잠금 뒤 변경 불가를 화면에서 신뢰할 수 있게 표시한다.
- 합 연출의 여러 공방은 한 번의 계산을 표현한다.
- 기본·등급·행운·제약 보상을 따로 설명한 뒤 합산한다.
- 절초는 하이라이트지만 미보유 경로를 막지 않는다.
- 절초 기세 HUD는 해금 상태·축적 이유·발동 가능·예약·소모를 구분한다.
- 모션 감소·점멸 감소·색상 독립·자산 누락 폴백을 제공한다.
- 연출 계층은 전투·성장 결과와 저장 상태를 결정하지 않는다.

## 10. 제약

- 전투 시작 전에 무협식 제약을 자청한다.
- 예: 삼수양보, 단전봉맥, 기혈역행, 일장지양.
- 특정 문파의 주력 계통을 완전히 봉쇄하는 제약은 만들지 않는다.
- 승리했을 때만 추가 수련·보상 선택지·새로고침·고급 보상 확률 등을 받는다.
- 상대 이명·풍문·정탐 정보를 확인한 뒤 제약을 고르는 흐름이다.

## 13. 구현 전 남은 결정

1. 절초 기세의 명칭, 최대치, 획득·중복 상한·소모·초기화 규칙과 상단 HUD 상태.
2. 등급 C/B/A/S 판정용 공용 지표, 가중치와 임계값.
3. 상대 강도 5단계와 A/B/행운의 경기별 출현표.
4. 이명·수식언·풍문·전적·정탐 의미 키와 첫 문장 사전.
5. 수련 스타일별 포인트 투자 알고리즘과 3·6·9·10성 경계 우선순위.
6. 제약 후보 수, 선택 상한, 위험도와 추가 보상.
7. 행동력·기력·내공의 정확한 카드 비용과 라운드 회복량.
8. 1수 결과 체력 0일 때 2수 취소·표현 규칙.

## 15. Base 적용·학습 순환

- Base의 공용 규칙·method·skill·template·case는 문서 지도에서 현재 작업에 필요한 원본만 읽는다.
- 프로젝트 적용 결과와 더 구체적인 규칙은 `11_BASE_ADOPTION_AND_LEARNING_LOG.md`에 기록한다.
- 프로젝트 고유 이름·수치·세계관·Godot 경로는 Base로 복사하지 않는다.
- 반복 가능한 교훈은 기존 Base 책임 문서와 중복을 확인한 뒤 case·method·skill·template 중 적절한 위치에 반영한다.
- 실제 구현·플레이 검증이 없는 설계는 바로 검증된 skill로 승격하지 않는다.
