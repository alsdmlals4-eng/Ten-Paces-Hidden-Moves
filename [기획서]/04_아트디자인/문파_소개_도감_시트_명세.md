# 문파 소개·도감 시트 명세

## 0. 목적

문파는 플레이어가 선택하는 클래스가 아니라, 강호낭인이 등반 중 조우하고 교류하는 세계관 세력이다. 문파를 처음 조우했을 때와 도감에서 **대표 인물·문파 설명·추구하는 무학·핵심 무공서**를 짧고 명확하게 보여준다.

## 1. 표시 시점

### 처음 조우

- 문파 또는 핵심 인물을 처음 만났을 때 한 번 표시한다.
- 전투 흐름을 끊지 않도록 짧은 소개 화면으로 사용한다.
- 확인 후 즉시 사건·교류·비무 화면으로 이동한다.

### 도감

- 조우한 문파의 전체 소개 시트를 다시 볼 수 있다.
- 발견한 핵심 무공서는 이름과 표지를 공개한다.
- 아직 발견하지 않은 인물·무공서는 실루엣 또는 `미확인`으로 표시할 수 있다.

## 2. 필수 정보 구조

```text
문파 문양 + 문파명
문파 설명
추구하는 무학
대표 캐릭터 1~2명
핵심 무공서 2~4권
```

| 항목 | 역할 | 권장 분량 |
|---|---|---:|
| 문파명·문양 | 즉시 식별 | 이름 1줄 |
| 문파 설명 | 역사·거점·강호 내 위치와 성격 | 2~4줄 |
| 추구하는 무학 | 문파 전체가 무공을 통해 이루려는 경지와 전투 원칙 | 1~2문장 |
| 대표 캐릭터 | 복식·무기·자세로 문파의 인상 전달 | 1~2명 |
| 핵심 무공서 | 해당 세력을 대표하는 전승 | 2~4권 |

## 3. 용어 책임

서로 다른 층위의 철학 문구를 다음처럼 구분한다.

- **추구하는 무학**: 문파·가문·유파 전체가 지향하는 무학과 전투관. 기존 문파 소개에서 사용하던 `무공의 묘리` 명칭을 이 용어로 교체한다.
- **무도(武道)**: 무공서 한 권이 추구하는 길과 수련 원리.
- **기술 묘리**: 개별 3성·7성·10성 초식의 핵심 원리를 카드 하단에 최대 두 줄로 표시하는 문장.

문파 시트에는 `추구하는 무학`, 무공서 상세에는 `무도`, 기술 카드에는 `기술 묘리`를 사용한다.

## 4. 권장 레이아웃

```text
┌─────────────────────────────────────┐
│ 문양  문파명             문파 설명  │
│                         추구하는 무학│
├──────────────────┬──────────────────┤
│ 대표 캐릭터 1~2명 │ 핵심 무공서 2~4권│
│                  │ 표지 + 이름 + 분류│
└──────────────────┴──────────────────┘
```

- 좌측 50~60%: 대표 캐릭터와 문파 거점 배경.
- 우측 상단: 문파 설명과 추구하는 무학.
- 우측 하단: 핵심 무공서 표지.
- 처음 조우 화면은 정보를 축약하고, 도감에서는 전체 문구와 발견 상태를 표시한다.
- 기술 카드 3장 전체를 문파 소개 시트에 강제로 넣지 않는다. 기술은 무공서 상세 화면에서 확인한다.

## 5. 합성 파이프라인

### 이미지 자산

- 대표 캐릭터와 거점 배경.
- 문파 문양.
- 핵심 무공서 표지.
- 공통 프레임과 장식.

### 편집 가능한 UI 데이터

- 문파명.
- 문파 설명.
- 추구하는 무학.
- 인물명과 직위.
- 무공서 이름·분류·발견 상태.
- 현재 관계와 교류 가능 상태가 필요할 경우 별도 UI로 표시한다.

생성 이미지에 포함된 글자는 `[아트 방향 참고용]`이며 최종 기준 데이터로 사용하지 않는다.

## 6. 작성 기준

### 문파 설명

- 역사 설명만 나열하지 않는다.
- 현재 강호에서 무엇을 지키고, 누구와 충돌하며, 강호낭인에게 어떤 존재인지 보여준다.
- 핵심 무공서의 전투 색채가 자연스럽게 연결되어야 한다.

### 추구하는 무학

- 단순 구호가 아니라 해당 문파의 전투 원칙이 드러나야 한다.
- 피해량이나 게임 수치를 직접 쓰지 않는다.
- 문파의 모든 핵심 무공을 포괄할 수 있어야 한다.
- 1~2문장으로 읽히게 압축한다.

예시:

> **무당파 — 추구하는 무학**  
> 억지로 힘을 겨루지 않고 흐름을 받아 돌려보내며, 검과 권과 호흡을 하나의 태극으로 완성한다.

## 7. 검증 기준

- 문파를 플레이어 클래스처럼 소개하지 않는다.
- 문파 설명과 추구하는 무학의 내용이 중복되지 않는다.
- 핵심 무공서 명칭과 분류는 게임 데이터와 일치한다.
- 캐릭터·복식·무기·거점이 같은 세력으로 읽힌다.
- 문파 시트의 모든 텍스트는 교체 가능한 UI 레이어다.
- 처음 조우 화면은 한 번에 읽을 수 있고 전투 진입을 지연시키지 않는다.
