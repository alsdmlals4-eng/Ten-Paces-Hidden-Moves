# 십보강호 아트 기획서

## 0. 한눈에 보기

시각 방향은 **선명한 반실사 무협 인물 + 수묵·한지 질감의 배경 + 먹색·금색 게임 UI**다. 인물·무기·기술 동작은 선명하게, 배경과 장식은 분위기를 만들되 정보 식별을 방해하지 않게 한다.

주인공은 특정 문파에 소속되지 않은 강호낭인이다. 문파·가문·유파는 플레이어 클래스가 아니라 등반 중 조우하는 세력과 무공서의 출처로 표현한다.

최신 이미지 자료의 유일한 책임 원본은 [이미지 자료 인덱스](이미지_인덱스.md)다.

## 1. 아트 팀의 책임

- 캐릭터, 배경, 무공서, 카드, 문양, 분류 아이콘, UI 장식, 지도 노드, 애니메이션과 VFX를 제작한다.
- 전투 규칙과 무공의 차이를 빠르게 읽게 만드는 시각 언어를 설계한다.
- 같은 세력과 무공 분류는 모든 화면에서 같은 문양·색·실루엣을 사용한다.
- 생성 이미지 속 잘못된 한글·수치·분류를 최종 자산으로 확정하지 않는다.
- 이미지의 경로·승인 상태·적용 범위를 이미지 인덱스에서 관리한다.
- 새 작업자가 이 문서만 보고 같은 품질로 제작할 수 있도록 색·구도·금지 요소를 기록한다.

## 2. 공통 스타일

### 2.1 그림체

- 캐릭터: 반실사 동양 무협 게임 일러스트.
- 얼굴·손·무기·복식 윤곽: 선명하고 정확하게.
- 배경: 수묵화와 한지 질감, 낮은 대비.
- 공통 UI: 먹색·짙은 갈색·금색.
- 세력색: 복식 안감, 무공서 테두리, 기술 궤적에 제한적으로 사용.
- 강한 광원: 절초·진의와 핵심 순간에만 사용.

### 2.2 품질 기준

- 축소 화면에서도 무기·자세·세력이 구분된다.
- 분류 아이콘은 텍스트 없이 실루엣만으로 읽힌다.
- 남녀 캐릭터는 얼굴만 바꾼 복제가 아니라 체형·자세·실루엣이 다르다.
- 기술 삽화는 이름을 읽지 않아도 공격 방식이 추정된다.
- 세력별 기술 궤적이 다르다. 모든 세력에 음양 원·금빛 폭발을 반복하지 않는다.
- 손가락·팔·무기 결합과 관절 방향을 최종 승인 전에 검수한다.

## 3. 문파 소개·도감 시트

문파 소개 시트는 플레이어의 시작 클래스 화면이 아니다. 강호낭인이 문파를 처음 조우했을 때 짧게 보여주고, 이후 도감에서 다시 확인하는 **세계관 세력 소개 화면**이다.

상세 규칙은 [문파 소개·도감 시트 명세](문파_소개_도감_시트_명세.md)를 따른다.

### 3.1 필수 정보

```text
문파 문양 + 문파명
문파 설명
추구하는 무학
대표 캐릭터 1~2명
핵심 무공서 2~4권
```

- **문파 설명**: 역사·거점·강호 내 위치와 현재 성격을 2~4줄로 설명한다.
- **추구하는 무학**: 문파 전체가 무공을 통해 이루려는 경지와 전투 원칙을 1~2문장으로 표현한다.
- **대표 캐릭터**: 복식·무기·자세로 문파의 인상을 전달한다.
- **핵심 무공서**: 해당 문파를 대표하는 표지와 이름·분류를 보여준다.

### 3.2 권장 레이아웃

```text
상단: 문파 문양 / 문파명 / 문파 설명 / 추구하는 무학
좌측: 대표 캐릭터 1~2명과 거점 배경
우측: 핵심 무공서 2~4권
```

- 좌측 50~60%는 인물과 거점, 우측은 문파 설명과 핵심 무공서에 사용한다.
- 인물은 겹치지 않는 독립 실루엣으로 구성한다.
- 처음 조우 화면은 축약본, 도감은 전체 문구와 발견 상태를 표시한다.
- 기술 카드 3장 전체를 문파 소개 시트에 강제로 넣지 않는다. 기술은 무공서 상세에서 확인한다.
- 문파명·설명·추구하는 무학·무공서명은 이미지에 굽지 않고 편집 가능한 UI 레이어로 합성한다.

### 3.3 용어 책임

- **추구하는 무학**: 문파·가문·유파 전체의 무학과 전투관. 문파 소개에서 사용하던 구 `무공의 묘리` 명칭을 이 용어로 교체한다.
- **무도(武道)**: 무공서 한 권이 추구하는 길과 수련 원리.
- **기술 묘리**: 개별 초식의 핵심 원리를 카드 하단에 표시하는 짧은 문장.

서로 다른 층위의 문구를 섞어 쓰지 않는다.

## 4. 무공서와 기술 카드

### 4.1 무공서 정보 구조

무공서 한 권은 다음 정보를 가진다.

```text
무공서 표지
무공서 이름·분류·출처
무공 설명
무도(武道)
3성 기술 / 7성 기술 / 10성 절초·진의
```

- 무공 설명은 기원과 전투 운용을 설명한다.
- 무도는 해당 무공이 추구하는 원리와 경지를 표현한다.
- 무공서 표지는 전승 집단의 상징과 무공 특성을 보여주되 생성형 한글을 기준 데이터로 사용하지 않는다.

### 4.2 기술 카드 앞면

- 좌상단: 무공서 출처 문양. 개방·작자 불명 전승은 `무(無)` 상징 이미지.
- 최상단 중앙: 사거리 슬롯. 숫자는 UI 레이어.
- 상단 중앙: 기술 이름. 편집 가능한 텍스트.
- 우상단: 검법·권법·심법 등 무공 종류의 아이콘 이미지.
- 중앙: 기술 삽화.
- 하단: 비용·효과 없이 **기술 묘리** 한 소절.

### 4.3 기술 묘리 문구

- 최대 두 줄.
- 한 줄 14~18자 권장.
- 기존 예시보다 약 1.4~1.6배 큰 글자.
- 가운데 정렬, 굵고 선명한 붓글씨.
- 삽화와 분리된 바탕과 충분한 여백.
- 문장이 길면 글자를 줄이지 않고 문장을 압축한다.

### 4.4 절초와 진의

**절초**

- 우측 세로 `절초` 표찰.
- 금장과 출처 대표색의 강한 테두리.
- 무기와 동세가 크게 보이는 필살기 연출.
- 일반 카드보다 화면 돌파감과 공격성을 강화한다.

**진의**

- 우측 세로 `진의` 표찰.
- 경맥·자연·호흡·깨달음의 상징.
- 절초보다 고요하지만 신비롭고 완성된 인상.
- 단순 공격 장면이 아니라 무공 원리가 완성되는 순간을 표현한다.

## 5. 문양과 아이콘

### 5.1 세력·출처 문양

| 출처 | 중심 상징 | 대표색 |
|---|---|---|
| 무당 | 태극·구름·산봉 | 남청·백색 |
| 화산 | 매화 다섯 잎·검광 | 백색·매화홍 |
| 소림 | 범종 또는 연꽃·불광 | 황토·금색 |
| 당문 | 비도·침·독방울 | 암청·자주 |
| 개방 | 대나무 봉·호리병 | 녹갈·황토 |
| 마교 | 균열난 원·역류 혈염 | 흑색·심홍 |
| 양씨 군문 | 창·방패·군기 | 군청·은색 |
| 설산 궁술 유파 | 활·화살·깃털 | 설백·빙청 |
| 강호 비정통 전승 | 외날도·비도·갈라진 원 | 암자주·먹색 |
| 산채·녹림 전승 | 산봉·창·화살·목책 | 짙은 녹색·갈색 |
| 개방·작자 불명 | 무(無) 원형 상징 | 먹색·낡은 금색 |

문양 안에 세력 이름을 넣지 않는다. 문양은 출처 표시이며 자동 사용 제한이나 보너스를 뜻하지 않는다.

### 5.2 무공 종류 아이콘 16종

| 종류 | 모티브 |
|---|---|
| 검법 | 곧은 양날검 |
| 도법 | 넓은 외날도와 초승달 참격 |
| 권법 | 정면 주먹 |
| 장법 | 펼친 손과 파동 원 |
| 창법 | 창날과 직선 궤적 |
| 곤봉법 | 봉과 회전 원호 |
| 궁술 | 당겨진 활과 화살 |
| 암기술 | 방사형 침·비도 3개 |
| 심법 | 좌선 인물과 단전 원 |
| 호신공 | 종 또는 보호막 |
| 독공 | 독방울과 뱀 |
| 마공 | 역회전 균열 문양 |
| 보법 | 발자국과 구름 잔상 |
| 방패술 | 정면 원형 방패 |
| 외공 | 암석처럼 갈라진 상체·팔 |
| 의술 | 금침과 약초 잎 |

아이콘 내부에 한글·한자를 넣지 않는다.

## 6. 세력별 아트 바이블

### 무당파
남청·백색 도포, 직검, 원형 검세와 권장, 운해·도관. 키워드: 균형·흐름·절제. 불꽃·중갑 금지.

### 화산파
백색·매화홍, 가벼운 검사, 절벽·매화. 키워드: 사선 검광·꽃잎·상승. 둔중한 전사형 금지.

### 소림사
황토·적갈 승복, 단단한 체형, 권·봉, 산문·석등. 키워드: 낮은 중심·불광·암석. 암살자형 금지.

### 당문
암청·자주, 몸에 붙는 복식, 비도·침통·약낭, 대나무·약방·기관. 키워드: 정밀·은폐·독과 의술. 중갑 금지.

### 개방
녹갈·황토, 덧댄 천옷, 대나무 봉·호리병, 장터·황야. 키워드: 호쾌함·먼지·변칙. 귀족풍 금지.

### 마교
흑색·심홍, 비대칭 겹옷·부분 갑주, 폐허·재. 키워드: 혈염·역류·과부하. 일반 교도에게 천마 문양 금지.

### 양씨 군문 전승
군청·은색, 실전 군복·장창·방패, 성벽·변방 진지. 키워드: 대오·수비·직선 돌파.

### 설산 궁술 유파 전승
설백·빙청, 장궁·팔 보호대·흐르는 천, 고지대 활터. 키워드: 정밀·풍향·백색 궤적. 산적형 금지.

### 강호 비정통 전승
암자주·먹색, 비대칭 실전 장비·흉터·도·암기, 폐허 객잔. 키워드: 사각·난전·생존. 정제된 정파 제복 금지.

### 산채·녹림 전승
짙은 녹색·가죽 갈색, 산악 생존복·창·활·밧줄, 산채·목책. 키워드: 매복·길목·낙엽·흙먼지. 귀족 궁수와 엘프풍 금지.

## 7. 전투와 등반 화면

### 7.1 전투 화면

- 중앙 비무 장면이 감정 중심이다.
- 양측 캐릭터는 전장 양끝에서 서로 마주본다.
- 전장 10칸이 배경 장식보다 우선해 읽힌다.
- 상단은 양측 상태와 절초 기세, 하단은 기술 목록과 두 수 슬롯.
- 합은 충돌점, 일반 공격은 일방향 타격으로 구분한다.

### 7.2 등반 지도

- 산 아래에서 정상으로 이어지는 세로형 분기 구조.
- 비무·강적·수련·기연·문파 교류·객잔·장터·돌파 노드를 무협식 문양으로 구분한다.
- `?` 사건은 실제 내용을 숨기되 위험도나 지역 분위기는 배경으로 암시할 수 있다.
- 현재 위치, 선택 가능한 다음 길과 되돌릴 수 없는 경로를 명확히 표시한다.

### 7.3 단일 이벤트 화면

- 인물 또는 장소 일러스트 1장과 짧은 상황문을 사용한다.
- 선택지는 2~4개를 한 화면에 표시한다.
- 능력치·성향·관계·무공 조건으로 잠긴 선택지는 조건과 함께 보여준다.
- 이벤트는 짧은 결과 후 즉시 지도나 전투로 복귀한다.

## 8. 제작 파이프라인

1. 게임 기획서의 명칭·분류·성급·출처 구조 확인.
2. 아트 기획서와 이미지 인덱스의 최신 방향 확인.
3. 텍스트 없는 캐릭터·배경·무공서·카드 삽화 제작.
4. 세력 문양·분류 아이콘·절초·진의 표찰을 별도 자산으로 제작.
5. 문파명·문파 설명·추구하는 무학·무공서명·기술명·사거리·기술 묘리를 편집 가능한 레이어로 합성.
6. 사용자 승인 뒤 원본 저장소 반입.
7. 이미지 인덱스에 경로·상태·적용 범위 기록.
8. 실제 UI 축소 크기에서 가독성과 데이터 매핑 검수.

## 9. 현재 상태

### 확정

- 반실사 인물 + 수묵 배경 + 먹색·금색 UI.
- 주인공은 특정 문파 복식으로 고정되지 않는 강호낭인.
- 문파는 처음 조우와 도감에서 보는 세계관 세력 소개 시트.
- 문파 시트의 `캐릭터 + 문파 설명 + 추구하는 무학 + 핵심 무공서` 구조.
- `추구하는 무학 / 무도 / 기술 묘리`의 층위별 용어 분리.
- 기술 카드 앞면 구조와 큰 기술 묘리 문구.
- 분류를 텍스트가 아닌 아이콘으로 표시.
- 절초·진의 세로 표찰.
- 이미지와 텍스트·수치의 편집형 합성 파이프라인.

### 진행 중

- 문파·가문·유파 소개 시트의 최종 레이아웃.
- 세력·출처 문양과 무공 종류 아이콘 최종 원본.
- 각 세력의 문파 설명·추구하는 무학 작성.
- 무공서 표지와 기술 카드 삽화.
- 강호낭인·등반 지도·단일 이벤트 화면 자산.
- 전투 화면 반실사 자산 적용.

### 검증 필요

- 대화에서 승인된 최신 시안의 저장소 원본 반입.
- 실제 카드 크기의 기술 묘리 가독성.
- 소개 시트에서 문파 설명과 추구하는 무학의 중복 여부.
- UI 합성 시 출처 문양·분류·무공서·기술 데이터 매핑.

## 10. 다음 작업

1. 문파 소개·도감 시트의 공통 와이어프레임 제작.
2. 무당파를 기준으로 문파 설명·추구하는 무학·핵심 무공서 배치 검증.
3. 세력·출처 문양과 무공 종류 아이콘을 개별 자산으로 분리.
4. 무공서와 기술 카드의 텍스트 없는 삽화 제작.
5. UI 합성에서 문구 크기와 축소 가독성 검증.
6. 승인 원본을 이미지 인덱스에 등록.
