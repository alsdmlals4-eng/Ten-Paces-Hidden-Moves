# 십보강호 이미지 자료 인덱스

이 문서는 프로젝트 이미지 자료의 **유일한 최신 인덱스**다. 게임·UI·프로그래밍 문서는 이미지를 복제하지 않고 이 인덱스 또는 아트 기획서의 해당 항목을 링크한다.

## 1. 상태 규칙

- **승인 기준**: 이후 작업자가 같은 품질과 방향으로 제작해야 하는 최신 기준.
- **레이아웃 참고**: 정보 위계나 배치만 참고하며 그림체·문구·수치는 복제하지 않음.
- **구식 참고**: 과거 방향 확인용. 신규 제작 기준으로 사용 금지.
- **원본 반입 필요**: 대화에서 승인됐으나 저장소 자산으로 아직 보존되지 않음.
- **교체 예정**: 최신 승인 방향과 차이가 있어 새 자산이 필요함.

## 3. 최신 승인 이미지 방향

### 3.1 무공 카드 앞면

상태: **승인 기준 / 원본 반입 필요**

구조:

- 좌상단: 문파 문양 이미지. 공용은 `무(無)` 상징 이미지.
- 최상단 중앙: 사거리 슬롯. 숫자는 후편집 UI 레이어.
- 상단 중앙: 기술 이름.
- 우상단: 검법·권법·심법 등 무공 종류의 **텍스트 없는 아이콘 이미지**.
- 중앙: 기술 삽화.
- 하단: 비용·효과 없이 무공의 묘리 한 소절.
- 묘리 글자는 크게, 최대 두 줄, 충분한 여백과 높은 대비.
- 10성 핵심무공은 우측 세로 `절초` 표찰, 보조무공은 `진의` 표찰.

최신 승인 예시의 핵심 인상:

- 반실사 무협 캐릭터와 수묵 배경의 결합.
- 한지색 카드, 먹색 선, 얇은 금장.
- 인물과 기술 동작은 선명하고 배경은 회화적.
- 분류와 문파를 문자보다 문양으로 빠르게 식별.

### 3.2 문파 종합 시트

상태: **승인 기준 / 원본 반입 필요**

구조:

```text
상단: 문파 문양·문파명·정체성 문장
좌측: 남자 제자 독립 패널 + 여자 제자 독립 패널
중앙: 무공서 세로 열
우측: 무공서별 3성 기술 / 7성 기술 / 10성 절초·진의 카드
```

- 남녀 캐릭터는 겹치지 않고 각각 독립된 세로 구역에 선다.
- 같은 문파 복식 체계를 공유하지만 자세·시선·실루엣이 달라야 한다.
- 한 캐릭터는 핵심무공, 다른 캐릭터는 보조무공 또는 다른 핵심무공의 성격을 보여준다.
- 별도 무기·복식 상세 패널은 만들지 않는다.
- 무공서는 카드와 같은 행에 대응되도록 배치한다.
- 3성·7성 카드는 일반 기술, 10성은 절초·진의로 특별하게 보인다.

### 3.3 전투 화면 그림체

상태: **방향 승인 / 상세 설계 필요**

- 해 질 무렵의 비무대처럼 따뜻한 환경광.
- 캐릭터는 화면 양끝에서 서로 마주보고, 전장 칸이 읽힌다.
- 상단은 대전·상태, 중앙은 비무 장면, 하단은 선택 카드와 행동 슬롯.
- 캐릭터 초상은 반실사 무협 일러스트로 얼굴·복식이 선명해야 한다.

## 4. 최신 아트 언어

- 공통 UI: 먹색·짙은 갈색·금색.
- 캐릭터: 선명한 반실사.
- 배경: 수묵·한지·낮은 채도.
- 문파색: 전체를 덮지 않고 복식 안감, 카드 테두리, 기운 궤적에 제한적으로 사용.
- 생성 이미지 속 한글은 기준 정보로 사용하지 않는다.
- 명칭·분류·수치는 편집 가능한 UI 텍스트와 승인 아이콘으로 합성한다.

## 5. 이미지 반입 절차

1. 사용자 승인 시안의 원본 파일을 저장소 자산 경로로 반입한다.
2. 파일명은 역할과 대상을 알 수 있게 작성한다.
3. 이 인덱스에 경로·승인 상태·적용 범위를 기록한다.
4. 아트 기획서 본문의 관련 섹션에 상대경로로 삽입한다.
5. 기존 이미지는 즉시 삭제하지 않고 `구식 참고`로 전환한 뒤 참조 여부를 검사한다.
6. 신규 기준으로 완전히 대체되고 고유 정보가 없을 때만 아카이브 또는 제거한다.

## 6. 현재 반입 대기

- 무공 카드 앞면 승인 예시 2종.
- 무당파 문파 종합 시트 최신 예시.
- 문파 문양 10종·무공 종류 아이콘 16종 콘셉트 시트.
- 문파 캐릭터 남녀 분리 패널 참고 원본.
- 전투 화면 반실사 무협 UI 참고 원본.

이 항목들은 대화에서 방향이 승인됐지만 현재 저장소에서 직접 열 수 있는 원본 파일로 보존되지 않았다. 원본 반입 전에는 본 문서의 구조·스타일 설명을 기준으로 사용한다.

## 7. 갱신 의무

이미지를 생성·수정·승인·폐기하는 모든 작업은 같은 작업에서 이 인덱스의 상태와 아트 기획서의 관련 방향을 갱신해야 한다.
