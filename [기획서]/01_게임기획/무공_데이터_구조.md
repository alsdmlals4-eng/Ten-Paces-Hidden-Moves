# 십보강호 무공 데이터 구조

## 0. 목적

문파·가문·유파·강호 전승 무공을 같은 규칙으로 구현하기 위한 데이터 책임과 필드를 정의한다. 주인공은 출신 배경을 가진 강호낭인이며 모든 무공을 자유롭게 습득·혼합할 수 있다.

현재 단계에서는 **구조와 의미만 확정**하고 피해·비용·사거리·수련 비용 수치는 후속 밸런스 작업에서 입력한다.

## 1. 데이터 계층

```text
무공서 MartialArtBook
 ├─ 출처 Origin
 ├─ 숙련 Mastery 1~10성
 ├─ 기술 Skill 3성·7성
 ├─ 절초/진의 Ultimate 10성
 └─ 표시 Presentation

회차 RunState
 ├─ 출신 배경 Background
 ├─ 습득 무공 LearnedBook[]
 ├─ 해금 기술 Skill[]
 ├─ 수련포인트 TrainingPoint
 └─ 일정 Schedule
```

- **무공서**: 출처, 분류, 설명, 무도, 성장표와 기술 참조를 소유한다.
- **출처**: 문파·가문·유파·산채·기인 등 무공의 역사와 사건 연결 정보를 소유한다.
- **기술**: 전투에서 선택하는 실제 행동 카드다.
- **숙련**: 해금과 강화 조건을 관리한다.
- **회차 상태**: 주인공이 현재 익힌 무공, 숙련도, 수련 자원과 일정을 관리한다.
- **표시**: 이름·묘리·아이콘·삽화·프레임 경로를 관리하며 전투 계산에 관여하지 않는다.

## 2. 무공서 데이터

| 필드 | 형식 | 의미 |
|---|---|---|
| `id` | string | 영문 소문자 고유 ID |
| `name_ko` | string | 편집 가능한 한국어 이름 |
| `name_hanja` | string/null | 필요한 경우 한자 표기 |
| `origin_id` | string | 문파·가문·유파·기인 등 출처 참조 |
| `access_tier` | enum | `OPEN`·`EVENT`·`RARE`·`FORBIDDEN` |
| `category` | enum | 검법·심법 등 공식 분류 |
| `attribute_tags` | enum[] | 태극·빙한·열양·음파·혈기 등 속성 |
| `role_tags` | enum[] | 공격·방어·이동·자원·상태·반격 등 역할 |
| `description` | string | 무공의 기원과 운용 방식 설명 |
| `martial_way` | string | 해당 무공이 추구하는 무도 문장 |
| `starting_eligible` | bool | 기본적으로 시작 후보가 될 수 있는지 |
| `background_requirements` | string[] | 특정 배경이 시작 후보를 여는 경우의 조건 |
| `acquisition_rules` | object[] | 사건·교류·성향·비무·거래 획득 조건 |
| `skill_ids` | string[] | 3성·7성·10성 카드 참조 |
| `mastery_table_id` | string | 1~10성 성장표 참조 |
| `presentation_id` | string | 무공서 표지·문양·프레임 표시 참조 |
| `status` | enum | 확정·설계 중·검증 필요·보류 |

### 2.1 출처와 사용 권한 분리

`origin_id`는 무공이 어디에서 생겨났는지를 나타내며 사용 자격을 제한하지 않는다.

예:

```json
{
  "id": "taegeuk_sword",
  "origin_id": "WUDANG",
  "access_tier": "EVENT",
  "starting_eligible": false
}
```

태극검은 무당 전승이지만, 주인공이 사건·교류·기연으로 익힌 뒤에는 문파 관계와 무관하게 사용할 수 있다.

출처 유형 예시:

- `ORTHODOX_SECT`: 정문 문파.
- `CULT_OR_HETERODOX`: 마교·이문 세력.
- `MILITARY_LINEAGE`: 군문·가문.
- `REGIONAL_SCHOOL`: 지역 유파.
- `MOUNTAIN_STRONGHOLD`: 산채·녹림 전승.
- `WANDERER_LINEAGE`: 강호 고수·낭인 계보.
- `LOST_SCHOOL`: 멸문·실전 유파.
- `ANONYMOUS_MANUAL`: 작자 불명 비급.

### 2.2 카드 문양

- 카드 좌상단 문양은 `origin_id`의 표시 데이터를 따른다.
- 작자 불명·개방 전승은 `무(無)` 문양을 사용한다.
- 문양은 출처 표시일 뿐 사용 제한이나 보너스를 자동 부여하지 않는다.

## 3. 기술 카드 데이터

| 필드 | 형식 | 의미 |
|---|---|---|
| `id` | string | 기술 고유 ID |
| `book_id` | string | 소속 무공서 ID |
| `unlock_star` | int | 3·7·10 중 하나 |
| `card_kind` | enum | `SKILL`·`ULTIMATE`·`ESSENCE` |
| `name_ko` | string | 기술 이름 |
| `principle_text` | string | 카드 하단 무공의 묘리 |
| `target_rule` | enum | 자신·적·칸·전체 등 대상 규칙 |
| `range_rule` | object | 최소·최대 사거리와 특수 거리 조건 |
| `costs` | object[] | 행동력·기력·내공·절초 기세 등의 비용 |
| `effects` | object[] | 순서대로 실행되는 효과 명령 |
| `keywords` | string[] | 합·막기·밀치기·반격 등 UI 해설 키워드 |
| `requirements` | object[] | 무기·거리·상태·직전 행동 등의 사용 조건 |
| `usage_limit` | object/null | 전투·라운드별 사용 제한 |
| `presentation_id` | string | 기술 삽화·분류 아이콘·표찰 참조 |

### 카드 종류

- `SKILL`: 3성 또는 7성 신규 기술.
- `ULTIMATE`: 핵심무공 10성 절초.
- `ESSENCE`: 보조무공 10성 진의.

해금한 기술은 기본적으로 모두 전투 사용 후보가 된다. 무작위 손패를 뽑지 않으며, UI에서 무공서·역할·즐겨찾기로 정리한다.

## 4. 숙련과 강화 데이터

| 성급 | 데이터 결과 |
|---:|---|
| 1성 | 무공서 기본 성질·패시브 |
| 2성 | 기초 수치 또는 조건 보정 |
| 3성 | 기술 1 해금 |
| 4성 | 공통 숙련 보정 |
| 5성 | 기술 1 강화 패치 적용 |
| 6성 | 자원·연계 보정 |
| 7성 | 기술 2 해금 |
| 8성 | 상위 숙련·연계 보정 |
| 9성 | 기술 2 강화 패치 적용 |
| 10성 | 절초 또는 진의 해금 |

강화는 새 카드를 만들지 않고 `skill_upgrade_patch`를 적용한다.

```json
{
  "skill_id": "example_skill",
  "at_star": 5,
  "operations": [
    {"op": "modify", "path": "costs[0].value", "value": null},
    {"op": "append", "path": "effects", "value": null}
  ]
}
```

핵심무공·보조무공의 10성 개수에 별도 강제 상한을 두지 않는다. 대성 수는 수련포인트, 일정, 돌파 조건과 회차 길이가 자연스럽게 제한한다.

## 5. 강호낭인과 시작 데이터

게임 시작 시 주인공의 신분은 항상 `WANDERING_HERO`다. 플레이어는 출신 배경을 고르고 초기 후보에서 무공서 4권을 선택하며 모두 3성으로 시작한다.

```text
강호낭인 생성
→ 출신 배경 선택
→ 배경과 개방 목록으로 초기 후보 생성
→ 무공서 4권 선택
→ 선택한 4권을 3성으로 초기화
→ 각 무공의 3성 기술 해금
→ 첫 전투부터 기술 4개 보유
```

### 5.1 시작 빌드 규칙

| 항목 | 확정 기준 |
|---|---:|
| 주인공 신분 | 강호낭인 고정 |
| 출신 선택 | 약한 배경 1개 |
| 선택 무공서 | 4권 |
| 시작 숙련 | 전부 3성 |
| 시작 기술 | 4개 |
| 이후 습득 | 출처·문파·계통과 무관하게 자유 |

데이터 예시:

```json
{
  "identity": "WANDERING_HERO",
  "background_id": "FALLEN_MARTIAL_FAMILY",
  "selected_book_ids": [
    "samjae_sword",
    "baji_fist",
    "tuna_inner_art",
    "light_body_step"
  ],
  "starting_mastery": 3
}
```

### 5.2 출신 배경 데이터

| 필드 | 형식 | 의미 |
|---|---|---|
| `id` | string | 배경 고유 ID |
| `name_ko` | string | 배경 이름 |
| `description` | string | 성장 환경과 강호 진입 계기 |
| `starting_book_pool_additions` | string[] | 시작 후보에 추가되는 무공서 |
| `starting_resources` | object | 수련포인트·금전·장비 등 |
| `event_tags` | string[] | 사건 선택지와 첫 반응에 사용 |
| `initial_tendencies` | object | 초기 성향 값 또는 태그 |

배경은 이후 무공 습득을 금지하거나 특정 무공 효과를 영구 강화하지 않는다.

## 6. 수련과 일정 데이터

회차 성장의 실제 제약은 `수련포인트`와 `일정`이다.

### 6.1 습득 무공 상태

```json
{
  "book_id": "taegeuk_sword",
  "mastery_star": 3,
  "unlocked_skill_ids": ["three_rings_pierce_moon"],
  "acquired_from": "event_wudang_rescue",
  "mastery_history": []
}
```

- 습득 가능한 무공서 수에 인위적인 기본 상한을 두지 않는다.
- 3성 도달 시 첫 기술, 7성 도달 시 둘째 기술, 10성 도달 시 절초·진의를 해금한다.
- 해금한 기술 수가 증가할수록 전투의 두 수 조합도 증가한다.

### 6.2 수련 행동

| 필드 | 형식 | 의미 |
|---|---|---|
| `book_id` | string | 수련할 무공서 |
| `from_star` | int | 현재 숙련 |
| `to_star` | int | 목표 숙련 |
| `training_point_cost` | int/null | 수련포인트 비용 |
| `schedule_cost` | int/null | 필요한 일정 |
| `requirements` | object[] | 구결·인물·장소·성향·사건 조건 |
| `result` | object | 성급 상승·기술 해금·강화 적용 |

수치는 후속 작업에서 입력한다. 비용 곡선은 고성급으로 갈수록 상승해 `집중 대성`과 `박학 수련`이 경쟁하도록 설계한다.

## 7. 효과 명령 구조

기술 효과는 문장 하나로 계산하지 않고 순서가 있는 명령 배열로 저장한다.

### 1차 효과 종류

- `DAMAGE`: 피해.
- `HEAL`: 회복.
- `GAIN_GUARD`: 방어도 획득.
- `MOVE_SELF`: 자신 이동.
- `MOVE_TARGET`: 밀치기·끌어당기기.
- `MODIFY_RESOURCE`: 행동력·기력·내공 증감.
- `APPLY_STATUS`: 상태 부여.
- `REMOVE_STATUS`: 상태·독·점혈 해제.
- `BLOCK`: 막기.
- `DODGE`: 회피.
- `COUNTER`: 반격 예약.
- `CLASH_MODIFIER`: 합 수치 수정.
- `SWAP_POSITION`: 위치 교환.

```json
{
  "type": "DAMAGE",
  "target": "ENEMY",
  "value": null,
  "condition": null,
  "timing": "ON_RESOLVE",
  "tags": []
}
```

수치와 세부 조건은 다음 단계에서 확정한다.

## 8. 표시·아트 데이터 분리

최종 카드는 합성 파이프라인으로 만든다.

### 이미지 자산

- 무공서 표지.
- 기술 삽화.
- 출처·무(無) 문양.
- 무공 종류 아이콘.
- 카드 프레임.
- 절초·진의 표찰.

### 편집 가능한 데이터·텍스트

- 무공서·기술 이름.
- 무공 설명과 무도.
- 묘리 문구.
- 3성·7성·10성.
- 사거리·비용·효과·키워드.

생성 이미지 속 문자와 숫자는 `[아트 방향 참고용]`이며 기준 데이터로 사용하지 않는다.

## 9. 검증 규칙

- 모든 무공서 ID와 기술 ID는 중복되지 않는다.
- 무공서에는 3성·7성·10성 카드가 정확히 하나씩 연결된다.
- 핵심무공 10성은 `ULTIMATE`, 보조무공 10성은 `ESSENCE`다.
- 5성·9성 강화는 신규 카드 ID를 만들지 않는다.
- `origin_id`는 사용 권한이나 자동 전투 보너스로 해석하지 않는다.
- 시작 빌드는 무공서 4권이며 모두 3성으로 초기화된다.
- 이후 습득에는 출처·문파·계통 제한을 적용하지 않는다.
- 핵심·보조 대성 개수에 하드캡을 적용하지 않는다.
- 표시 데이터가 전투 결과를 계산하지 않는다.

## 10. 현재 상태와 다음 작업

### 확정

- 강호낭인 고정과 출신 배경 선택 구조.
- 데이터 계층과 책임 분리.
- 시작 무공서 4권·3성·기술 4개 규칙.
- 모든 무공의 자유 습득과 혼합.
- 수련포인트·일정에 의한 자연 제약.
- 집중 대성과 박학 수련의 성장 경쟁.
- 3·5·7·9·10성 해금·강화 구조.
- 효과를 명령 배열로 저장하는 방향.
- 이미지와 텍스트·전투 데이터 분리.

### 다음 작업

1. 출신 배경 데이터와 초기 무공서 후보 확정.
2. 수련포인트·일정의 기본 단위와 비용 곡선 확정.
3. 무공서 획득 사건의 조건 데이터 정의.
4. 대상·사거리 규칙의 enum과 판정 순서 확정.
5. 피해·합·막기·회피·이동의 계산 순서 확정.
6. 3성 기술부터 비용·효과의 기준값 범위 설계.
