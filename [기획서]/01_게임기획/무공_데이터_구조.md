# 십보강호 무공 데이터 구조

## 0. 목적

문파 무공 28권과 공용 무공 21권을 같은 규칙으로 구현하기 위한 데이터 책임과 필드를 정의한다. 현재 단계에서는 **구조와 의미만 확정**하며 피해·비용·사거리·강화 수치는 후속 밸런스 작업에서 입력한다.

## 1. 데이터 계층

무공은 네 계층으로 분리한다.

```text
무공서 MartialArtBook
 ├─ 숙련 Mastery 1~10성
 ├─ 기술 Skill 3성·7성
 ├─ 절초/진의 Ultimate 10성
 └─ 표시 Presentation
```

- **무공서**: 소속, 분류, 설명, 무도, 성장표와 기술 참조를 소유한다.
- **기술**: 전투에서 선택하는 실제 행동 카드다.
- **숙련**: 해금과 강화 조건을 관리한다.
- **표시**: 이름·묘리·아이콘·삽화·프레임 경로를 관리하며 전투 계산에 관여하지 않는다.

## 2. 무공서 데이터

| 필드 | 형식 | 의미 |
|---|---|---|
| `id` | string | 영문 소문자 고유 ID. 저장 데이터와 참조에 사용 |
| `name_ko` | string | 편집 가능한 한국어 이름 |
| `name_hanja` | string/null | 필요한 경우 한자 표기 |
| `affiliation` | enum | 문파 ID 또는 `COMMON` |
| `category` | enum | 검법·심법 등 공식 16종 |
| `role` | enum[] | 공격·방어·이동·자원·상태 등 검색용 역할 태그 |
| `description` | string | 무공의 기원과 운용 방식 설명 |
| `martial_way` | string | 해당 무공이 추구하는 무도 문장 |
| `starting_eligible` | bool | 게임 시작 4개 선택 후보 여부 |
| `skill_ids` | string[] | 3성·7성·10성 카드 참조 |
| `mastery_table_id` | string | 1~10성 성장표 참조 |
| `presentation_id` | string | 무공서 표지·문양·프레임 표시 참조 |
| `status` | enum | 확정·설계 중·검증 필요·보류 |

### 소속 규칙

- 문파 무공은 해당 문파를 선택한 플레이어만 시작 후보로 고를 수 있다.
- 공용 무공은 모든 문파의 시작 후보가 될 수 있다.
- 카드 좌상단 문양은 소속 데이터에서 결정한다. 공용은 `무(無)` 문양을 사용한다.

## 3. 기술 카드 데이터

| 필드 | 형식 | 의미 |
|---|---|---|
| `id` | string | 기술 고유 ID |
| `book_id` | string | 소속 무공서 ID |
| `unlock_star` | int | 3·7·10 중 하나 |
| `card_kind` | enum | `SKILL`·`ULTIMATE`·`ESSENCE` |
| `name_ko` | string | 기술 이름 |
| `principle_text` | string | 카드 하단 무공의 묘리 |
| `target_rule` | enum | 자신·적·칸·전체 등 대상 규칙 |
| `range_rule` | object | 최소·최대 사거리와 특수 거리 조건 |
| `costs` | object[] | 행동력·기력·내공·절초 기세 등의 비용 |
| `effects` | object[] | 순서대로 실행되는 효과 명령 |
| `keywords` | string[] | 합·막기·밀치기·반격 등 UI 해설 키워드 |
| `requirements` | object[] | 무기·거리·상태·직전 행동 등의 사용 조건 |
| `usage_limit` | object/null | 전투·라운드별 사용 제한 |
| `presentation_id` | string | 기술 삽화·분류 아이콘·표찰 참조 |

### 카드 종류

- `SKILL`: 3성 또는 7성 신규 기술.
- `ULTIMATE`: 핵심무공 10성 절초.
- `ESSENCE`: 보조무공 10성 진의.

## 4. 숙련과 강화 데이터

| 성급 | 데이터 결과 |
|---:|---|
| 1성 | 무공서 기본 성질·패시브 |
| 2성 | 기초 수치 또는 조건 보정 |
| 3성 | 기술 1 해금 |
| 4성 | 공통 숙련 보정 |
| 5성 | 기술 1 강화 패치 적용 |
| 6성 | 자원·연계 보정 |
| 7성 | 기술 2 해금 |
| 8성 | 상위 숙련 보정 |
| 9성 | 기술 2 강화 패치 적용 |
| 10성 | 절초 또는 진의 해금 |

강화는 새 카드를 만들지 않고 `skill_upgrade_patch`를 적용한다.

```json
{
  "skill_id": "example_skill",
  "at_star": 5,
  "operations": [
    {"op": "modify", "path": "costs[0].value", "value": null},
    {"op": "append", "path": "effects", "value": null}
  ]
}
```

현재 `value`는 미확정이며, 구조만 고정한다.

## 5. 시작 무공 선택 데이터

게임 시작 시 플레이어는 **무공 4개를 선택하며 모두 3성으로 시작**한다.

```text
문파 선택
→ 해당 문파 무공 + 공용 무공에서 4개 선택
→ 선택한 4개를 3성으로 초기화
→ 각 무공의 3성 기술을 해금
→ 첫 전투부터 기술 4개 보유
```

### 시작 빌드 규칙

| 항목 | 확정 기준 |
|---|---:|
| 선택 무공 수 | 4개 |
| 시작 숙련 | 전부 3성 |
| 시작 기술 수 | 4개 |
| 선택 후보 | 본인 문파 무공 + 공용 무공 |
| 타 문파 전용 무공 | 시작 선택 불가 |

데이터 예시:

```json
{
  "faction_id": "WUDANG",
  "selected_book_ids": ["taegeuk_sword", "taegeuk_fist", "taegeuk_divine_art", "samjae_sword"],
  "starting_mastery": 3
}
```

## 6. 효과 명령 구조

기술 효과는 문장 하나로 계산하지 않고 순서가 있는 명령 배열로 저장한다.

### 1차 효과 종류

- `DAMAGE`: 피해
- `HEAL`: 회복
- `GAIN_GUARD`: 방어도 획득
- `MOVE_SELF`: 자신 이동
- `MOVE_TARGET`: 밀치기·끌어당기기
- `MODIFY_RESOURCE`: 행동력·기력·내공 증감
- `APPLY_STATUS`: 상태 부여
- `REMOVE_STATUS`: 상태·독·점혈 해제
- `BLOCK`: 막기
- `DODGE`: 회피
- `COUNTER`: 반격 예약
- `CLASH_MODIFIER`: 합 수치 수정
- `SWAP_POSITION`: 위치 교환

각 명령은 다음 공통 필드를 가진다.

```json
{
  "type": "DAMAGE",
  "target": "ENEMY",
  "value": null,
  "condition": null,
  "timing": "ON_RESOLVE",
  "tags": []
}
```

수치와 세부 조건은 다음 단계에서 확정한다.

## 7. 표시·아트 데이터 분리

최종 카드는 합성 파이프라인으로 만든다.

### 이미지 자산

- 무공서 표지
- 기술 삽화
- 문파·공용 문양
- 무공 종류 아이콘
- 카드 프레임
- 절초·진의 표찰

### 편집 가능한 데이터·텍스트

- 무공서·기술 이름
- 무공 설명과 무도
- 묘리 문구
- 3성·7성·10성
- 사거리·비용·효과·키워드

생성 이미지 속 문자와 숫자는 기준 데이터로 사용하지 않는다.

## 8. 검증 규칙

- 모든 무공서 ID와 기술 ID는 중복되지 않는다.
- 무공서에는 3성·7성·10성 카드가 정확히 하나씩 연결된다.
- 핵심무공 10성은 `ULTIMATE`, 보조무공 10성은 `ESSENCE`다.
- 5성·9성 강화는 신규 카드 ID를 만들지 않는다.
- 문파 무공은 올바른 문파 소속을 가져야 한다.
- 공용 무공은 궁술·암기술·독공·마공·방패술·의술을 포함하지 않는다.
- 시작 빌드는 정확히 4개이며 모두 3성으로 초기화된다.
- 표시 데이터가 전투 결과를 계산하지 않는다.

## 9. 현재 상태와 다음 작업

### 확정

- 데이터 계층과 책임 분리.
- 시작 무공 4개·3성·기술 4개 규칙.
- 3·5·7·9·10성 해금·강화 구조.
- 효과를 명령 배열로 저장하는 방향.
- 이미지와 텍스트·전투 데이터 분리.

### 다음 작업

1. 자원 종류와 기본 최대치 확정.
2. 대상·사거리 규칙의 enum과 판정 순서 확정.
3. 피해·합·막기·회피·이동의 계산 순서 확정.
4. 3성 기술부터 비용·효과의 기준값 범위 설계.
5. 5성·9성 강화 패치 규칙 확정.
6. JSON 스키마와 Godot Resource 구조 구현.
