## 3. 기존 승인 구조

다음은 이번 Plan에서 바꾸지 않는다.

```text
회차 성장 조건: 핵심무공 10성 → 절초 해금
+
전투 발동 조건: 전투 중 절초 기세 충전 → 절초 사용 가능
```

- 절초 해금은 `TrainingState` 계열의 회차 성장 상태다.
- 절초 기세는 `CombatState` 계열의 전투 한정 상태다.
- UI·VFX·오디오는 해금·누적·소모를 계산하지 않는다.
- 플레이어와 상대 모두 상단 HUD에서 같은 상태 체계를 사용한다.
- 절초 보유 여부와 관계없이 5전 예선 결승을 완주할 수 있다.

## 4. 대안 비교

### 안 A — 5칸·사건 전용 충전

- 합·회피·막기·피격 같은 사건에서만 1칸을 얻는다.
- 장점: 행동 결과와 기세의 연결이 강하다.
- 위험: 거리 재기와 자원 회복 중심 전투에서는 지나치게 늦거나 끝내 충전되지 않을 수 있다.
- 판정: 핵심 하이라이트의 체험 편차가 커서 비추천.

### 안 B — 6칸·교전 경과 + 결과 보너스

- 2수 묶음 하나가 끝날 때 기본 1칸을 얻고, 의미 있는 결과가 있었으면 추가 1칸을 얻는다.
- 장점: 교착전에서도 진행되며, 적극적인 공격·방어·합 판단은 더 빠른 발동으로 보상된다.
- 위험: 결과 보너스가 너무 자주 발생하면 절초가 빠르게 준비될 수 있다.
- 대응: 한 전투원은 한 묶음에서 결과 보너스를 최대 1칸만 얻는다.
- 판정: **권장안**.

### 안 C — 10칸·타이밍 단위 정밀 충전

- 10타이밍 구조와 맞춰 매 타이밍 점수를 계산한다.
- 장점: 전투 구조와 숫자가 직관적으로 대응한다.
- 위험: 작은 증감이 너무 자주 표시돼 2수·거리·자원보다 HUD 주목도가 높아지고, 로그 노이즈가 커진다.
- 판정: T0·T1 범위에는 과도해 비추천.

## 5. 권장 규칙

### 5.1 명칭과 내부 구분

| 항목 | 권장 값 |
|---|---|
| 플레이어 표시명 | `절초 기세` |
| 짧은 HUD 표기 | `기세` |
| 개념 키 | `ultimate_momentum` |
| 최대치 | 6 |
| 발동 요구치 | 6 |
| 전투 시작값 | 0 |
| 기본 사용 횟수 | 전투당 1회 |

- 개방의 문파 기믹 등 프로젝트 내부에 이미 존재하는 일반 `기세`와 충돌하지 않도록 데이터·이벤트·로그 키에는 반드시 `ultimate_` 접두 책임을 둔다.
- 문파 전용 기세는 `faction_momentum` 또는 해당 기믹 전용 ID로 분리하고 절초 기세를 수정하지 않는다.
- 플레이어용 HUD에서는 절초 아이콘과 함께 표시하므로 짧게 `기세`라고 쓸 수 있지만, 상세·로그·접근성 문구에서는 `절초 기세`라고 쓴다.

### 5.2 획득 규칙

한 2수 묶음이 완전히 해상될 때 각 생존 전투원에게 다음 순서로 적용한다.

1. **교전 경과 +1**
   - 해당 2수 묶음이 정상 종료되면 양측이 각각 1을 얻는다.
   - 공격이 없거나 서로 거리를 재도 얻는다.
   - 전투가 끝난 묶음에서는 이후 사용 기회가 없으므로 패배한 전투원에게 표시 이벤트를 만들지 않는다.

2. **결과 보너스 +1**
   - 해당 전투원이 두 타이밍 중 하나 이상에서 아래 결과를 만들었으면 추가 1을 얻는다.
   - 한 묶음에 여러 결과가 있어도 추가 획득은 최대 1이다.

결과 보너스 후보:

| 우선순위 | 이유 키 | 플레이어 문구 | 조건 |
|---:|---|---|---|
| 1 | `clash_win` | `합 승리 +1` | 합에서 최종 수치 우위 |
| 2 | `evade_success` | `회피 성공 +1` | 유효 공격을 회피해 피해를 막거나 경감 |
| 3 | `guard_success` | `막기 성공 +1` | 막기로 유효 피해를 막거나 경감 |
| 4 | `attack_hit` | `공격 적중 +1` | 일반 공격 또는 합 결과로 상대 체력 피해 발생 |
| 5 | `clash_participation` | `합 대치 +1` | 합이 발생했으나 승리 이유가 아닌 경우 |
| 6 | `hit_survived` | `피격 버팀 +1` | 체력 피해를 받고 전투를 계속할 수 있음 |

- 우선순위는 **표시할 대표 이유**를 정할 뿐 추가 수치를 늘리지 않는다.
- 로그에는 같은 묶음에서 발생한 모든 후보 이유를 보조 정보로 남길 수 있다.
- `invalid_action`만 발생했거나 실제 공격·방어 결과가 없는 준비 행동은 결과 보너스를 만들지 않는다.
- 묶음당 총 획득 상한은 `교전 경과 1 + 결과 보너스 1 = 2`다.
- 이미 6이면 추가 획득 이벤트와 반복 토스트를 만들지 않는다.

### 5.3 예상 충전 속도

- 결과 보너스가 전혀 없는 경우 첫 라운드의 5개 묶음 종료 시 5/6, 다음 라운드 첫 묶음 종료 시 6/6이다.
- 매 묶음 결과 보너스를 얻는 적극적 전투는 첫 라운드 세 번째 묶음 종료 시 6/6이 될 수 있다.
- 목표 체험은 `적극적 판단 시 첫 라운드 후반`, `교착전이어도 둘째 라운드 초반`에 첫 사용 기회가 생기는 것이다.
- 이 속도는 구현 필수 규칙이 아니라 첫 플레이테스트의 조정 기준이다. 너무 빠르거나 느리면 최대치보다 먼저 결과 보너스 빈도와 조건을 조정한다.

### 5.4 선택·예약·소모

- 절초 카드는 다음 조건을 모두 만족할 때만 1수·2수 슬롯에 배치할 수 있다.
  - 핵심무공 10성으로 절초 해금.
  - 절초 기세 6/6.
  - 이번 전투에서 절초를 아직 사용하지 않음.
  - 해당 절초의 일반 비용·사거리·상태 조건을 현재 미리보기에서 만족.
- 절초를 슬롯에 놓으면 `발동 예약` 상태가 되지만 기세는 아직 소모하지 않는다.
- 잠금 전에는 슬롯에서 제거하거나 1수·2수 사이로 이동할 수 있다.
- 같은 절초를 한 묶음의 두 슬롯에 중복 배치할 수 없다.
- 양측 행동이 잠긴 뒤에는 다른 행동과 동일하게 변경할 수 없다.
- 실제 타이밍에서 행동 가능 여부·비용·사거리 검증을 통과해 절초가 **발동을 시작하는 순간** 6을 소모한다.
- 절초가 발동한 뒤 빗나가거나 막히거나 합에서 패해도 기세는 소모된다.
- 1수 결과로 쓰러짐, 타이밍 취소, 자원·사거리·상태 조건 변화로 발동 전에 무효가 되면 기세는 소모하지 않는다.
- 무효 뒤 전투가 계속되면 상태는 `발동 가능`으로 돌아간다. 행동 슬롯을 잃은 것이 실패 비용이다.
- 데모 기준 기본 사용 횟수는 전투당 1회다. 사용 뒤 상태는 `사용 완료`이며 같은 전투에서 다시 충전하지 않는다.
- 향후 무공·진의가 사용 횟수를 늘리는 확장은 별도 승인 전에는 만들지 않는다.
- 전투 종료 시 절초 기세·예약·사용 완료 상태는 초기화한다. 회차 사이에는 이월하지 않는다.
- 중간 전투 저장을 지원하게 되면 전투 스냅샷에만 저장하고 `TournamentState`의 영구 성장 자원으로 저장하지 않는다.

## 6. 상태 모델

```text
[미해금]
  핵심무공 10성 달성
      ↓
[축적 중 0~5]
  기세 6 도달
      ↓
[발동 가능 6/6]
  절초를 1수·2수 슬롯에 배치
      ↓
[발동 예약]
  ├─ 잠금 전 제거·이동 → [발동 가능]
  ├─ 발동 전 무효·취소 → [발동 가능]
  └─ 정상 발동 시작 → 기세 6 소모
                          ↓
                    [사용 완료]
  전투 종료
      ↓
[축적 중 0/6]
```

| 상태 | 카드 | 게이지 | 입력 |
|---|---|---|---|
| 미해금 | 잠금 | 빈 윤곽 + 자물쇠 | 상세에서 해금 조건 확인 |
| 축적 중 | 잠금 | 0~5칸 | 획득 이유·남은 칸 확인 |
| 발동 가능 | 선택 가능 | 6칸 + `절초 가능` | 1수·2수에 배치 가능 |
| 발동 예약 | 해당 슬롯 번호 표시 | 6칸 + 예약 표식 | 잠금 전 이동·제거 가능 |
| 사용 완료 | 사용 불가 | 빈 칸 + 사용 완료 표식 | 이번 전투 재사용 불가 |

## 7. 상단 HUD 권장 배치

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ [플레이어 초상·HP·행동력·기력·내공] [절초 ◆◆◆◇◇◇ 3/6]                  │
│                      [라운드 1] [타이밍 5-6 / 10]                          │
│                  [절초 ◇◇◇◇◇◇ 잠금] [상대 HP·자원·초상]                 │
└─────────────────────────────────────────────────────────────────────────────┘
│                              10칸 전장                                      │
│                                                                             │
│                    1수 슬롯      2수 슬롯      잠금                         │
│                    무공 카드·상세·결과 로그                                 │
```

실제 화면에서는 양측 정보를 좌우 대칭으로 같은 상단 띠에 놓는다.

- 절초 게이지는 체력·기력보다 작고, 타이밍 진행도와 1수·2수 선택을 가리지 않는다.
- 6개의 칸은 채움/빈 윤곽의 형태 차이로 구분한다.
- 숫자 `3/6`을 함께 표시해 색상만으로 상태를 전달하지 않는다.
- 발동 가능 시 `절초 가능`, 예약 시 `1수 예약` 또는 `2수 예약`, 사용 뒤 `사용 완료` 문구를 제공한다.
- 플레이어 미해금 문구: `절초 미해금`.
- 상대 미해금 문구: `절초 징후 없음`.
- 상대도 정확한 0~6칸과 발동 가능·예약·사용 완료 상태를 공개한다. 기세를 만드는 사건이 모두 공개 정보이고, 상대 절초 위험을 읽는 것이 2수 예측의 핵심 판단이기 때문이다.

## 8. 증가 이유·툴팁·로그

- 획득 시 게이지 근처에 대표 이유와 증가량을 짧게 표시한다.
  - `교전 경과 +1`
  - `합 승리 +1`
  - `회피 성공 +1`
- 같은 묶음에서 기본 획득과 결과 보너스가 함께 생기면 `교전 경과 +1 / 합 승리 +1`처럼 두 줄 또는 순차 표기한다.
- HUD 포커스·호버 상세:
  - `절초 기세 4/6`
  - `발동까지 2`
  - `이번 묶음: 교전 경과 +1, 막기 성공 +1`
  - `전투당 사용 가능: 1회`
- 전투 로그는 모든 기세 변화, 대표 이유, 보조 이유, 상한 적용, 예약·해제·발동·소모를 재열람할 수 있게 한다.
- 6/6이 된 순간에는 반복 점멸 대신 한 번의 `절초 가능` 상태 전환만 발생한다.
- 정확한 토스트 지속 시간과 음향 길이는 실제 플레이테스트에서 정하며 규칙 값으로 고정하지 않는다.

## 9. 접근성·폴백

- 색상 독립: 채운 칸/빈 칸 형태, 자물쇠, 준비, 예약, 사용 완료 아이콘과 문구를 함께 사용한다.
- 모션 감소: 게이지 파동·확대 대신 테두리 전환과 정적 `절초 가능` 라벨을 사용한다.
- 점멸 감소: 강한 플래시 대신 명도 변화와 아이콘 교체를 사용한다.
- 음향 없음: 숫자·형태·로그로 같은 상태를 전달한다.
- 키보드·패드 포커스: 상단 게이지에 포커스하면 `절초 기세 4/6, 발동까지 2`를 읽는다.
- 화면 읽기: 변화 이벤트는 `플레이어 절초 기세, 합 승리로 1 증가, 현재 6/6, 절초 사용 가능`처럼 actor·reason·delta·current를 포함한다.
- 자산 누락: 공용 절초 아이콘, 6칸 기본 게이지, 텍스트 로그로 결정론적으로 폴백한다.

## 10. 구조화 이벤트 계약

실제 Godot 클래스·필드명은 저장소 감사 뒤 확정한다. 아래 이름은 책임과 최소 페이로드를 고정하는 개념 계약이다.

| 이벤트 | 최소 데이터 | 발생 시점 |
|---|---|---|
| `ultimate_momentum_changed` | actor, before, delta, after, max, primary_reason, reason_keys, pair_index, timing_indices, cap_applied | 묶음 종료 후 획득 확정 |
| `ultimate_ready` | actor, current, max, source_pair | 6에 처음 도달 |
| `ultimate_reserved` | actor, action_id, slot_index | 절초를 1수·2수 슬롯에 배치 |
| `ultimate_reservation_cleared` | actor, action_id, reason | 잠금 전 제거 또는 발동 전 무효 |
| `ultimate_activated` | actor, action_id, slot_index, target | 행동 검증 통과 후 발동 시작 |
| `ultimate_spent` | actor, before, spent, after, uses_remaining | 정상 발동 시작과 동시에 소모 |
| `ultimate_locked` | actor, reason_key | 미해금·기세 부족·사용 완료·일반 조건 불충족 |

소유 경계:

| 계층 | 책임 |
|---|---|
| 전투 도메인 | 현재 기세, 획득 상한, 준비, 사용 횟수, 예약 유효성, 발동·소모·초기화 |
| 선택·AI | 공개 상태에서 절초 선택 가능 여부와 행동 평가 |
| 구조화 이벤트 | 변화 원인과 확정 결과 전달 |
| UI·프레젠테이션 | 게이지·라벨·토스트·로그·VFX·오디오·접근성 표현 |
| 저장 | 필요 시 전투 스냅샷만 직렬화, 회차 영구 성장과 분리 |

UI·애니메이션 완료 콜백에서 기세를 더하거나 빼지 않는다.

## 11. AI 절초 사용 기준

- AI도 플레이어와 같은 해금·6/6·전투당 1회·일반 행동 조건을 사용한다.
- AI는 각 2수 묶음 시작 시 공개된 위치·자원·체력·상태·양측 기세만 사용한다.
- 플레이어의 비공개 1수·2수를 보고 절초를 선택하거나 취소할 수 없다.
- 6/6이 됐다고 즉시 강제 사용하지 않는다.
- 행동 평가 우선순위 예시:
  1. 확정 또는 높은 확률의 승리 마무리.
  2. 다음 상대 공격을 견디지 못할 위험을 뒤집는 방어·회복 절초.
  3. 문파 기믹·거리·자원과 맞는 큰 우위.
  4. 일반 초식보다 기대 가치가 충분히 높은 상황.
- 난이도는 평가 깊이·성향·실수 확률을 바꾸며 숨은 플레이어 행동 열람으로 조정하지 않는다.
- 상대 예약 상태는 양측 잠금 후 행동 공개 시 다른 행동과 함께 공개한다.

## 12. 필수 테스트

### 규칙

- [ ] 미해금 전투원은 기세를 획득하거나 절초를 선택하지 않는다.
- [ ] 한 묶음 정상 종료 시 생존 전투원이 각각 교전 경과 +1을 얻는다.
- [ ] 결과 후보가 여러 개여도 한 전투원 결과 보너스는 묶음당 +1이다.
- [ ] 묶음당 총 획득이 2를 넘지 않는다.
- [ ] 6을 넘는 획득은 6에서 멈추고 반복 준비 이벤트를 만들지 않는다.
- [ ] 결과 보너스가 없는 전투는 둘째 라운드 첫 묶음 뒤 6/6이 된다.
- [ ] 매 묶음 결과 보너스가 있는 전투는 첫 라운드 세 번째 묶음 뒤 6/6이 될 수 있다.
- [ ] 같은 시드·입력에서 기세와 이유 키가 결정론적으로 재현된다.

### 선택·소모

- [ ] 5/6에서는 절초 카드가 잠기고 `기세 1 부족` 이유를 표시한다.
- [ ] 6/6에서 한 슬롯에만 예약할 수 있다.
- [ ] 잠금 전 제거·이동은 기세를 소모하지 않는다.
- [ ] 발동 전 무효는 기세를 소모하지 않고 전투가 계속되면 준비 상태로 돌아간다.
- [ ] 정상 발동 시작 시 6을 소모한다.
- [ ] 발동 뒤 빗나감·막힘·합 패배에도 소모 상태가 유지된다.
- [ ] 기본 설정에서 한 전투에 두 번째 절초를 사용할 수 없다.
- [ ] 다음 전투 시작 시 0/6·미사용 상태다.

### UI·접근성

- [ ] 플레이어가 3초 안에 양측 현재 기세와 절초 가능 여부를 말할 수 있다.
- [ ] 기세 증가 이유를 HUD 또는 로그에서 설명할 수 있다.
- [ ] `기세`라는 짧은 표기가 문파 전용 기세와 혼동되지 않는다.
- [ ] 색상을 제거해도 미해금·축적·가능·예약·사용 완료를 구분한다.
- [ ] 모션·점멸·음향을 줄여도 같은 정보가 전달된다.
- [ ] 최소 해상도에서 양측 게이지가 체력·타이밍·1수·2수 UI를 밀어내지 않는다.
- [ ] 상대 게이지와 실제 AI 선택 가능 상태가 일치한다.

### 소유·회귀

- [ ] UI·VFX·오디오를 제거해도 기세 획득·발동·소모 결과가 같다.
- [ ] 이벤트 재생 건너뛰기에서 중복 획득·중복 소모가 발생하지 않는다.
- [ ] 저장·불러오기 지원 범위에서 기세·예약·사용 상태가 중복 적용되지 않는다.
- [ ] 절초 미보유 경로로 5전 예선 결승을 정상 완주한다.
- [ ] 절초 보유 경로에서도 사용을 강제하지 않고 정상 완주한다.

## 13. 플레이테스트 질문

1. 플레이어는 게이지가 왜 늘었는지 한 묶음 뒤 설명할 수 있는가?
2. 상대가 언제 절초를 쓸 수 있는지 다음 2수를 고르기 전에 예측하는가?
3. 적극적 교전이 교착전보다 빠르게 준비된다는 차이를 체감하는가?
4. 첫 사용 가능 시점이 너무 빠르거나 늦다고 느끼는가?
5. 전투당 1회 제한이 하이라이트를 보존하는가, 불필요하게 답답한가?
6. 발동 전 무효에서 기세가 유지되는 규칙을 공정하다고 이해하는가?
7. `절초 기세`와 개방 등 문파 전용 기세를 혼동하는가?
8. 상대의 정확한 게이지 공개가 예측 재미를 높이는가, 정보량을 과도하게 늘리는가?
9. 절초를 얻지 못한 회차도 5전 결승이 정상 빌드로 느껴지는가?

## 14. 승인 항목

이 Plan을 기준 문서에 확정 반영하려면 다음 묶음을 승인한다.

1. 표시명 `절초 기세`, 내부 책임 `ultimate_momentum`.
2. 6칸·6 요구치·전투 시작 0.
3. 묶음 종료 기본 +1, 결과 보너스 최대 +1, 묶음 총 상한 2.
4. 적극적 전투는 첫 라운드 후반, 교착전은 둘째 라운드 초반 준비 목표.
5. 발동 시작 시 소모, 발동 전 무효 시 미소모.
6. 데모 기준 전투당 1회, 전투 종료 초기화·회차 이월 없음.
7. 플레이어·상대 모두 정확한 0~6칸과 준비·예약·사용 상태 공개.
8. 상단 좌우 대칭 HUD, 이유 토스트·전체 로그·접근성 폴백.
9. UI가 아닌 전투 도메인이 획득·상한·준비·소모를 소유.
