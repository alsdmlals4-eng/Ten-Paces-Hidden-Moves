# T0 전투 POC — M1 10칸 전투판 마일스톤

## 목적

현재 승인된 전투 UX·규칙을 실제 Godot POC로 옮기기 전에, 모든 전투 표현의 기준이 되는 **10칸 전투판·카메라·캐릭터 배치 규격**을 먼저 확정한다.

이 마일스톤에서는 카드, 상단 HUD, 행동 슬롯, 로그 패널, 진행 버튼을 구현하지 않는다. 전투판과 캐릭터 위치 관계만 독립적으로 검증한다.

## 승인된 전투판 규칙

- 전장은 일렬로 연결된 정확히 10칸이다.
- 칸 번호는 왼쪽부터 `1`~`10`이다.
- 캐릭터 한 명은 항상 정확히 한 칸의 중앙에 선다.
- 캐릭터 발 중심과 칸 중심 앵커가 일치해야 한다.
- 캐릭터 원화가 인접 칸을 시각적으로 침범하지 않도록 최대 폭을 제한한다.
- 전투판은 배경보다 먼저 읽혀야 한다.
- 배경은 전투판·캐릭터 크기와 원근을 확정한 뒤 배치한다.
- 플레이어 위치는 청색 계열 외곽선과 문양, 상대 위치는 적색 계열 외곽선과 문양으로 구분한다.
- 색상만으로 상태를 구분하지 않고 외곽선·문양·명도 차이를 함께 사용한다.

## POC 기본 배치

- 플레이어 시작 위치: 3번 칸
- 상대 시작 위치: 8번 칸
- 시작 거리: 5
- 카메라: 전투판 10칸 전체와 양측 캐릭터 전신이 동시에 보이는 고정 와이드 카메라
- 캐릭터 높이: 칸 폭의 약 1.6~1.9배
- 캐릭터 발밑 기준점: 칸 중앙 하단이 아닌 **칸 평면의 중앙점**

초기 위치는 화면 판독과 이동 검증을 위한 POC 값이며, 실제 밸런스 값은 전투 규칙 문서에서 별도로 관리한다.

## 칸 상태 최소 세트

| 상태 | 시각 표현 |
|---|---|
| 기본 | 낡은 석판·금속 테두리 |
| 플레이어 점유 | 청색 외곽선 + 플레이어 문양 |
| 상대 점유 | 적색 외곽선 + 상대 문양 |
| 이동 가능 | 녹색/청록 점선 + 발자국 문양 |
| 선택 목적지 | 금색 외곽선 + 중앙 표식 |
| 공격 가능 | 붉은 사선 테두리 + 검격 문양 |
| 사용 불가 | 명도 감소 + 잠금 문양 |

M1에서는 `기본`, `플레이어 점유`, `상대 점유`, `선택 목적지`만 실제 구현한다. 나머지는 확장 가능한 상태로 구조만 예약한다.

## 아트 제작 순서

1. 10개 칸을 동일 크기와 동일 원근으로 먼저 제작한다.
2. 칸 번호와 중앙 앵커를 배치한다.
3. 플레이어·상대 캐릭터를 각각 한 칸 중앙에 배치한다.
4. 캐릭터 크기와 실루엣 판독성을 확정한다.
5. 칸 점유 외곽선과 선택 상태를 추가한다.
6. 마지막에 성문·산수·깃발·관중·먼지 배경을 배치한다.
7. 배경 대비를 낮춰 전투판과 캐릭터가 가장 먼저 읽히게 한다.

## Godot 구현 최소 계약

- `BoardState` 또는 동등한 전투 도메인이 위치를 `1..10` 정수로 소유한다.
- UI는 도메인 위치를 읽어 칸 앵커로 캐릭터를 배치한다.
- 캐릭터 노드가 직접 게임 규칙 위치를 소유하지 않는다.
- 칸 노드는 `tile_index`, `occupant`, `visual_state`를 표현할 수 있어야 한다.
- 캐릭터 이동은 칸 앵커 사이 Tween으로 표현하되, 완료 시 최종 위치는 도메인 상태와 재동기화한다.
- 현재 M1에서는 충돌·합·피해·카드 실행을 구현하지 않는다.

## 검증 체크리스트

- [ ] 화면에 칸이 정확히 10개 존재한다.
- [ ] 번호가 1부터 10까지 중복·누락 없이 표시된다.
- [ ] 플레이어가 3번 칸 중앙에 선다.
- [ ] 상대가 8번 칸 중앙에 선다.
- [ ] 캐릭터가 인접 칸의 중앙 영역을 침범하지 않는다.
- [ ] 1번과 10번 칸이 화면 밖으로 잘리지 않는다.
- [ ] 16:9 기준 화면에서 전투판 전체가 한 번에 보인다.
- [ ] 최소 해상도에서 칸 번호와 점유 상태를 구분할 수 있다.
- [ ] 색각 이상 환경에서도 점유 상태를 외곽선·문양으로 구분할 수 있다.
- [ ] 배경을 제거해도 전투판과 위치 판독이 완전하다.
- [ ] 배경을 추가해도 전투판이 가장 먼저 읽힌다.
