## STEP 0 승인 카드 UI 계약

- 좌측 상단: 소속 배지.
- 중앙 상단: 사거리.
- 우측 상단: `이동 / 공격 / 대응 / 회복 / 강화` 기능 배지.
- 기초 행동의 소속은 `[기초]`.
- 하단은 `행동 슬롯 / 기력 / 내력`만 표시.
- 행동력과 공통 `막기 경감` 항목 제거.
- 카드명·원화·배지·사거리·수치·효과는 데이터로 교체 가능.
- UI는 표현만 담당하고 전투 결과를 계산하지 않음.

세부 구현은 `plans/2026-07-21-card-ui-component-step0.md`가 책임진다.

## STEP 2 승인 자산 기준

- 플레이어 시작 위치: 3번 칸.
- 상대 시작 위치: 8번 칸.
- 시작 거리: 5.
- 양측 캐릭터 동일 스케일.
- 캐릭터 높이: 칸 폭의 약 `1.4~1.6배`.
- 발 앵커: 칸 평면의 가로·세로 중앙선 교차점.
- 몸체 최대 폭: 인접 칸 중앙 영역을 침범하지 않음.
- 카메라: 10칸 전체와 양측 전신을 동시에 표시하는 고정 와이드.
- 승인 기준 자산: `assets/reference/step_02_character_scale_and_tile_placement.svg`.

초기 위치와 크기는 POC 판독 기준이며 밸런스 값이 아니다.

## Godot 진입 방식

- 프로젝트 실행(F5) 시 `scenes/combat/combat_board_preview.tscn`으로 바로 진입한다.
- Windows에서는 `tools/run_combat_board.cmd`를 더블클릭해 같은 전투판을 바로 실행할 수 있다.
- `tools/verify_and_commit_step0.cmd`는 시각 실행이 아니라 통합 자동 검증·보고서·commit·push 용도다.

## Godot 구현 최소 계약

- 전투 도메인이 위치를 `1..10` 정수로 소유한다.
- UI는 도메인 위치를 읽어 칸 앵커로 캐릭터를 배치한다.
- 캐릭터 노드가 직접 규칙 위치를 소유하지 않는다.
- 카드 UI는 JSON 정의를 표현하며 피해·회복·상태를 계산하지 않는다.
