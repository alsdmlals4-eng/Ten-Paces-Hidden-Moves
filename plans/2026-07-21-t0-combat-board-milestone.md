# T0 전투 POC — M1 카드 UI·전장 표현 마일스톤

## 목적

승인된 전투 UX를 실제 Godot POC로 옮기기 위해 **카드 UI 컴포넌트·10칸 전투판·캐릭터 배치·전투 배경·상단 HUD**를 순서대로 구현한다.

- STEP 0 카드 템플릿 실제 UI 컴포넌트 분리: **저장소 구현 완료 / 로컬 런타임 재검증 필요**

- STEP 2 캐릭터 크기·칸 배치: **Godot 씬 구현 완료**
- STEP 3 전투 배경: **승인된 수묵 산성 방향으로 독립 컴포넌트 구현 완료**
- STEP 4 상단 HUD: **플레이어·상대 정보와 절초 기세·중앙 라운드 패널 구현 완료**

## STEP 0 승인 카드 UI 계약

- 좌측 상단: 소속 배지.
- 중앙 상단: 사거리.
- 우측 상단: `이동 / 공격 / 대응 / 회복 / 강화` 기능 배지.
- 기초 행동의 소속은 `[기초]`.
- 하단은 `행동 슬롯 / 기력 / 내력`만 표시.
- 행동력과 공통 `막기 경감` 항목 제거.
- 카드명·원화·배지·사거리·수치·효과는 데이터로 교체 가능.
- UI는 표현만 담당하고 전투 결과를 계산하지 않음.

세부 구현은 `plans/2026-07-21-card-ui-component-step0.md`가 책임진다.

## STEP 2 승인 배치 기준

- 플레이어 시작 위치: 3번 칸.
- 상대 시작 위치: 8번 칸.
- 시작 거리: 5.
- 양측 캐릭터 동일 스케일.
- 캐릭터 높이: 칸 폭의 약 `1.4~1.6배`, POC 구현값 `1.5배`.
- 발 앵커: 칸 평면의 가로·세로 중앙선 교차점.
- 몸체 최대 폭: 인접 칸 중앙 영역을 침범하지 않음.
- 카메라: 10칸 전체와 양측 전신을 동시에 표시하는 고정 와이드.
- 승인 기준 자산: `assets/reference/step_02_character_scale_and_tile_placement.svg`.

초기 위치와 크기는 POC 판독 기준이며 밸런스 값이 아니다.

## STEP 3 승인 배경 기준

- 산성·절벽·안개·소나무를 포함한 수묵 무협풍.
- 전투판과 캐릭터보다 낮은 대비.
- 배경은 독립 `BattleBackground` 컴포넌트로 교체 가능.
- 배경은 전투판·캐릭터·HUD보다 먼저 그린다.
- 10칸 가시성과 양측 실루엣 판독성을 우선한다.

## STEP 4 승인 상단 HUD 기준

화면 순서는 다음과 같이 고정한다.

```text
플레이어 상태 | 플레이어 절초 기세 | 라운드·행동 묶음 | 상대 절초 기세 | 상대 상태
```

- 상단 좌측: 플레이어 초상 자리·이름·이명·체력·기력·내력·상태.
- 상단 중앙 좌측: 플레이어 절초 기세 6칸.
- 상단 중앙: 라운드·행동 묶음 `1/3~3/3`·현재 선택 수·해상 순서.
- 상단 중앙 우측: 상대 절초 기세 6칸.
- 상단 우측: 상대 초상 자리·이름·이명·체력·기력·내력·공개 상태.
- 해상 순서: `대응 → 속공 → 이동 → 일반 공격`.
- 좌우 세로 패널과 하단 플레이어·상대 능력치 패널은 사용하지 않는다.
- HUD는 전투 상태를 표현하며 직접 계산하지 않는다.
- 미리보기 값은 `data/combat/combat_hud_preview.json`에서 주입한다.

## Godot 진입 방식

- 프로젝트 실행(F5) 시 `scenes/combat/combat_board_preview.tscn`으로 바로 진입한다.
- Windows에서는 `tools/run_combat_board.cmd`를 더블클릭해 같은 전투판을 바로 실행할 수 있다.
- `tools/verify_and_commit_step0.cmd`는 시각 실행이 아니라 STEP 0~4 통합 자동 검증·보고서·commit·push 용도다.

## Godot 구현 최소 계약

- 전투 도메인이 위치를 `1..10` 정수로 소유한다.
- UI는 도메인 위치를 읽어 칸 앵커로 캐릭터를 배치한다.
- 캐릭터 노드가 직접 규칙 위치를 소유하지 않는다.
- 카드 UI와 HUD는 데이터를 표현하며 피해·회복·상태·기세를 직접 계산하지 않는다.
