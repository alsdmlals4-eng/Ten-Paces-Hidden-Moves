# 십보강호 전투 PoC 실행 Plan

> 이 문서는 Codex 구현 작업의 기준 Plan이다. Goal 문서 대신 이 Plan과 테스트 체크리스트를 사용한다.

## 목적

10칸 전장과 비공개 동시 선택 전투가 실제 플레이에서 거리 심리전으로 작동하는지 검증한다.

## 완료 산출물

- Godot에서 실행 가능한 1대1 전투 PoC
- 권법·창법·궁술의 대표 기본 행동
- 6슬롯 비공개 배치와 동시 공개 UI
- 슬롯별 결과 해설
- [전투 PoC 테스트 체크리스트](../docs/08_TEST_CHECKLIST.md) 전 항목의 결과 기록

## 구현 범위

- 기준 문서: [전투 PoC 수치 명세](../docs/05_COMBAT_POC_SPEC.md)
- UI 기준: [전투 UI 명세](../docs/07_COMBAT_UI_SPEC.md)
- 시작 위치: A=4, B=7, 초기 거리=3
- 자원: 행동력 6, 기력 12, 내공 6
- 무기: 권법, 창법, 궁술
- AI: 공개 정보 기반 행동열 사전 잠금

## 제외 범위

- 문파 선택, 대성, 장비, 방패, 독, 의술, 마공
- 9전 로그라이트 회차
- 대사·인물·세계관 연출
- 멀티플레이

## 작업 순서

### 1. 전장 상태와 데이터 모델

- [ ] 10칸, A·B 위치, 거리 계산, 같은 칸, 상호 통과를 구현한다.
- [ ] 행동력·기력·내공·체력의 라운드 상태를 구현한다.
- [ ] 거리, 전장 밖 이동, 자원 부족을 단위 테스트로 검증한다.
- [ ] 체크리스트 A·B를 실행하고 결과를 기록한다.

### 2. 행동열과 동시 실행기

- [ ] 양측 6슬롯 비공개 행동열을 구현한다.
- [ ] 잠금 뒤 양측 행동열을 동시에 공개한다.
- [ ] 슬롯마다 이동 → 방어 → 공격·밀치기 → 후처리 순서로 실행한다.
- [ ] 같은 슬롯의 동시 피해·동시 밀치기를 구현한다.
- [ ] 체크리스트 C·E를 실행하고 결과를 기록한다.

### 3. 권법·창법·궁술

- [ ] 권법의 거리 0 기본 권격과 연환권을 구현한다.
- [ ] 창법의 거리 2 찌르기와 자동 회수를 구현한다.
- [ ] 궁술의 거리 3~5 사격과 장전을 구현한다.
- [ ] 각 행동의 슬롯·행동력·기력·내공·피해가 수치 명세와 일치하는지 검증한다.
- [ ] 체크리스트 D를 실행하고 결과를 기록한다.

### 4. 공통 행동과 UI

- [ ] 보법, 경공, 막기, 회피보, 밀치기, 대기를 구현한다.
- [ ] 자원 부족·사거리 밖·미장전·회수 중 행동을 UI에서 비활성화한다.
- [ ] 전투 UI에 전장 번호, 현재 거리, 양측 자원, 6슬롯, 결과 해설을 표시한다.
- [ ] 체크리스트 G를 수동 점검한다.

### 5. AI 공정성과 재현

- [ ] AI가 라운드 시작 공개 정보로 행동열을 먼저 생성하고 잠그게 한다.
- [ ] 행동열 잠금 뒤 AI 데이터가 변하지 않는지 검증한다.
- [ ] 재현 시드와 슬롯별 결과 로그를 기록한다.
- [ ] 체크리스트 F를 실행하고 결과를 기록한다.

## 승인 기준

- [ ] 체크리스트 A~G의 필수 항목이 모두 통과했다.
- [ ] 권법·창법·궁술로 최소 3회씩 전투해 각 무기의 거리 목표가 구분된다.
- [ ] AI 치팅 의심 없이 행동열의 이유를 결과 해설로 확인할 수 있다.
- [ ] 테스트 실패 항목은 다음 Plan으로 넘기지 않고 원인·재현 절차를 기록했다.
