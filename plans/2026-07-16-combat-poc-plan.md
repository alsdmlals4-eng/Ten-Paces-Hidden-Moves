# 12세력 대회 세로 슬라이스 구현 Plan

## 목적

Godot 4.7.1에서 12세력의 공용 전투 엔진, 2수 비공개 동시 선택, 합, 핵심무공·심법 10성, 성과·행운·수련·상대 성장·제약을 구현한다. 먼저 5전 예선 결승까지의 데모를 완성하고 같은 구조를 10전 결승까지 확장한다.

## Codex 작업 시작 규칙

Codex는 즉시 코드를 작성하지 않는다. 반드시 Plan Mode에서 다음을 먼저 수행한다.

1. 루트 `AGENTS.md`를 읽는다.
2. `docs/ACTIVE_CONTEXT.md`와 `docs/DOCUMENTATION_MAP.md`를 읽는다.
3. 문서 지도에 연결된 활성 기준 문서를 순서대로 읽는다.
4. `docs/[백업]/`과 `docs/[보류]/`는 구현 근거에서 제외한다.
5. 실제 저장소 트리, `project.godot`, GDScript, 데이터와 테스트 존재 여부를 확인한다.
6. 문서와 실제 파일의 충돌, 오래된 6슬롯·1~5성·9전·구 성과 보상 흔적을 목록화한다.
7. 데모 우선 구현 순서, 수정·신규 파일, 데이터 계약, 테스트, 위험과 중단 기준을 Plan Mode 결과로 제출한다.
8. 사용자 승인 전에는 코드·씬·Resource를 생성하거나 수정하지 않는다.

## 고정 제약

- 전장 10칸, A=4, B=6, 초기 거리 2.
- 라운드 행동 타이밍 10개, 2수 묶음 5회.
- 전체 대회 10전, 8전 8강·9전 준결승·10전 결승.
- 데모 1~2전 내부전, 3~4전 예선, 5전 예선 결승.
- AI는 각 묶음에서 플레이어 선택 전에 행동 두 개를 잠근다.
- 양측 두 행동을 동시에 공개하고 1수→2수 순으로 해상한다.
- 같은 타이밍 쌍방 공격은 합 판정 후 낮은 쪽이 차이 피해를 받는다.
- 공용 행동은 접근, 후퇴, 공격, 막기, 회피, 밀치기, 교체, 재장전, 명상이다.
- 핵심무공·심법·진의는 공용 수정자 계약을 사용한다.
- 심법 보유 최대 2개, 핵심무공 10성 최대 1개, 심법 10성 최대 1개.
- 수련 정산은 기본 5 + 등급 보너스(C1/B3/A5/S7) + 행운 0~3.
- 행운은 등급별 결과표에서 비공개로 추첨하고 `A++`처럼 표시한다.
- 상대는 A계열 60%, B계열 40%와 행운·수련 스타일을 사용한다.
- 상대 성장 결과는 회차 생성 시 고정하고 재도전에서 다시 굴리지 않는다.
- 개발용 상대 태그를 플레이어에게 직접 노출하지 않는다.
- 문파별 기믹 수행도 성과 평가는 `[보류]`이며 구현하지 않는다.

## 작업 0 — 저장소 검수와 승인 Plan

- [ ] 활성 문서의 확정 규칙을 요약한다.
- [ ] 실제 Godot 프로젝트·스크립트·데이터·테스트 상태를 확인한다.
- [ ] 폐기된 구조·수치·용어의 잔존 위치를 찾는다.
- [ ] Resource ID와 저장 포맷 규칙을 제안한다.
- [ ] 데모 5전 우선순위와 전체판 10전 확장 경계를 제안한다.
- [ ] 단계별 검증 명령을 작성한다.
- [ ] 사용자 승인 전 구현하지 않는다.

## 작업 1 — 공용 전투 엔진

- [ ] Godot 4.7.1 프로젝트와 headless 테스트 기반을 만든다.
- [ ] 10칸 위치·거리·중첩·교차·자리 교환을 구현한다.
- [ ] `ActionChoice`, `ActionPair`, `PairLockCoordinator`를 구현한다.
- [ ] 10타이밍과 5개 행동 묶음을 구현한다.
- [ ] AI 사전 잠금, 동시 공개, 1수→2수 해상을 구현한다.
- [ ] 행동력·기력·내공·체력과 라운드 종료 회복을 구현한다.
- [ ] 공용 행동과 이동 후 사거리 판정을 구현한다.
- [ ] 유효하지 않은 행동은 `invalid_action` 뒤 대기로 처리한다.

## 작업 2 — 합·무기·공용 수정자

- [ ] `ClashResolver`를 구현한다.
- [ ] 기본 합→무기→성급→심법·기믹→반격→최종 합→차이 피해 이벤트를 구현한다.
- [ ] 합 동률, 단독 공격, 공격 대 방어를 분리한다.
- [ ] 권법·검·도·곤·창·암기·활·방패 데이터를 구현한다.
- [ ] 비용·합·피해 경감·사거리·이동·상태·명상·장전 수정자를 구현한다.
- [ ] 독·마화·장전·방패·암기벨트·화살통을 공통 상태·장비로 구현한다.

## 작업 3 — 핵심무공·심법

- [ ] `MartialArtData`, `HeartMethodData`, `ModifierData`를 구현한다.
- [ ] 1~10성 비용과 부분 수련 포인트를 구현한다.
- [ ] 핵심무공 3·6·9성 기술과 10성 절초를 구현한다.
- [ ] 심법의 패시브·기믹·명상 강화와 10성 진의를 구현한다.
- [ ] 핵심무공 여러 개, 심법 최대 2개 보유를 구현한다.
- [ ] 계열별 10성 슬롯을 각각 1개로 제한한다.
- [ ] 각 핵심무공 종류에 문파 전용 1종과 공용 2종을 구현한다.
- [ ] 화산 연격과 다른 문파 기믹을 엔진 전용 분기 없이 공용 수정자로 연결한다.

## 작업 4 — 성과·행운·수련

- [ ] `PerformanceGradeData`, `LuckOutcomeTable`, `TrainingState`를 구현한다.
- [ ] 승리 기본 5와 등급 보너스 C1/B3/A5/S7을 구현한다.
- [ ] A가 평균 성과가 되도록 공용 평가 지표와 임계값을 데이터화한다.
- [ ] 행운 결과표를 구현한다.
  - C: 0/0/5/95%.
  - B: 5/5/5/85%.
  - A: 20/5/20/55%.
  - S: 60/10/15/15%.
- [ ] 행운 앞면 수를 `A`, `A+`, `A++`, `A+++`로 표시한다.
- [ ] 개별 다이스 면은 공개하지 않는다.
- [ ] 수련 포인트 분배, 성급 상승과 해금 이벤트를 구현한다.
- [ ] A4 약 50%, A3+S1 약 60%, A3+B1 약 30%의 데모 절초 목표를 시뮬레이션 테스트한다.

## 작업 5 — 상대 생성·성장·정보 표현

- [ ] `OpponentGrowthProfile`, `TrainingAllocator`, `OpponentPresentationData`를 구현한다.
- [ ] 상대 등급 A계열 60%, B계열 40% 추첨을 구현한다.
- [ ] 상대도 등급별 행운 결과표에서 `+`를 추첨한다.
- [ ] 등장 경기 전까지의 누적 수련 예산을 계산한다.
- [ ] 균형형, 핵심무공 올인형, 심법 연계형, 다무공형 투자 우선순위를 구현한다.
- [ ] 실제 단계 비용을 순차 지불하고 남은 부분 포인트를 저장한다.
- [ ] 내부 종합 강도 5단계를 계산한다.
- [ ] 이명·풍문·전적·정탐 문구 키를 내부 프로필과 연결한다.
- [ ] `[강적] [A++] [올인형]` 같은 개발용 태그가 UI에 노출되지 않게 한다.
- [ ] 같은 의미 키는 같은 어휘 계열을 사용한다.
- [ ] 상대 성장과 표현 키를 회차 생성 시 고정하고 재도전·저장에서 유지한다.

## 작업 6 — 제약

- [ ] `ConstraintData`, `ConstraintResolver`를 구현한다.
- [ ] 상대 정보 확인 뒤 제약 선택 화면으로 진입한다.
- [ ] 삼수양보·단전봉맥·기혈역행·일장지양 등을 구현한다.
- [ ] 제약 승리 시에만 추가 수련·보상을 정산한다.
- [ ] 문파 핵심무공 전체를 봉쇄하는 제약이 생성되지 않게 한다.

## 작업 7A — 데모 5전

- [ ] 시작 6세력 선택과 초기 3성 핵심무공·심법을 구현한다.
- [ ] 1~2전 내부전, 3~4전 예선, 5전 예선 결승을 구현한다.
- [ ] 3전부터 공용·외문 무공 또는 보조장비 보상 후보를 제공한다.
- [ ] 1~4전 승리 후 수련 화면을 거쳐 5전 전 절초 여부를 확정한다.
- [ ] 절초가 있거나 없어도 5전을 시작하고 완료할 수 있게 한다.
- [ ] 5전 종료 후 데모 종료 화면과 전체판 예고를 구현한다.
- [ ] 데모 종료 상태를 저장·불러오기에서 구분한다.

- [ ] 무공·장비·회복·정탐 보상 선택을 구현한다.
- [ ] 재도전 1회, 최초 무료·이후 명망 소모를 구현한다.
- [ ] 저장 포맷에 회차, 경기·단계, 데모 종료, 플레이어 성장, 상대 성장 프로필·표현 키, 장비, 명망, 재도전, 제약을 포함한다.
