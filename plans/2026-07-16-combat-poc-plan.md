# 12세력 대회 세로 슬라이스 구현 Plan

## Plan의 목적

Godot 4.7.1에서 12세력의 공용 전투 엔진, 2수 비공개 동시 선택, 합, 핵심무공·심법 10성, 성과·수련·제약을 구현한다. 먼저 5전째 지역 예선 결승까지의 데모를 완성하고, 같은 구조를 10전째 결승까지 확장한다.

## Codex 작업 시작 규칙

Codex는 이 문서를 읽은 즉시 코드를 작성하지 않는다. 반드시 Plan Mode에서 다음 사전 검수를 먼저 수행한다.

1. 저장소 루트 `AGENTS.md`와 `docs/DOCUMENTATION_MAP.md`를 읽는다.
2. 문서 지도에 연결된 활성 기준 문서를 순서대로 읽는다.
3. `docs/[백업]/`과 `docs/[보류]/`는 구현 근거에서 제외한다.
4. 실제 저장소 트리, `project.godot`, 기존 GDScript, 테스트, 데이터 파일을 검사한다.
5. 문서와 구현의 충돌, 누락 파일, 오래된 6슬롯·1~5성·9전 흔적을 목록화한다.
6. 데모 우선 구현 순서, 전체판 확장 순서, 수정 파일, 신규 파일, 테스트, 위험과 중단 기준을 Plan Mode 결과로 제출한다.
7. 사용자 승인 전에는 코드·씬·Resource를 생성하거나 수정하지 않는다.

## 고정 제약

- 전장은 10칸, 기본 배치는 A=4, B=6, 거리 2다.
- 한 라운드는 행동 타이밍 10개이며 2수 묶음을 5번 선택한다.
- 전체 대회는 10전이며 8전은 8강, 9전은 준결승, 10전은 결승이다.
- 데모는 5전째 지역 예선 결승까지다.
- AI는 매 묶음에서 플레이어 선택 전에 행동 두 개를 잠근다.
- 양측 행동 두 개를 동시에 공개하고 1수→2수 순으로 해상한다.
- 같은 타이밍 쌍방 공격은 합 판정 후 낮은 쪽이 합 수치 차이만큼 피해를 받는다.
- 공용 행동은 접근, 후퇴, 공격, 막기, 회피, 밀치기, 무기 교체, 재장전, 명상이다.
- 핵심무공·심법·진의는 공용 수정자만 사용한다.
- 심법 보유는 최대 2개다.
- 핵심무공 10성 최대 1개, 심법 10성 최대 1개다.
- 승리 기본 수련 포인트는 5다.
- 등급 총 보상은 C 6~7, B 8~10, A 13~14, S 16이다.
- 첫 4전을 모두 A 이상으로 승리하고 핵심무공에 집중하면 5전 전 절초가 개방되어야 한다.
- 문파별 기믹 수행도 성과 평가는 `[보류]`이며 구현하지 않는다.

## 작업 0 — 저장소 검수와 승인 Plan

- [ ] 활성 문서의 확정 규칙을 요약한다.
- [ ] 실제 Godot 프로젝트·스크립트·데이터·테스트 존재 여부를 확인한다.
- [ ] 6슬롯, 1~5성, 9전, 동시 개별 피해 등 폐기 규칙의 잔존 위치를 찾는다.
- [ ] 구현 파일 구조와 Resource ID 규칙을 제안한다.
- [ ] 데모 5전 우선순위와 전체판 10전 확장 경계를 제안한다.
- [ ] 단계별 검증 명령을 작성한다.
- [ ] Plan Mode 결과를 사용자에게 제출하고 승인을 기다린다.

## 작업 1 — Godot 프로젝트와 공용 전투 엔진

- [ ] Godot 4.7.1 프로젝트와 headless 테스트 실행 기반을 만든다.
- [ ] 10칸 위치·거리·중첩·교차·자리 교환을 구현한다.
- [ ] `ActionChoice`, `ActionPair`, `PairLockCoordinator`를 구현한다.
- [ ] 10타이밍과 5개 행동 묶음을 구현한다.
- [ ] AI 사전 잠금, 동시 공개, 1수→2수 해상을 구현한다.
- [ ] 행동력·기력·내공·체력과 라운드 종료 회복을 구현한다.
- [ ] 공용 행동과 이동 후 사거리 판정을 구현한다.
- [ ] 유효하지 않은 행동은 `invalid_action` 뒤 대기로 처리한다.

## 작업 2 — 합·무기·공용 수정자

- [ ] `ClashResolver`를 구현한다.
- [ ] 기본 합→무기→성급→심법·기믹→반격→최종 합→차이 피해 이벤트를 구현한다.
- [ ] 합 동률, 단독 공격, 공격 대 방어를 분리한다.
- [ ] 권법·검·도·곤·창·암기·활·방패 무기 데이터를 구현한다.
- [ ] 비용·합·피해 차이 경감·사거리·이동·상태·명상·장전 공용 수정자를 구현한다.
- [ ] 독·마화·장전·방패·암기벨트·화살통을 공통 상태·장비로 구현한다.

## 작업 3 — 핵심무공·심법 Resource

- [ ] `MartialArtData`, `HeartMethodData`, `ModifierData`를 구현한다.
- [ ] 1~10성 비용과 부분 수련 포인트를 구현한다.
- [ ] 핵심무공 3·6·9성 기술과 10성 절초를 구현한다.
- [ ] 심법의 패시브·기믹·명상 강화와 10성 진의를 구현한다.
- [ ] 핵심무공 여러 개 보유, 심법 최대 2개 보유를 구현한다.
- [ ] 계열별 10성 슬롯을 각각 1개로 제한한다.
- [ ] 각 핵심무공 종류에 문파 전용 1종과 공용 2종 데이터를 구현한다.
- [ ] 화산 심법 3성 연격과 10성 진의 강화를 공용 수정자로 구현한다.
- [ ] 다른 11세력도 엔진 전용 분기 없이 공용 수정자로 연결한다.

## 작업 4 — 성과·수련·제약

- [ ] `PerformanceGradeData`, `TrainingState`, `ConstraintData`를 구현한다.
- [ ] 승리 기본 포인트 5를 구현한다.
- [ ] C: 확정 +1, 30% +1을 구현한다.
- [ ] B: 확정 +3, 50% +2를 구현한다.
- [ ] A: 확정 +8, 50% +1을 구현한다.
- [ ] S: 확정 +9, 확률 보너스 +2 확정을 구현한다.
- [ ] 수련 포인트 분배와 성급 상승, 해금 이벤트를 구현한다.
- [ ] 삼수양보·단전봉맥·기혈역행·일장지양 등 제약을 구현한다.
- [ ] 제약 승리 시에만 추가 수련·보상 효과를 정산한다.
- [ ] 문파별 기믹 수행도 평가가 코드에 포함되지 않았는지 확인한다.

## 작업 5A — 데모 5전

- [ ] 시작 6세력 선택과 초기 3성 핵심무공·심법을 구현한다.
- [ ] 1~2전 내부전, 3전 추천패 쟁탈전, 4전 지역 예선, 5전 지역 예선 결승을 구현한다.
- [ ] 1~4전 승리 후 수련 화면을 거쳐 5전 시작 전 절초 해금 여부를 계산한다.
- [ ] A등급 4회가 최소 52포인트를 지급해 절초를 확정 개방하는지 검증한다.
- [ ] B등급 4회가 최대 40포인트로 절초를 개방하지 못하는지 검증한다.
- [ ] 5전 전투 준비 화면에서 신규 절초를 확인하고 선택할 수 있게 한다.
- [ ] 5전 종료 후 데모 종료 화면과 전체판 예고를 구현한다.
- [ ] 데모 종료 상태를 저장·불러오기에서 구분한다.

## 작업 5B — 전체판 10전과 저장

- [ ] 6~7전 본선, 8전 8강, 9전 준결승, 10전 결승을 구현한다.
- [ ] 12세력 상대 풀과 반복 방지 규칙을 구현한다.
- [ ] 공용·외문 무공, 장비, 회복, 정탐 보상 선택을 구현한다.
- [ ] 재도전 1회, 최초 무료·이후 명망 소모를 구현한다.
- [ ] 저장 포맷 버전, 회차, 경기·단계, 데모 종료 여부, 무공·심법·성급·부분 포인트, 10성 슬롯, 장비, 명망, 재도전, 제약을 저장·복원한다.

## 작업 6 — 화면과 카드 UI

- [ ] `MainMenu`, `FactionSelect`, `TournamentScreen`, `ConstraintSelect`, `BattleScreen`, `TrainingScreen`, `ResultScreen`을 구현한다.
- [ ] 전장 1~10칸과 행동 타이밍 1~10, 현재 2수 묶음을 표시한다.
- [ ] 데모 1~5전과 전체판 1~10전 진행도를 구분해 표시한다.
- [ ] 행동 두 개 선택·잠금·동시 공개 UI를 구현한다.
- [ ] 카드에 문파/공용, 종류, 사거리, 타이밍·행동력·기력·내공 비용을 표시한다.
- [ ] 합 예상, 합 결과, 반격 수정, 차이 피해를 설명한다.
- [ ] 수련 화면에 기본·등급·확률·제약 포인트를 분리해 표시한다.
- [ ] 예선 결승 전 절초 목표 49포인트와 현재 진행도를 표시한다.
- [ ] 심법 보유 2개와 계열별 10성 제한을 표시한다.

## 작업 7 — 테스트와 빌드 검증

- [ ] [테스트 체크리스트](../docs/08_TEST_CHECKLIST.md)의 공용 전장·2수·합 항목을 headless 테스트로 검증한다.
- [ ] 12세력 각각을 AI 또는 플레이어로 전투 종료까지 검증한다.
- [ ] 핵심무공·심법 3·6·9·10성 해금을 검증한다.
- [ ] 심법 세 번째 습득 거부와 계열별 두 번째 10성 차단을 검증한다.
- [ ] 첫 4전 A 이상 핵심무공 집중 시 5전에서 절초가 사용 가능한지 검증한다.
- [ ] 첫 4전 B 중심 플레이에서 5전 전 절초가 보장되지 않는지 검증한다.
- [ ] 일반 B등급 기대값으로 9회 수련 시 결승 전 핵심무공 10성·심법 8성 전후가 나오는지 검증한다.
- [ ] 제약 없이도 일반 성장 목표가 가능하고 제약이 추가 선택지를 제공하는지 검증한다.
- [ ] 저장·불러오기 뒤 데모 5전과 전체판 10전을 각 종료 지점까지 진행한다.
- [ ] Windows 데모 빌드와 전체판 빌드를 생성하고 핵심 흐름을 수동 검증한다.
