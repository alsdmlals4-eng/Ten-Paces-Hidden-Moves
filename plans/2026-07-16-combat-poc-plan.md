# 12세력 대회 세로 슬라이스 구현 Plan

## 목적

Godot 4.7.1에서 12세력의 공용 전투 엔진, 2수 비공개 동시 선택, 합, 핵심무공·심법 10성, 성과·행운·수련·상대 성장·제약과 전투·대회 연출을 구현한다. 먼저 5전 예선 결승까지의 데모를 완성하고 같은 구조를 10전 결승까지 확장한다.

## Codex 작업 시작 규칙

Codex는 즉시 코드를 작성하지 않는다. 반드시 Plan Mode에서 다음을 먼저 수행한다.

1. 위 문서를 읽고 승인 구조, 미확정 값, 보류 범위를 요약한다.
2. 실제 저장소 트리, `project.godot`, GDScript, 데이터, 씬, 자산과 테스트 존재 여부를 확인한다.
3. 문서와 실제 파일의 충돌, 오래된 6슬롯·1~5성·9전·구 성과 보상 흔적을 목록화한다.
4. 사용자 변경, 저장·진행·ID·공개 인터페이스의 보호 경로를 확인한다.
5. 데모 우선 구현 순서, 수정·신규 파일, 데이터 계약, 테스트 명령, 위험과 중단 기준을 제출한다.
6. 구현 필수 미확정 값은 질문으로 분리하고 임시 값으로 기준 문서를 바꾸지 않는다.
7. 사용자 승인 전에는 코드·씬·Resource·저장 포맷을 생성하거나 수정하지 않는다.

## 고정 제약

- 전장 10칸, A=4, B=6, 초기 거리 2.
- 라운드 행동 타이밍 10개, 2수 묶음 5회.
- 전체 대회 10전, 8전 8강·9전 준결승·10전 결승.
- 데모 1~2전 내부전, 3~4전 예선, 5전 예선 결승.
- AI는 각 묶음에서 플레이어 선택 전에 행동 두 개를 잠근다.
- 양측 두 행동을 동시에 공개하고 1수→2수 순으로 해상한다.
- 같은 타이밍 쌍방 공격은 합 판정 후 낮은 쪽이 차이 피해를 받는다.
- 공용 행동은 접근, 후퇴, 공격, 막기, 회피, 밀치기, 교체, 재장전, 명상이다.
- 핵심무공·심법·진의는 공용 수정자 계약을 사용한다.
- 심법 보유 최대 2개, 핵심무공 10성 최대 1개, 심법 10성 최대 1개.
- 수련 정산은 기본 5 + 등급 보너스(C1/B3/A5/S7) + 행운 0~3.
- 행운은 등급별 결과표에서 비공개로 추첨하고 `A++`처럼 표시한다.
- 상대는 A계열 60%, B계열 40%와 행운·수련 스타일을 사용한다.
- 상대 성장 결과와 플레이어용 표현 키는 회차 생성 시 고정하고 재도전에서 다시 굴리지 않는다.
- 개발용 상대 태그를 플레이어에게 직접 노출하지 않는다.
- 프레젠테이션은 전투·성장 결과를 표현하며 피해·보상·수련·저장을 직접 변경하지 않는다.
- 문파별 기믹 수행도 성과 평가는 `[보류]`이며 구현하지 않는다.

## 미확정 게이트

다음 값은 관련 작업 전에 사용자 승인이 필요하다.

- 성과 평가 지표·가중치·임계값.
- 경기별 상대 강도·등급·행운 출현표.
- 이명·풍문·전적·정탐 의미 키 첫 데이터 세트.
- 수련 스타일별 투자 알고리즘.
- 제약 후보 수·선택 상한·위험도·보상.
- 행동력·기력·내공의 정확한 비용·회복량.
- 1수 사망 시 2수 처리와 결투 라운드 상한.
- 신규 무공의 시작 성급과 결승 뒤 수련 처리.
- 최소 해상도, 입력 장치 범위와 연출 지속 시간.
