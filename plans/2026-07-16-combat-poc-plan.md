# 소림 대 당문 전투 PoC 실행 Plan

> 이 문서는 Codex 구현의 기준 Plan이다. Goal 문서 대신 이 Plan과 테스트 체크리스트로 범위·완료 조건을 관리한다.

## 목적

공용 행동을 뼈대로 두고, 권법·암기 무기 규칙과 소림·당문 유파 규칙이 10칸 거리 전투에서 재대결 욕구를 만드는지 검증한다.

## 고정 조건

- 시작 위치는 A 소림=4번 칸, B 당문=6번 칸, 초기 거리=2.
- 모든 라운드는 6시간 슬롯이다.
- 공용 행동이 전투의 뼈대이며, 유파 규칙은 공용 행동을 변형할 뿐 새 행동 체계를 만들지 않는다.
- PoC 구현 세력은 소림과 당문뿐이다.
- 문파 추가, 대성 전체, 장비 강화, 경제, 대화는 구현하지 않는다.

## 작업 체크리스트

### 1. 전장과 상태

- [ ] 10칸 전장, 위치, 거리, 같은 칸, 상호 통과를 구현한다.
- [ ] A=4, B=6 초기 배치를 자동 생성한다.
- [ ] 체력 100, 행동력 6, 기력 12, 내공 6과 라운드 회복을 구현한다.
- [ ] 체크리스트 A를 통과한다.

### 2. 공용 행동 엔진

- [ ] 접근, 후퇴, 공격, 막기, 회피, 밀치기, 재장전, 대기를 데이터 기반으로 구현한다.
- [ ] 각 행동의 시간 슬롯·행동력·기력·내공 비용을 잠금 전에 검증한다.
- [ ] 이동 → 방어 → 공격·밀치기 → 상태이상·회복 순서로 실행한다.
- [ ] 체크리스트 B를 통과한다.

### 3. 무기·유파·보조장비

- [ ] 권법과 당문 암기의 사거리·공격·막기·재장전 규칙을 공용 행동 데이터에 연결한다.
- [ ] 금강호신과 독침을 공용 행동 결과 수정 규칙으로 구현한다.
- [ ] 암기벨트 3회와 슬롯 부착을 구현한다.
- [ ] 체크리스트 C·D를 통과한다.

### 4. 행동열 UI와 AI

- [ ] 6슬롯 비공개 행동열과 잠금 상태를 구현한다.
- [ ] AI가 공개 정보로 행동열을 먼저 잠그게 한다.
- [ ] 플레이어 잠금 후 양측 행동열을 동시에 공개한다.
- [ ] 전장, 거리, 자원, 장전, 독, 암기벨트, 결과 해설을 UI에 표시한다.
- [ ] 체크리스트 E를 통과한다.

### 5. 재미 검증

- [ ] 소림과 당문으로 각각 최소 5판씩 플레이한다.
- [ ] 각 패배 뒤 플레이어가 다음 행동열의 수정안을 한 가지 이상 말할 수 있는지 기록한다.
- [ ] 테스트 참가자에게 재대결 의향을 5점 척도로 기록한다.
- [ ] 평균 4점 이상 또는 구체적인 재대결 이유 3개 이상을 PoC 통과 조건으로 삼는다.
- [ ] 체크리스트 F를 통과한다.
