# 십보강호 테스트 체크리스트

> 책임: 현재 T0 전투 POC의 자동·런타임·사용자 검증과 T1 이후 진입 게이트  
> 규칙 원본: `docs/02_COMBAT_RULES.md`  
> 범위 원본: `docs/05_COMBAT_POC_SPEC.md`

## 1. 판정 원칙

각 항목은 다음 상태 중 하나로 기록한다.

- `PASS`: 요구한 환경에서 실제로 확인함
- `PARTIAL`: 일부 경로·환경만 확인함
- `FAIL`: 기대 결과와 다름
- `NOT_RUN`: 실행하지 않음

다음을 서로 대체하지 않는다.

- 파일 존재
- 정적 계약 검사
- Godot 파싱·headless 실행
- Windows 포인터·키보드 조작
- 사용자 이해·재도전 의사
- 접근성 검수
- 성능 프로파일

## 2. T0 필수 전장 계약

- [ ] 전장이 정확히 10칸이다.
- [ ] 플레이어는 3번, 상대는 8번에서 시작한다.
- [ ] 한 칸에는 한 전투원만 위치한다.
- [ ] 전장 밖 이동이 거부된다.
- [ ] 상대 점유 칸 이동이 거부된다.
- [ ] 양측이 같은 빈 칸으로 동시에 이동하면 둘 다 제자리에 남는다.
- [ ] 플레이어와 상대의 캐릭터 스케일이 같다.
- [ ] 발 중심이 타일 앵커에 맞는다.
- [ ] 배경→전장→캐릭터→HUD 순서로 렌더된다.

## 3. 라운드·행동 슬롯

- [ ] 전투가 1라운드 1묶음 1수에서 시작한다.
- [ ] 1묶음은 1~3수다.
- [ ] 2묶음은 4~6수다.
- [ ] 3묶음은 7~10수다.
- [ ] 현재 묶음만 편집 가능하다.
- [ ] 과거 묶음은 완료 상태다.
- [ ] 미래 묶음은 잠금 상태다.
- [ ] 10수 판정 뒤 다음 라운드 1수로 이동한다.
- [ ] 10수 완료만으로 전투가 종료되지 않는다.

### 다중 슬롯

- [ ] 1슬롯 행동이 한 수를 점유한다.
- [ ] 강공이 연속된 두 슬롯을 점유한다.
- [ ] 다중 슬롯 행동은 마지막 점유 수에서 실행된다.
- [ ] 현재 묶음 경계를 넘는 배치가 거부된다.
- [ ] 점유한 어느 칸을 눌러도 행동 전체가 해제된다.

## 4. 기초 행동 8종

### 이동

- [ ] 기력·내력 비용이 없다.
- [ ] 좌·우 인접 빈 칸만 선택할 수 있다.
- [ ] 선택한 칸으로 이동한다.

### 보법

- [ ] 내력 1을 소모한다.
- [ ] 좌·우 1칸과 2칸을 선택할 수 있다.
- [ ] 1칸·2칸 중 선택한 목적지로 이동한다.
- [ ] 전장 밖·상대 점유 칸이 제외된다.

### 막기

- [ ] 공격과 같은 수면 피해 50% 감소 후보가 된다.
- [ ] 같은 행동 묶음이면 방어도 4 감소 후보가 된다.
- [ ] 두 후보 중 실제 감소량이 높은 하나를 적용한다.
- [ ] 최종 피해와 선택된 감소 효과가 로그에 표시된다.

### 회피

- [ ] 공격과 같은 수면 완전히 회피한다.
- [ ] 일반 회피는 같은 묶음의 다른 수를 보호하지 않는다.

### 속공

- [ ] 슬롯 1, 기력 1이다.
- [ ] 지정 방향 거리 1에서 피해 6을 준다.
- [ ] 이동 단계 전에 실행된다.
- [ ] 반대 방향 또는 거리 밖이면 빗나간다.

### 강공

- [ ] 슬롯 2, 기력 1·내력 1이다.
- [ ] 지정 방향 거리 1 또는 2에서 피해 8을 준다.
- [ ] 마지막 점유 수에서 실행된다.
- [ ] 반대 방향 또는 거리 3 이상이면 빗나간다.

### 명상

- [ ] 비용이 없다.
- [ ] 실행 시 기력 2·내력 1을 회복한다.
- [ ] 최대치를 넘지 않는다.
- [ ] 계획 예상값에도 실행 순서대로 반영된다.

### 태세

- [ ] 내력 1을 소모한다.
- [ ] 단독 사용 시 다음 공격 강화가 데이터와 로그에 일치한다.
- [ ] 막기 또는 회피와 결합할 수 있다.

## 5. 태세+대응 결합

### 태세+막기

- [ ] 한 슬롯을 점유한다.
- [ ] 태세와 막기 비용을 합산한다.
- [ ] 현재 행동 묶음 전체에 막기를 적용한다.
- [ ] 방어도 4가 50% 증가해 6이 된다.
- [ ] 같은 수 50% 후보와 강화 방어도 후보 중 높은 감소량을 적용한다.

### 태세+회피

- [ ] 한 슬롯을 점유한다.
- [ ] 태세와 회피 비용을 합산한다.
- [ ] 현재 행동 묶음 전체의 공격을 완전 회피한다.

### 입력

- [ ] 태세→대응 순서로 결합할 수 있다.
- [ ] 대응→태세 순서로 결합할 수 있다.
- [ ] 결합을 취소하면 자원 예상값이 복원된다.

## 6. 대상·방향 지정

- [ ] 이동 배치 후 유효 목적지만 녹색·이동 표식으로 표시된다.
- [ ] 보법은 1칸·2칸 목적지를 구분해 선택할 수 있다.
- [ ] 공격 배치 후 좌·우 방향 선택지가 붉은색·공격 표식으로 표시된다.
- [ ] 방향 지정 전에는 진행할 수 없다.
- [ ] 이동 목적지 지정 전에는 진행할 수 없다.
- [ ] 슬롯에 `← 2번`, `→ 4번`, `← 공격`, `→ 공격`이 표시된다.
- [ ] 앞선 이동을 반영한 예상 위치로 이후 대상 후보를 계산한다.
- [ ] 기본 전장 칸은 불필요하게 포인터를 가로채지 않는다.

## 7. 진행 조건

- [ ] 현재 묶음의 모든 슬롯이 채워져야 한다.
- [ ] 모든 이동·공격 대상이 지정돼야 한다.
- [ ] 다중 슬롯 점유가 유효해야 한다.
- [ ] 실행 순서 기준 예상 자원이 0 미만이면 안 된다.
- [ ] 조건을 만족하면 진행 버튼이 활성화된다.
- [ ] 진행 후 해당 묶음은 수정할 수 없다.

## 8. 자원·시작 상태

### 시작

- [ ] 체력은 20/20으로 시작한다.
- [ ] 기력은 5/5로 시작한다.
- [ ] 내력은 4/4로 시작한다.
- [ ] 시작 패널티가 없으면 현재값이 최대값과 같다.
- [ ] `start_penalties`가 있으면 해당 자원만 최대치에서 감소한다.
- [ ] 시작값이 0 아래로 내려가지 않는다.
- [ ] 절초 기세는 자동으로 최대치가 되지 않는다.

### 계획 미리보기

- [ ] 행동 배치 즉시 기력·내력 예상값이 HUD에 반영된다.
- [ ] 명상 전후 실행 순서가 예상값에 반영된다.
- [ ] 행동 해제 시 예상값이 다시 계산된다.
- [ ] 자원 부족 행동은 `자원 부족`으로 표시된다.
- [ ] 자원 부족 계획은 진행 버튼을 잠근다.
- [ ] `[진행]` 전에는 실제 전투 자원이 확정 변경되지 않는다.

## 9. 판정 순서·동시성

- [ ] 판정 순서가 `대응 → 속공 → 이동 → 일반 공격`이다.
- [ ] 속공이 이동 전에 실행된다.
- [ ] 이동 후 위치가 일반 공격의 거리 판정에 반영된다.
- [ ] 같은 수·같은 단계의 쌍방 공격 피해를 동시에 적용한다.
- [ ] 한쪽 피해가 먼저 표시됐다는 이유로 같은 단계 상대 공격이 취소되지 않는다.
- [ ] 현재 POC에 별도 합 수치 비교·차이 피해가 발생하지 않는다.
- [ ] 공격 실패 원인이 방향·거리·자원·점유 중 하나로 기록된다.

## 10. UI·상호작용

### 카드

- [ ] 기초 행동 8종이 정해진 순서로 모두 보인다.
- [ ] 카드에 `[기초]`, 이름, 기능, 사거리, 비용이 보인다.
- [ ] 비용은 행동 슬롯·기력·내력만 사용한다.
- [ ] 행동력·공통 막기 경감·덱·손패 정보가 없다.
- [ ] 호버하면 임시 상세가 열린다.
- [ ] 마우스를 벗어나면 임시 상세가 닫힌다.
- [ ] 클릭하면 상세가 고정된다.
- [ ] 같은 카드 재클릭 또는 빈 전장 클릭으로 닫힌다.

### HUD

- [ ] 상단 순서가 플레이어 상태→플레이어 기세→라운드→상대 기세→상대 상태다.
- [ ] 절초 기세가 정확히 5칸이다.
- [ ] 하단 진행 표기가 `라운드 1`처럼 라운드만 표시한다.
- [ ] 예상 자원과 실제 자원이 구분된다.

### 로그

- [ ] 기록 패널이 기본적으로 접혀 있다.
- [ ] 기록 버튼으로 펼치고 접을 수 있다.
- [ ] 행동 배치·해제·대상·판정·자원 변화가 기록된다.
- [ ] 개발 이벤트 이름보다 플레이어가 이해할 원인이 우선한다.

## 11. STEP 11 예정 검증

다음 항목은 아직 `NOT_RUN`이며 STEP 11 구현 후 활성화한다.

- [ ] 실제 피해를 받으면 이후 미실행 행동이 기본적으로 중단된다.
- [ ] 같은 단계에서 이미 성립한 공격의 동시 피해는 유지된다.
- [ ] `[집중]`은 일반 피해 중단을 방지한다.
- [ ] `[강건]`은 일반 피해 중단을 방지한다.
- [ ] 사망과 강제 제어는 집중·강건보다 우선한다.
- [ ] 취소 슬롯과 원인이 UI·로그에 표시된다.

## 12. STEP 12 AI 예정 검증

- [ ] AI가 플레이어의 비공개 배치를 읽지 않는다.
- [ ] AI가 공개 위치·자원·상태·과거 결과만 사용한다.
- [ ] AI가 관찰 가능한 성향을 유지한다.
- [ ] 완전 무작위가 아니라 상황 판단과 제한된 실수를 사용한다.
- [ ] 같은 시드와 입력에서 선택을 재현할 수 있다.

## 13. STEP 13 종료·재시작 예정 검증

- [ ] 체력 0에서 승패가 판정된다.
- [ ] 양측 동시 체력 0의 처리 규칙이 일관된다.
- [ ] 종료 뒤 입력이 잠긴다.
- [ ] 결과 화면에서 재시작할 수 있다.
- [ ] 재시작 시 전장·라운드·자원·로그·행동 슬롯이 초기화된다.

## 14. STEP 14 POC 플레이테스트

기록할 질문:

- [ ] 플레이어가 전장·거리·행동 슬롯을 설명할 수 있는가?
- [ ] `3수 → 3수 → 4수` 세 묶음의 차이를 느끼는가?
- [ ] 이동 목적지와 공격 방향을 혼동하지 않는가?
- [ ] 실패 이유를 스스로 설명할 수 있는가?
- [ ] 상대에게 반복되는 성향을 발견하는가?
- [ ] 대응과 위치 선택이 단순 피해 경쟁보다 중요하다고 느끼는가?
- [ ] 한 판 후 바로 다시 시도하고 싶은가?
- [ ] 다시 한다면 어떤 계획을 바꾸겠다고 말하는가?

## 15. 접근성 검증

- [ ] 색 없이도 이동·공격·잠금·부족 상태를 구분한다.
- [ ] 키보드만으로 카드→슬롯→대상→진행→상세→기록을 조작한다.
- [ ] 긴 한국어 문구가 잘리지 않는다.
- [ ] 최소 해상도에서 전장과 진행 버튼이 화면 밖으로 밀리지 않는다.
- [ ] 점멸·흔들림·빠른 모션을 줄이는 경로가 있다.
- [ ] 자산 누락 시 기본 도형·텍스트로 기능을 이해할 수 있다.

현재 실제 접근성 사용자 검수는 `NOT_RUN`이다.

## 16. 성능 검증

- [ ] 목표 Windows 사양을 정의한다.
- [ ] 동일 빌드에서 baseline을 기록한다.
- [ ] 대표 장면과 최악 장면을 분리한다.
- [ ] frame time·CPU·GPU·메모리·로딩 시간을 기록한다.
- [ ] 평균 FPS 하나만으로 통과시키지 않는다.

현재 목표 플랫폼 성능 프로파일은 `NOT_RUN`이다.

## 17. T1·T2·전체판 게이트

T0 통과 전에는 다음을 현재 빌드 완료 기준으로 사용하지 않는다.

- 상대 3명과 플레이 스타일 2개
- 전투 전 상대 정보
- 전투 후 보상과 성장
- 5전 데모
- 12세력·10전 전체판
- 무공·심법·절초·진의
- 정탐·제약·행운·재도전·저장

T1 진입 조건:

- STEP 0~13 실제 런타임 통과
- STEP 14에서 핵심 규칙 이해와 재도전 의사 확인
- 치명적 입력·판정·복기 문제 없음
- POC에서 유지할 규칙과 바꿀 규칙을 명시
