## 1. 판정 상태

| 상태 | 의미 |
|---|---|
| `PASS` | 명시한 환경·버전·입력에서 기대 결과를 실제 확인 |
| `PARTIAL` | 일부 환경·경로만 확인 |
| `FAIL` | 실제 결과가 기대와 다름 |
| `NOT_RUN` | 실행하지 않음 |
| `BLOCKED` | 도구·권한·환경·선행 구현 부족으로 실행 불가 |

다음을 서로 대체하지 않는다.

1. 파일 존재.
2. 형식·문법·정적 계약.
3. 자동 테스트.
4. Godot 파싱·headless·에디터 런타임.
5. Windows 포인터·키보드·렌더.
6. 실제 사용자 규칙 이해와 재도전 행동.
7. 보조기기 사용성.
8. 목표 플랫폼 Release 성능.
9. Branch protection Required Check 강제.

## 4. 전장·점유·대상

### BOARD-001 기본 구조

- [ ] 전장이 정확히 10칸이고 1~10 순서다.
- [ ] 플레이어는 4번, 상대는 7번에서 시작한다.
- [ ] 시작 거리는 정확히 3이다.
- [ ] 양측 발 중심이 타일 앵커에 맞는다.
- [ ] JSON·fallback·runtime state·씬 메타·참조 자산이 같은 값을 사용한다.

### BOARD-002 밀착·공동 목적지

- [ ] 정지 상대 칸에 진입하면 양측이 같은 칸에 남는다.
- [ ] 같은 빈 칸을 공동 목적지로 선택하면 양측이 함께 이동한다.
- [ ] 같은 칸은 거리 0 `[밀착]`으로 표시된다.
- [ ] 한 타일 최대 전투원 수는 2다.
- [ ] 밀착에서 사거리 1 이상 공격은 방향과 무관하게 유효하다.
- [ ] 밀착·공동 이동 이유가 로그와 상태에 나타난다.

### BOARD-003 금지 이동

- [ ] 전장 밖 목적지는 선택되지 않는다.
- [ ] 양측 자리 교환을 차단한다.
- [ ] 상대를 통과해 반대편으로 이동하지 않는다.
- [ ] 유효하지 않은 이동은 비용·실패 이유 계약에 따라 처리된다.

### BOARD-004 시작 위치 후보

- [ ] 기본 이동은 4번에서 3번·5번을 표시한다.
- [ ] 보법은 4번에서 2번·3번·5번·6번을 표시한다.
- [ ] 시작 거리 3에서 이동 없는 속공·강공은 사거리 실패다.
- [ ] 첫 이동 fixture 4→5·7→6 뒤 거리 1이 된다.

## 5. 라운드·슬롯·계획

### TIMING-001 묶음

- [ ] 1묶음 1~3, 2묶음 4~6, 3묶음 7~10이다.
- [ ] 현재 묶음만 편집 가능하다.
- [ ] 지난 묶음은 수정할 수 없고 미래 묶음은 잠긴다.
- [ ] 10수 뒤 승패가 없으면 다음 라운드 1수로 간다.

### TIMING-002 진행 게이트

- [ ] 빈 슬롯이 있으면 진행할 수 없다.
- [ ] 이동 목적지·공격 방향 미지정은 진행을 막는다.
- [ ] 예상 기력·내력 부족은 진행을 막는다.
- [ ] 모든 조건 충족 뒤 한 번만 판정 요청한다.
- [ ] resolving 동안 카드·슬롯·대상·절초 입력을 잠근다.

### TIMING-003 다중 슬롯

- [ ] 2슬롯 강공은 연속 두 칸을 점유하고 마지막 수에 실행한다.
- [ ] 3슬롯 파공검기는 한 묶음 안에서만 배치한다.
- [ ] 앞 수는 `[준비]`, 마지막 수는 `[실행]`으로 표시한다.
- [ ] 점유 슬롯 클릭은 `[진행]` 전 행동 전체를 해제한다.
- [ ] 묶음 경계 초과 배치를 거부한다.

## 6. 기초 행동·자원

### ACTION-001 데이터 계약

- [ ] 기초 행동 ID가 정확히 8개다.
- [ ] 행동력·내공·공통 `guard_reduction` 필드가 없다.
- [ ] 슬롯·기력·내력·이동·사거리·피해가 `basic_cards.json`과 일치한다.

### RESOURCE-001 시작 상태

- [ ] 양측 체력 30, 공격력 8, 기력 5, 내력 4, 기세 0/5다.
- [ ] `start_penalties`가 최대치에서 차감된다.
- [ ] 재시작도 같은 시작 상태를 만든다.

### RESOURCE-002 계획 미리보기

- [ ] 배치 즉시 예상 기력·내력이 갱신된다.
- [ ] 명상 회복이 실행 순서에 따라 이후 행동에 반영된다.
- [ ] 행동 해제 시 예상값이 복원된다.
- [ ] 예상 상태가 실제 `combat_state`를 변경하지 않는다.
- [ ] 판정 뒤 실제 값이 유효한 직전 예상값과 일치한다.

## 7. 합·방어·회피·필중

### CLASH-001 기본 합

- [ ] 같은 실행 수의 양측 유효 공격은 속공/일반 구분 없이 합이다.
- [ ] 높은 원공격력과 낮은 원공격력의 차이가 원피해가 된다.
- [ ] 동점은 `[합 상쇄]`로 양측 피해 0이다.
- [ ] 합 승자는 최종 피해가 방어로 0이어도 기세 1을 얻는다.
- [ ] 중단·사망은 합 정산 뒤 한 번만 처리된다.

### DEFENSE-001 막기 순서

- [ ] 기본 방어도는 4다.
- [ ] 먼저 `max(0, 원피해-방어도)`를 계산한다.
- [ ] 같은 수 막기면 남은 피해를 `floor(×0.5)`한다.
- [ ] 태세+막기는 현재 묶음 방어도 6이다.
- [ ] 합 차이 피해에도 같은 순서를 적용한다.

### DEFENSE-002 회피·필중

- [ ] 기본 회피는 같은 수 공격을 완전 회피한다.
- [ ] 태세+회피는 현재 묶음 전체를 회피한다.
- [ ] 파공검기 `[필중]`은 회피만 무시한다.
- [ ] 필중도 방어도·합·방향·사거리·중단·전투 불능을 무시하지 않는다.

## 8. 중단·강건·상태

### INTERRUPT-001 같은 수 중단

- [ ] 실제 피해가 같은 수의 아직 실행 전 행동만 중단한다.
- [ ] 이미 성립한 합·공격은 유지된다.
- [ ] 앞선 `[준비]` 이벤트는 역사로 남는다.
- [ ] 이후 수 계획은 유지된다.
- [ ] 전투 불능은 이후 수 행동까지 취소한다.
- [ ] UI·로그가 중단 대상과 이유를 설명한다.

### FORTITUDE-001 강건

- [ ] 단독 태세 실행 뒤 다음 공격 +2와 `[강건]`을 저장한다.
- [ ] 강건은 속공 단계 피격 중단을 한 번 막는다.
- [ ] 피해와 체력 감소는 유지된다.
- [ ] 1슬롯 공격만 보호한다.
- [ ] 2·3슬롯 행동과 사망은 보호하지 않는다.
- [ ] 보호 사용 뒤 상태가 제거된다.
- [ ] `[집중]` 상태가 데이터·UI·활성 문서에 없다.

## 9. 절초

### ULTIMATE-002 데이터

- [ ] 일반·절초 적중 자체는 기세를 올리지 않는다.
