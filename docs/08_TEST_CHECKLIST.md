## 1. 판정 상태

| 상태 | 의미 |
|---|---|
| `PASS` | 명시한 환경·버전·입력에서 실제 결과를 확인 |
| `PARTIAL` | 일부 환경·경로만 확인 |
| `FAIL` | 실제 결과가 기대 결과와 다름 |
| `NOT_RUN` | 실행하지 않음 |
| `BLOCKED` | 도구·권한·선행 구현 부족으로 실행 불가 |

다음을 서로 대체하지 않는다.

1. 파일 존재.
2. 문법·정적 계약.
3. Godot 파싱·headless 실행.
4. Windows 포인터·키보드 조작.
5. 사용자 규칙 이해·재도전 의사.
6. 접근성 실제 검수.
7. 목표 플랫폼 성능 프로파일.
8. Branch protection Required Check 강제.

체크박스는 해당 항목의 실제 증거가 기록됐을 때만 완료한다.

## 4. T0 필수 전장 계약

### BOARD-001 기본 구조

- [ ] 전장이 정확히 10칸이다.
- [ ] 타일 번호가 1~10 순서다.
- [ ] 플레이어는 4번, 상대는 7번에서 시작한다.
- [ ] 시작 거리는 정확히 3칸이다.
- [ ] 플레이어와 상대의 스케일이 같다.
- [ ] 양측 발 중심이 해당 타일 앵커에 맞는다.
- [ ] 전장 JSON·씬 메타·런타임 state가 같은 시작 위치를 사용한다.

### BOARD-002 점유·경계

- [ ] 한 칸에 한 전투원만 위치한다.
- [ ] 전장 밖 목적지가 선택 후보에 나타나지 않는다.
- [ ] 상대 점유 칸이 이동 후보에 나타나지 않는다.
- [ ] 조작되지 않은 기본 칸이 포인터 입력을 가로채지 않는다.

### BOARD-003 시작 위치 대상 후보

- [ ] 기본 이동은 4번에서 3번·5번만 후보로 표시한다.
- [ ] 보법은 4번에서 2번·3번·5번·6번을 후보로 표시한다.
- [ ] 상대가 7번을 점유하므로 7번은 보법 후보가 아니다.
- [ ] 시작 거리 3에서 속공은 어느 방향을 선택해도 적중하지 않는다.
- [ ] 시작 거리 3에서 강공은 어느 방향을 선택해도 적중하지 않는다.

### BOARD-004 동시 이동

정상 fixture:

- [ ] 플레이어가 4번→5번, 상대가 7번→6번으로 동시에 이동한다.
- [ ] 판정 후 두 전투원은 5번·6번에서 인접한다.

경계:

- [ ] 양측이 같은 빈 목적지를 고르면 둘 다 제자리에 남는다.
- [ ] 한쪽 이동만 무효면 유효한 쪽만 정상 처리된다.

금지 회귀:

- [ ] 현재 POC에서 같은 칸 중첩·통과·자리 교환이 발생하지 않는다.

## 5. 라운드·행동 묶음

### TIMING-001 시작

- [ ] 전투가 1라운드 1묶음 1수에서 시작한다.
- [ ] 1~3수 모두 편집 가능하다.
- [ ] 4~10수는 잠금 상태다.

### TIMING-002 묶음

- [ ] 1묶음은 1~3수다.
- [ ] 2묶음은 4~6수다.
- [ ] 3묶음은 7~10수다.
- [ ] 현재 묶음만 편집 가능하다.
- [ ] 판정 완료 묶음은 다시 수정할 수 없다.

### TIMING-003 라운드 진행

- [ ] 1묶음 판정 뒤 4수부터 열린다.
- [ ] 2묶음 판정 뒤 7수부터 열린다.
- [ ] 10수 판정 뒤 다음 라운드 1수로 이동한다.
- [ ] 10수 완료만으로 승패 화면이 열리지 않는다.

### TIMING-004 다중 슬롯

정상:

- [ ] 강공이 연속 두 슬롯을 점유한다.
- [ ] 강공은 마지막 점유 수에서 실행된다.
- [ ] 점유한 어느 슬롯을 눌러도 전체가 해제된다.

실패·경계:

- [ ] 묶음 끝에 한 칸만 남았으면 강공을 배치하지 못한다.
- [ ] 점유 슬롯과 겹치면 배치하지 못한다.
- [ ] 미래 묶음으로 넘어가는 배치를 허용하지 않는다.

## 6. 기초 행동 8종

### ACTION-001 이동

- [ ] 슬롯 1, 기력 0, 내력 0이다.
- [ ] 4번 시작에서 3번 또는 5번을 선택한다.
- [ ] 선택한 칸으로 이동한다.
- [ ] 비용이 없어도 목적지 지정은 필요하다.

### ACTION-002 보법

- [ ] 슬롯 1, 내력 1이다.
- [ ] 4번 시작에서 좌우 1칸·2칸을 구분해 선택한다.
- [ ] 4번→6번 2칸 이동이 가능하다.
- [ ] 내력이 부족하면 계획이 무효다.

### ACTION-003 막기

정상:

- [ ] 같은 묶음의 공격은 방어도 4 감소 후보를 가진다.
- [ ] 같은 수 공격은 원피해 50% 감소 후보도 가진다.
- [ ] 두 후보 중 감소량이 큰 하나만 적용한다.
- [ ] 최종 피해는 0 아래로 내려가지 않는다.

반례:

- [ ] 피해 6·같은 수에서 방어도 4가 선택되어 최종 피해 2다.
- [ ] 피해 12·같은 수에서 50% 감소량 6이 선택되어 최종 피해 6이다.
- [ ] 피해 12·다른 수 같은 묶음에서 방어도 4만 적용되어 최종 피해 8이다.

### ACTION-004 회피

- [ ] 같은 수 공격은 최종 피해 0이다.
- [ ] 같은 묶음의 다른 수는 일반 회피로 보호하지 않는다.
- [ ] 방향·사거리 실패 공격과 회피 성공을 같은 원인으로 기록하지 않는다.

### ACTION-005 속공

- [ ] 슬롯 1, 기력 1, 사거리 1, 피해 6이다.
- [ ] 같은 수 이동 전에 실행된다.
- [ ] 시작 거리 3에서는 적중하지 않는다.
- [ ] 양측 이동 후 거리 1에서는 오른쪽 지정 시 적중한다.
- [ ] 반대 방향 지정 시 실패한다.
- [ ] 비용은 실패해도 반환하지 않는다.

### ACTION-006 강공

- [ ] 슬롯 2, 기력 1·내력 1, 피해 8이다.
- [ ] 지정 방향 거리 1에서 적중한다.
- [ ] 지정 방향 거리 2에서 적중한다.
- [ ] 시작 거리 3에서는 실패한다.
- [ ] 반대 방향에서 실패한다.
- [ ] 마지막 점유 수 전에 실행하지 않는다.

### ACTION-007 명상

- [ ] 슬롯 1, 비용 0이다.
- [ ] 실행 시 기력 2·내력 1을 회복한다.
- [ ] 최대치를 넘지 않는다.
- [ ] 실제 실행 순서대로 계획 예상값에 반영된다.

### ACTION-008 태세

- [ ] 슬롯 1, 내력 1이다.
- [ ] 단독 실행 후 다음 유효 공격 피해가 2 증가한다.
- [ ] 공격이 실행된 뒤 보너스가 소모된다.
- [ ] 막기 또는 회피와 결합할 수 있다.
- [ ] 대응 결합 시 단독 공격 보너스를 중복 적용하지 않는다.

## 7. 태세+대응

### RESPONSE-001 입력

- [ ] 태세→막기와 막기→태세가 모두 결합된다.
- [ ] 태세→회피와 회피→태세가 모두 결합된다.
- [ ] 결합 행동은 한 슬롯을 점유한다.
- [ ] 결합 취소 시 예상 자원이 복원된다.

### RESPONSE-002 태세+막기

- [ ] 묶음 전체에 막기를 적용한다.
- [ ] 방어도 4×1.5를 올림해 6으로 사용한다.
- [ ] 같은 수에는 50% 후보와 방어도 6 후보 중 큰 감소량을 사용한다.
- [ ] 결합 비용이 예상 자원과 실제 판정에서 일치한다.

### RESPONSE-003 태세+회피

- [ ] 묶음의 모든 수에서 공격 피해를 0으로 만든다.
- [ ] 방향·사거리 실패와 회피 성공을 로그에서 구분한다.
- [ ] 결합 비용이 예상 자원과 실제 판정에서 일치한다.

밸런스 관찰:

- [ ] 태세+회피가 비용 대비 과도하게 안전한지 기록한다.
- [ ] 적 AI가 해당 상태에 대응할 수단이 필요한지 기록한다.

## 8. 대상·방향 지정

### TARGET-001 이동

- [ ] 이동 후보는 녹색·이동 표식으로 구분된다.
- [ ] 시작 위치 4번에서 기본 이동 후보가 3번·5번이다.
- [ ] 보법 1칸·2칸 후보를 모두 표시한다.
- [ ] 선택 후 슬롯에 방향과 도착 칸 번호가 보인다.
- [ ] 목적지 지정 전 진행 버튼은 비활성이다.

### TARGET-002 공격

- [ ] 속공은 좌·우 거리 1 후보를 표시한다.
- [ ] 강공은 좌·우 거리 1·2 후보를 표시한다.
- [ ] 선택 후 슬롯에 `← 공격` 또는 `→ 공격`이 보인다.
- [ ] 방향 지정 전 진행 버튼은 비활성이다.
- [ ] 적이 반대 방향에 있어도 자동으로 방향을 바꾸지 않는다.

### TARGET-003 예상 위치

- [ ] 1수 이동 4번→5번을 반영해 3수 공격 후보를 5번 기준으로 계산한다.
- [ ] 이동을 해제하면 이후 대상 후보도 4번 기준으로 다시 검증한다.
- [ ] 실행 시점 실제 위치가 계획과 다르면 무효 원인을 기록한다.

## 9. 진행·자원 계획

### GATE-001 진행 조건

진행 패널은 캡션 `행동 묶음 판정`과 상태 문구 `배치 완료`, `행동·대상·자원 확인 필요`, `판정 완료`를 실제 상태와 일치시킨다.

- [ ] 현재 묶음의 모든 슬롯이 채워져야 한다.
- [ ] 모든 이동·공격 대상이 지정돼야 한다.
- [ ] 다중 슬롯 점유가 유효해야 한다.
- [ ] 실행 순서 기준 예상 자원이 0 미만이면 안 된다.
- [ ] 조건 충족 시 상태 문구가 `배치 완료`다.
- [ ] 조건 미충족 시 `행동·대상·자원 확인 필요`다.
- [ ] 판정 완료 후 `판정 완료`다.

현재 판정은 동기 처리다. 별도 `resolving` 상태 테스트는 단계별 연출 구현 후 추가한다.

### RESOURCE-001 시작 상태

- [ ] 체력 20/20, 기력 5/5, 내력 4/4로 시작한다.
- [ ] 시작 패널티가 없으면 현재값이 최대값과 같다.
- [ ] `start_penalties`가 해당 자원에서만 감소한다.
- [ ] 시작값이 0 아래로 내려가지 않는다.
- [ ] 절초 기세를 자동 최대치로 만들지 않는다.

### RESOURCE-002 계획 예상값

- [ ] 행동 배치 즉시 기력·내력 예상값이 HUD에 반영된다.
- [ ] 명상 전후 실행 순서가 예상값에 반영된다.
- [ ] 행동 해제 시 처음부터 다시 계산한다.
- [ ] 부족 행동 슬롯에 `자원 부족`이 보인다.
- [ ] 부족 계획은 진행 버튼을 잠근다.
- [ ] 진행 전 실제 전투 자원은 변경되지 않는다.
- [ ] 진행 후 실제 결과와 직전 예상값이 일치한다.

## 10. 판정 순서·동시성

### RESOLVE-001 순서

현재 판정 순서는 `대응 → 속공 → 이동 → 일반 공격`이다.

- [ ] 대응 프로필을 먼저 준비한다.
- [ ] 속공이 이동 전에 실행된다.
- [ ] 이동 후 위치가 일반 공격에 반영된다.
- [ ] 일반 공격 후 명상·단독 태세를 처리한다.

### RESOLVE-002 4/7 첫 묶음 fixture

- [ ] 판정 전 플레이어 4번·상대 7번이다.
- [ ] 1수 이동 후 플레이어 5번·상대 6번이다.
- [ ] 3수 속공이 5번에서 오른쪽을 향해 거리 1의 상대에게 적중한다.
- [ ] 다음 묶음이 4수부터 열린다.

### RESOLVE-003 동시 공격

- [ ] 같은 수·같은 공격 단계 피해를 모두 계산한 뒤 동시에 적용한다.
- [ ] 화면 표시 순서 때문에 상대 공격을 취소하지 않는다.
- [ ] 현재 POC에서 별도 합 승패·차이 피해가 발생하지 않는다.

### RESOLVE-004 실패 이유

- [ ] 방향 실패.
