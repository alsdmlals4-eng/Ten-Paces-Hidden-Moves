## 1. 판정 상태

| 상태 | 의미 |
|---|---|
| `PASS` | 명시한 환경·버전·입력에서 실제 결과를 확인 |
| `PARTIAL` | 일부 환경·경로만 확인 |
| `FAIL` | 실제 결과가 기대 결과와 다름 |
| `NOT_RUN` | 실행하지 않음 |
| `BLOCKED` | 도구·권한·선행 구현 부족으로 실행 불가 |

다음을 서로 대체하지 않는다.

1. 파일 존재.
2. 문법·정적 계약.
3. Godot 파싱·headless 실행.
4. Windows 포인터·키보드 조작.
5. 사용자 규칙 이해·재도전 의사.
6. 접근성 실제 검수.
7. 목표 플랫폼 성능 프로파일.
8. Branch protection Required Check 강제.

체크박스는 해당 항목의 실제 증거가 기록됐을 때만 완료한다.

### Issue #13 STEP 14 증거 상태 (2026-07-23)

| 시나리오 | 자동·Godot 증거 | 사람 관찰 |
|---|---|---|
| 3수→3수→4수와 다음 라운드 | MCP 런타임 `1~3 → 4~6 → 7~10 → 2라운드 1수`, `verify_combat_presentation_liveness.gd` 회귀 | `NOT_RUN` |
| 속공 대 일반 공격 합 | MCP 런타임 엔진 6 대 8 → 플레이어 체력 28, `clash` 이벤트 | `NOT_RUN` |
| 동점 합 상쇄 | MCP 런타임 엔진 양측 체력 30, `clash_draw` | `NOT_RUN` |
| 태세 강화·방어도 차감 후 반감 | MCP 런타임 태세 강화 막기 피해 12 → 체력 24, `verify_response_rules.gd` 계약 | `NOT_RUN` |
| 파공검기 필중·회피 및 종료 후 재시작 | MCP 런타임 파공검기 대 회피 적 체력 0·`sure_hit`, 재시작 4/7·30·`planning`·입력 해제 | `NOT_RUN` |

Windows HiGodot 실시간 렌더는 960×600 프레임 캡처와 게임 로그 무오류로 확인했다. 이는 개발자 또는 실제 사용자의 규칙 이해·연출 가독성 관찰을 대체하지 않는다.

현재 작업 환경의 별도 `Godot_v4.7-stable_win64_console.exe --headless`는 프로젝트 로드 전 네이티브 `signal 11`로 종료되어 이번 변경의 재실행 근거로 사용하지 않았다. 같은 프로젝트는 MCP 연결 Godot 4.7 에디터에서 live 실행·렌더·무오류 게임 로그로 확인했다. 이 실행기 문제는 사람 관찰이나 실제 사용자 접근성 검증을 대체하지 않으며, 해당 별도 headless 재실행은 `NOT_RUN`이다.

#### 개발자 런타임 기록

| 관찰 항목 | 기록 |
|---|---|
| 규칙·피격 이유 | 합 6 대 8은 차이 피해 2와 `clash`, 동점은 `clash_draw`, 파공검기 대 회피는 `sure_hit`으로 엔진 반환값을 직접 확인했다. 중단이 발생하지 않는 두 합 시나리오에는 중단 사유가 없다. |
| 입력·키보드 | 진행 중 `resolving`은 잠금 상태이고, 재시작 버튼은 숨김일 때 Tab 순서를 가로채지 않으며 표시되면 모션 감소 뒤·음향 제어 앞에 연결됨을 MCP 런타임에서 확인했다. |
| 모션 감소 | 제어 자체와 결과 텍스트 보존은 기존 회귀 증거가 있으나, 다섯 시나리오 각각의 주관적 읽기성 관찰은 `NOT_RUN`이다. |
| 오류·프레임 | Issue #13 런타임은 live·게임 로그 오류 없음으로 확인했다. 이 표의 기계적 시나리오별 frame-time 표본은 별도 측정하지 않아 `NOT_RUN`; 대표/최악 장면의 이전 DEBUG 표본은 아래 성능 기록을 따른다. |

## 4. T0 필수 전장 계약

### BOARD-001 기본 구조

- [ ] 전장이 정확히 10칸이다.
- [ ] 타일 번호가 1~10 순서다.
- [ ] 플레이어는 4번, 상대는 7번에서 시작한다.
- [ ] 시작 거리는 정확히 3칸이다.
- [ ] 플레이어와 상대의 스케일이 같다.
- [ ] 양측 발 중심이 해당 타일 앵커에 맞는다.
- [ ] 전장 JSON·씬 메타·런타임 state가 같은 시작 위치를 사용한다.

### BOARD-002 점유·경계

- [ ] 한 칸에 한 전투원만 위치한다.
- [ ] 전장 밖 목적지가 선택 후보에 나타나지 않는다.
- [ ] 상대 점유 칸이 이동 후보에 나타나지 않는다.
- [ ] 조작되지 않은 기본 칸이 포인터 입력을 가로채지 않는다.

### BOARD-003 시작 위치 대상 후보

- [ ] 기본 이동은 4번에서 3번·5번만 후보로 표시한다.
- [ ] 보법은 4번에서 2번·3번·5번·6번을 후보로 표시한다.
- [ ] 상대가 7번을 점유하므로 7번은 보법 후보가 아니다.
- [ ] 시작 거리 3에서 속공은 어느 방향을 선택해도 적중하지 않는다.
- [ ] 시작 거리 3에서 강공은 어느 방향을 선택해도 적중하지 않는다.

### BOARD-004 동시 이동

정상 fixture:

- [ ] 플레이어가 4번→5번, 상대가 7번→6번으로 동시에 이동한다.
- [ ] 판정 후 두 전투원은 5번·6번에서 인접한다.

경계:

- [ ] 양측이 같은 빈 목적지를 고르면 둘 다 제자리에 남는다.
- [ ] 한쪽 이동만 무효면 유효한 쪽만 정상 처리된다.

금지 회귀:

- [ ] 현재 POC에서 같은 칸 중첩·통과·자리 교환이 발생하지 않는다.

## 5. 라운드·행동 묶음

### TIMING-001 시작

- [ ] 전투가 1라운드 1묶음 1수에서 시작한다.
- [ ] 1~3수 모두 편집 가능하다.
- [ ] 4~10수는 잠금 상태다.

### TIMING-002 묶음

- [ ] 1묶음은 1~3수다.
- [ ] 2묶음은 4~6수다.
- [ ] 3묶음은 7~10수다.
- [ ] 현재 묶음만 편집 가능하다.
- [ ] 판정 완료 묶음은 다시 수정할 수 없다.

### TIMING-003 라운드 진행

- [ ] 1묶음 판정 뒤 4수부터 열린다.
- [ ] 2묶음 판정 뒤 7수부터 열린다.
- [ ] 10수 판정 뒤 다음 라운드 1수로 이동한다.
- [ ] 10수 완료만으로 승패 화면이 열리지 않는다.

### TIMING-004 다중 슬롯

정상:

- [ ] 강공이 연속 두 슬롯을 점유한다.
- [ ] 강공은 마지막 점유 수에서 실행된다.
- [ ] 점유한 어느 슬롯을 눌러도 전체가 해제된다.

실패·경계:

- [ ] 묶음 끝에 한 칸만 남았으면 강공을 배치하지 못한다.
- [ ] 점유 슬롯과 겹치면 배치하지 못한다.
- [ ] 미래 묶음으로 넘어가는 배치를 허용하지 않는다.

## 6. 기초 행동 8종

### ACTION-001 이동

- [ ] 슬롯 1, 기력 0, 내력 0이다.
- [ ] 4번 시작에서 3번 또는 5번을 선택한다.
- [ ] 선택한 칸으로 이동한다.
- [ ] 비용이 없어도 목적지 지정은 필요하다.

### ACTION-002 보법

- [ ] 슬롯 1, 내력 1이다.
- [ ] 4번 시작에서 좌우 1칸·2칸을 구분해 선택한다.
- [ ] 4번→6번 2칸 이동이 가능하다.
- [ ] 내력이 부족하면 계획이 무효다.

### ACTION-003 막기

정상:

- [ ] 같은 묶음의 공격은 방어도 4 감소 후보를 가진다.
- [ ] 같은 수 공격은 원피해 50% 감소 후보도 가진다.
- [ ] 두 후보 중 감소량이 큰 하나만 적용한다.
- [ ] 최종 피해는 0 아래로 내려가지 않는다.

반례:

- [ ] 피해 6·같은 수에서 방어도 4가 선택되어 최종 피해 2다.
- [ ] 피해 12·같은 수에서 50% 감소량 6이 선택되어 최종 피해 6이다.
- [ ] 피해 12·다른 수 같은 묶음에서 방어도 4만 적용되어 최종 피해 8이다.

### ACTION-004 회피

- [ ] 같은 수 공격은 최종 피해 0이다.
- [ ] 같은 묶음의 다른 수는 일반 회피로 보호하지 않는다.
- [ ] 방향·사거리 실패 공격과 회피 성공을 같은 원인으로 기록하지 않는다.

### ACTION-005 속공

- [ ] 슬롯 1, 기력 1, 사거리 1, 피해 6이다.
- [ ] 같은 수 이동 전에 실행된다.
- [ ] 시작 거리 3에서는 적중하지 않는다.
- [ ] 양측 이동 후 거리 1에서는 오른쪽 지정 시 적중한다.
- [ ] 반대 방향 지정 시 실패한다.
- [ ] 비용은 실패해도 반환하지 않는다.

### ACTION-006 강공

- [ ] 슬롯 2, 기력 1·내력 1, 피해 8이다.
- [ ] 지정 방향 거리 1에서 적중한다.
- [ ] 지정 방향 거리 2에서 적중한다.
- [ ] 시작 거리 3에서는 실패한다.
- [ ] 반대 방향에서 실패한다.
- [ ] 마지막 점유 수 전에 실행하지 않는다.

### ACTION-007 명상

- [ ] 슬롯 1, 비용 0이다.
- [ ] 실행 시 기력 2·내력 1을 회복한다.
- [ ] 최대치를 넘지 않는다.
- [ ] 실제 실행 순서대로 계획 예상값에 반영된다.

### ACTION-008 태세

- [ ] 슬롯 1, 내력 1이다.
- [ ] 단독 실행 후 다음 유효 공격 피해가 2 증가한다.
- [ ] 공격이 실행된 뒤 보너스가 소모된다.
- [ ] 막기 또는 회피와 결합할 수 있다.
- [ ] 대응 결합 시 단독 공격 보너스를 중복 적용하지 않는다.

## 8. 대상·방향 지정

### TARGET-001 이동

- [ ] 이동 후보는 녹색·이동 표식으로 구분된다.
- [ ] 시작 위치 4번에서 기본 이동 후보가 3번·5번이다.
- [ ] 보법 1칸·2칸 후보를 모두 표시한다.
- [ ] 선택 후 슬롯에 방향과 도착 칸 번호가 보인다.
- [ ] 목적지 지정 전 진행 버튼은 비활성이다.

### TARGET-002 공격

- [ ] 속공은 좌·우 거리 1 후보를 표시한다.
- [ ] 강공은 좌·우 거리 1·2 후보를 표시한다.
- [ ] 선택 후 슬롯에 `← 공격` 또는 `→ 공격`이 보인다.
- [ ] 방향 지정 전 진행 버튼은 비활성이다.
- [ ] 적이 반대 방향에 있어도 자동으로 방향을 바꾸지 않는다.
