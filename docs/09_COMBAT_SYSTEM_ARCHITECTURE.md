# 전투 시스템 아키텍처

> 원칙: 무공이 전투 규칙을 새로 만들지 않는다. 공용 행동과 무기 규칙을 먼저 해상하고, 유파 무공·대성은 정해진 수정 지점만 바꾼다.

## 1. 다섯 층

| 층 | 책임 | 예시 |
|---|---|---|
| 공용 행동 | 전투에서 가능한 기본 선택과 슬롯 해상 | 접근, 후퇴, 공격, 막기, 회피, 밀치기, 무기 교체, 재장전 |
| 무기 규칙 | 사거리, 행동력·기력 비용, 막기·밀치기 효율, 재장전 | 권법 0, 창 2, 활 3~5·재장전 |
| 보조장비 | 기존 행동에 붙는 제한 횟수의 무행동 효과 | 암기벨트, 화살통, 방패 |
| 유파 무공 | 공용 행동의 조건·효율·결과 수정 | 금강호신이 막기를 강화, 독침이 공격에 독 부여 |
| 대성 | 문파 규칙의 단계 해금과 외문 수련 | 1~5성, 외문 규칙 슬롯 |

## 2. 엔진 계약

1. 모든 행동은 공용 행동 타입으로 기록한다.
2. 무기는 행동 가능 여부와 기본 수치를 판정한다.
3. 장비·유파·대성은 허용된 수정값만 제공한다: 비용, 사거리, 피해, 경감, 이동량, 상태 부여, 재장전 상태, 사용 횟수.
4. 문파 전용 행동 타입·전용 슬롯·전용 해상 순서는 만들지 않는다.
5. 상태는 독·마화·장전·방패 등 공통 상태만 쓴다.
6. 모든 수정값은 결과 로그에서 원인과 함께 표시한다.

## 3. 슬롯 해상 순서

1. 양측의 해당 슬롯 행동과 보조장비를 확인한다.
2. 행동 가능 여부와 비용을 판정한다.
3. 이동을 동시에 처리한다.
4. 막기·회피·방패 효과를 적용한다.
5. 이동 뒤 최종 거리를 기준으로 공격·밀치기를 판정한다.
6. 피해·독·마화·장전 등 상태를 동시에 반영한다.
7. 라운드 종료 시 기력·내공 회복과 독·마화 피해를 처리한다.

## 4. 11세력 적용

| 계열 | 공용 행동의 주 수정 지점 |
|---|---|
| 근접 압박 | 소림 권법, 오독 독장: 거리 0 공격·상태 |
| 검·도 기동 | 화산·무당·마교: 접근·후퇴·회피·체력 대가 |
| 거리 2 장악 | 개방 곤법, 양가장 창법, 관군 창방패: 밀치기·막기·회수 |
| 원거리 견제 | 당문 암기, 백우궁 활: 재장전·보조장비·거리 유지 |
| 자원전 | 아미 점혈·의술, 무당 심법, 마교 마화: 기력·내공·체력 변환 |

## 5. 세로 슬라이스 검증

- 시작 6세력과 추가 5세력은 모두 같은 엔진 계약으로 동작한다.
- 한 회차 9전은 모든 대진 단계를 통과하지만, 전 세력을 한 번에 만날 필요는 없다.
- 테스트 매트릭스는 11세력 전부의 무기·유파·대성·AI 운용을 별도로 검증한다.
- 새로운 문파를 추가할 때 엔진 수정 없이 데이터와 허용된 수정자로 구현할 수 있어야 한다.
