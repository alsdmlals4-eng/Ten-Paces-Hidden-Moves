# 전투 시스템 아키텍처

> 원칙: 문파 무공이 새 전투 절차를 만들지 않는다. 공용 행동·합·자원 절차를 먼저 해상하고, 무기·성급·심법·진의는 허용된 수정 지점만 바꾼다.

## 1. 기술·출시 기준

- 엔진은 **Godot 4.7.1 + GDScript**를 사용한다.
- 출시 목표는 Steam의 Windows 데스크톱 빌드다.
- 데모 빌드는 5전째 지역 예선 결승까지 제공한다.
- 전체판은 10전째 결승까지 진행한다.
- Steamworks 연동은 현재 세로 슬라이스 범위에서 제외한다.
- 전투 규칙은 Godot 씬과 분리된 순수 GDScript 시뮬레이터로 실행한다.
- UI는 입력을 수집하고 구조화 이벤트를 재생하며 전투 결과를 직접 계산하지 않는다.

## 2. 시스템 층

| 층 | 책임 | 예시 |
|---|---|---|
| 공용 행동 | 타이밍에서 가능한 기본 선택 | 접근, 후퇴, 공격, 막기, 회피, 밀치기, 교체, 재장전, 명상 |
| 선택 묶음 | 두 행동 잠금·동시 공개·순차 해상 | 1수와 2수 |
| 무기 규칙 | 사거리, 기본 합, 일반 피해, 비용, 장전 | 권법 0, 창 2, 활 3~5 |
| 합 규칙 | 쌍방 공격 비교와 차이 피해 | 태극권 반격 수정 포함 |
| 보조장비 | 기존 행동에 붙는 제한 횟수 효과 | 암기벨트, 화살통, 방패 |
| 핵심무공 | 3·6·9성 기술과 10성 절초 | 검법·권법·창법 등 |
| 심법 | 패시브·기믹·명상 강화와 10성 진의 | 연격, 반격, 독맥 순환 |
| 회차 성장 | 성과, 수련 포인트, 제약, 보상 | 기본 5포인트, C/B/A/S |
| 대회 진행 | 데모 5전·전체판 10전의 단계와 저장 | 예선 결승, 본선, 8강, 준결승, 결승 |

## 3. 시뮬레이션 계층

| 구성 요소 | 책임 |
|---|---|
| `CombatState` | 위치, 체력, 행동력, 기력, 내공, 장전, 상태, 장비, 기믹 보관 |
| `ActionChoice` | 타이밍 하나의 공용 행동과 무공·장비 참조 보관 |
| `ActionPair` | 한쪽 전투원의 1수·2수와 잠금 상태 보관 |
| `PairLockCoordinator` | AI 사전 잠금, 플레이어 잠금, 동시 공개 보장 |
| `TimingResolver` | 타이밍 하나의 이동·방어·공격·합·상태를 해상 |
| `ClashResolver` | 양측 최종 합 수치와 차이 피해 계산 |
| `RoundResolver` | 5개 행동 묶음, 10타이밍, 라운드 종료 회복·상태 처리 |
| `CombatModifierSet` | 비용·합·피해·경감·사거리·상태·명상 수정자 제공 |
| `AiPlanner` | 공개 정보와 성향만 받아 행동 두 개를 생성·잠금 |
| `TournamentState` | 현재 경기, 단계, 데모 종료 여부, 상대 풀, 수련 횟수 보관 |

전투 이벤트는 `pair_locked`, `pair_revealed`, `move`, `guard`, `evade`, `attack`, `clash`, `hit`, `push`, `status`, `reload`, `meditate`, `resource`, `invalid_action`으로 반환한다.

## 5. 공용 수정자 계약

허용 수정자:

- 행동력·기력·내공·체력 비용
- 합 수치와 상대 합 수치 참조 비율
- 일반 피해와 합 차이 피해 경감
- 사거리와 이동량
- 막기·회피·밀치기 효율
- 상태 부여·중첩·지속
- 장전·회수·명상 회복
- 다음 행동·다음 같은 계통 행동 강화
- 2수 묶음 내 연속 사용 조건
- 발동 횟수와 최대 중첩

금지:

- 문파 전용 행동 타입
- 문파 전용 타이밍 해상 순서
- UI에서만 계산되는 전투 결과
- AI가 플레이어 비공개 행동을 읽는 기능
- 심법 3개 이상 보유
- 같은 계열의 두 번째 10성 진입

## 6. 타이밍 해상 순서

1. 양측 행동과 부착 장비 확인
2. 행동 가능 여부와 비용 판정
3. 준비·다음 행동 수정자 적용
4. 이동 동시 처리
5. 막기·회피·방패 적용
6. 이동 후 공격·밀치기 성립 확인
7. 쌍방 공격이면 `ClashResolver` 실행
8. 단독 공격이면 일반 피해 실행
9. 상태·장전·회수·명상·자원 반영
10. 구조화 이벤트와 다음 상태 반환

`ClashResolver`는 기본 합 → 무기 → 성급 기술 → 심법·기믹 → 상대 참조 반격 → 최종 합 → 차이 피해 순서의 원인 목록을 함께 반환한다.

## 7. 화산 연격의 공용 표현

- 심법 3성은 `next_same_category_cost` 수정자를 생성한다.
- 대상 계통은 `sword`이며 다음 검법 1회에 적용한다.
- 지속은 다음 검법 또는 2수 묶음 종료까지다.
- 10성 진의는 감소량, 유지 조건, 합 보너스, 최대 중첩 중 허용된 필드만 강화한다.
- 엔진에 `if faction == hwasan` 분기를 만들지 않는다.

다른 문파의 기세·반격·독맥·마화 연계도 같은 조건부 수정자와 공용 상태 계약을 사용한다.

## 8. 성장 규칙

- 시작 핵심무공과 심법은 3성이다.
- 성급 n의 상승 비용은 n포인트다.
- 승리 기본 포인트는 5다.
- C는 6~7점, B는 8~10점, A는 13~14점, S는 16점을 지급한다.
- 등급 확정 보너스와 확률 보너스는 `PerformanceGradeData`에서 계산한다.
- 핵심무공은 여러 개 보유할 수 있다.
- 심법 습득은 보유 2개에서 원자적으로 거부한다.
- 핵심무공 10성 슬롯과 심법 10성 슬롯을 별도로 저장한다.
- 깨달음은 상승 가능한 최고 성급 무공을 찾되 계열별 10성 제한을 검사한다.
- 첫 4전이 모두 A 이상이면 핵심무공 집중 투자로 최소 52점을 확보해 5전 전 절초 개방이 가능하다.
- B등급 기대값 9점×9회의 결승 전 수련은 81점이며, 핵심무공 10성+심법 8성 목표 79점과 맞는다.

## 9. 제약 규칙

- 제약은 전투 시작 전에 선택하고 `ConstraintData` 수정자를 초기 상태에 적용한다.
- 삼수양보 같은 시간 조건은 `timing_index` 조건으로 표현한다.
- 문파·무공 ID를 직접 금지하는 제약보다 공용 자원·거리·행동 조건을 우선한다.
- 승리했을 때만 추가 보상을 정산한다.

## 10. 대회 상태 규칙

- 전체판은 10전이며 8전은 8강, 9전은 준결승, 10전은 결승이다.
- 데모는 5전째 지역 예선 결승에서 종료한다.
- 1~4전 승리 뒤 수련을 진행하고 5전 시작 전에 조기 절초 개방 여부를 확정한다.
- 데모 종료 상태는 전체판의 6전 본선 진입 상태와 구분해 저장한다.
- 상대 반복 방지와 단계별 난이도는 `TournamentStageData`에서 관리한다.

## 12. 저장과 검증

- 저장 데이터는 포맷 버전, 회차, 선택 문파, 경기, 단계, 데모 종료 여부, 상대 풀, 보유 무공·심법·성급·부분 포인트, 10성 슬롯, 장비, 명망, 재도전, 제약을 보관한다.
- headless 테스트는 위치·동시 이동·2수 잠금·합·상태·수련·10성 제한·제약을 검증한다.
- 데모 5전과 전체판 10전은 같은 엔진 계약으로 각 종료 지점까지 진행되어야 한다.
- 첫 4전 A 이상 집중 투자 시 5전에서 절초가 사용 가능해야 한다.
- 새 문파와 무공은 공용 엔진 수정 없이 데이터와 허용 수정자로 추가할 수 있어야 한다.
