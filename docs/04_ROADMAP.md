## 6. 다음 작업 — 코어 검증 슬라이스

### 6.1 결정적 복기

상세 로그보다 먼저 다음을 보여 주는 최소 표현을 검증한다.

```text
내 예상
→ 상대의 실제 계획
→ 승부를 가른 거리·합·대응·중단
→ 다음에 바꿀 수 있는 선택
```

새 판정 규칙은 추가하지 않는다.

### 6.2 읽을 수 있는 라이벌 성향

현행 비치팅 AI 입력 경계를 유지하면서 다음을 실험한다.

- 라이벌별 2~3개 성향.
- 같은 공개 상태에서 복수의 합리적 후보.
- seed 기반 재현 가능성.
- 위험 선호·거리 선호·자원 선호·대응 성향.
- 플레이어가 관찰할 수 있는 행동 이력과 단서.
- 미확정 슬롯·대상·절초 예약 입력 금지.

완전 랜덤 AI와 단일 고정 스크립트 모두 피한다.

### 6.3 수치 관찰

- 행동 선택 분포.
- 합 발생과 승패 기여.
- 절초 선택·성공·실패.
- 기세 선점과 눈덩이.
- 첫 계획·재도전 계획의 변화.

수치 공식은 관찰 전에 변경하지 않는다.

## 7. STEP 14 — 실제 사용자 POC 플레이테스트

### 표본

- 신규 플레이어 5명 발견형 테스트.
- 전술·격투 관심은 있으나 현재 빌드 사전 지식이 없는 참가자 우선.
- 동일 빌드·동일 기본 안내.
- 안내 결투 1회 뒤 자율 재도전 1회.

### 관찰

- 첫 계획 확정 시간과 수정 횟수.
- 이동 목적지·공격 방향·준비/실행 혼동.
- 3/3/4의 묶음 차이 설명.
- 합·방어·필중·중단의 결정적 원인 설명.
- 상대 성향 하나 이상 발견.
- 재도전에서 실제 계획 변경.
- 자발적 재도전 또는 구체적 다음 전략.
- 색·모션·음향·입력·보조기기 장벽.

### 발견형 통과 신호

- [ ] 5명 중 4명 이상이 치명적 차단 없이 한 판 완료.
- [ ] 5명 중 4명 이상이 3/3/4와 결정적 원인 하나를 설명.
- [ ] 5명 중 3명 이상이 안내 없이 상대 성향 하나를 발견.
- [ ] 5명 중 3명 이상이 재도전에서 계획을 실질적으로 변경.
- [ ] 5명 중 3명 이상이 자발적으로 다시 하거나 구체적 다음 수를 제시.
- [ ] 핵심 정보가 색·모션·음향 하나에만 의존한 참가자가 없음.

실패하면 성공 기준을 바꾸지 않고 `REPEAT_POC`한다.

## 8. T1 — 최소 세로 슬라이스

STEP 14와 코어 검증 슬라이스를 통과한 뒤 다음 범위만 검토한다.

- 플레이 스타일 2개.
- 성향이 다른 상대 3명.
- 전투 3회.
- 전투 전 상대 정보.
- 수평적 보상 선택 1회 이상.
- 최종 라이벌 1회.

목적은 `정보 → 가설 → 계획 → 전투 → 복기 → 보상 → 다음 라이벌`이 반복 동기를 만드는지 확인하는 것이다.

### T1에서 금지

- 12세력 전체 제작.
- 무공·심법 1~10성 전체 성장.
- 10전 전체 대회.
- 대형 시장·정탐·세력 관계·연구 트리.
- 원시 능력치 누적으로 수읽기 실패 무력화.

## 9. T2와 전체판 가설

## 11. 중단·축소 조건

- 상대 읽기보다 고정 AI 암기가 우세하다.
- 상세 로그를 봐도 실패 이유를 설명하지 못한다.
- 3/3/4가 의미 있는 결단이 아니라 입력 대기열로 느껴진다.
- 합·절초·기세 하나가 대부분의 선택을 지배한다.
- 코어보다 콘텐츠 수·성장 수치·대회 분량이 우선된다.
- 새 상대·무공마다 독립 판정 절차가 필요하다.
- 접근성·성능·제작 비용이 목표 범위를 초과한다.

해당 조건이 나타나면 다음 단계로 넘어가지 않고 범위·규칙·정보 표현을 축소·재조정한다.
