## 1. 로드맵 원칙

- 현재 단계에서 가장 위험한 가설을 먼저 검증한다.
- 문서·정적 검사·Godot 런타임·사용자 플레이를 서로 다른 증거로 기록한다.
- T0 전투 코어가 통과하기 전 문파·무공·성장 콘텐츠를 대량 제작하지 않는다.
- 각 단계는 앞 단계 결과에 따라 축소·수정·중단할 수 있다.
- 사용자 확정값과 실제 구현이 충돌하면 책임 원본과 소비자를 같은 작업에서 정렬한다.

## 5. STEP 11 — 피격 중단·집중·강건

목적: 상대의 수를 먼저 맞히는 것이 후속 계획에 실제 영향을 주게 한다.

### 규칙

- 실제 피해를 받으면 이후 미실행 행동을 기본 중단한다.
- 같은 단계에서 이미 성립한 공격의 동시 피해는 유지한다.
- `[집중]`은 일반 피해로 인한 중단을 방지한다.
- `[강건]`은 일반 피해로 인한 중단을 방지한다.
- 사망과 강제 제어는 집중·강건보다 우선한다.

### 구현

- 중단 가능 행동 데이터
- 준비·실행·중단 상태
- 취소된 다중 슬롯 전체 처리
- 집중·강건 태그와 예외
- 로그·슬롯 취소 표시
- 정상·동시 공격·사망·경계 테스트

### 완료 기준

- 중단 시점과 대상 행동이 결정론적이다.
- 이미 성립한 동시 공격은 보존된다.
- 플레이어가 어떤 행동이 왜 취소됐는지 설명한다.

## 6. STEP 12 — 성향 기반 단순 AI

목적: 비공개 정보를 보지 않으면서 읽을 수 있는 상대를 만든다.

### 최소 AI 입력

- 공개 위치·거리
- 체력·기력·내력·절초 기세
- 공개 상태
- 이전 수·묶음의 행동과 결과
- 자신의 행동 목록과 성향

### 최소 성향

- 접근 선호
- 거리 유지
- 공격 선호
- 대응 선호
- 자원 부족 시 명상
- 강공 준비 빈도
- 제한된 실수 확률

### 완료 기준

- 플레이어 현재 배치를 읽지 않는다.
- 완전 무작위가 아니며 반복 성향이 관찰된다.
- 같은 시드와 입력에서 선택을 재현한다.
- AI 행동도 플레이어와 같은 `BundlePlan` 계약을 사용한다.

## 7. STEP 13 — 전투 종료·재시작

### 구현

- 체력 0 승패
- 동시 0 처리
- 종료 후 입력 잠금
- 마지막 수 요약
- 재시작
- 전장·자원·라운드·슬롯·대상·로그 초기화

### 완료 기준

- 승패 뒤 추가 행동이 실행되지 않는다.
- 재시작이 항상 같은 fixture로 돌아간다.
- 종료·재시작을 반복해 상태가 누적되지 않는다.

## 8. STEP 14 — POC 플레이테스트

### 핵심 질문

- 플레이어가 `3수 → 3수 → 4수`를 이해하는가?
- 이동 목적지와 공격 방향을 혼동하지 않는가?
- 대응·중단·자원 판단이 의미 있다고 느끼는가?
- 실패 이유를 설명할 수 있는가?
- 상대의 성향을 발견하는가?
- 다시 플레이하고 싶은가?

### 판정

- `KEEP`: 유지
- `AMPLIFY`: 더 강조
- `CHANGE`: 수정
- `REMOVE`: 제거
- `DEFER`: 이후 단계
- `RETEST`: 추가 실험

### T1 진입 조건

- STEP 0~13 실제 런타임 통과
- 치명적 입력·판정 오류 없음
- 플레이어가 실패 이유를 대체로 설명
- 최소한 일부 상대 성향을 발견
- 재도전 의사 또는 계획 수정 의도 관찰

## 9. T1 — 최소 세로 슬라이스

T0 통과 뒤 제작한다.

- 플레이 스타일 2개
- 상대 3명 내외
- 전투 3회 내외
- 전투 전 상대 정보
- 전투 후 보상 선택
- 무공 또는 능력 강화
- 최종 강적

목적은 5전 전체를 제작하기 전에 전투→정보→보상→성장의 연결을 검증하는 것이다.
