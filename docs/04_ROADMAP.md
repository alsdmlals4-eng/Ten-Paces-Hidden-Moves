목적: 문서 계획과 Godot 실제 파일의 차이를 확인하고 구현 전에 안전한 기준선을 만든다.

- 구현 사실, 승인 계획, 미확정, 보류와 백업을 구분한다.
- 수정·신규 파일, 데이터 ID, 저장 포맷, 테스트 명령과 중단 기준을 Plan Mode로 제출한다.
- 사용자 승인 전에는 코드·씬·데이터를 수정하지 않는다.

### 종료 기준

## 단계 0B — 미확정 설계 고정

목적: 구현 중 임시 수치와 임시 문구가 기준으로 굳는 것을 막는다.

우선 확정할 항목:

1. 성과 C/B/A/S의 공용 지표, 가중치와 임계값.
2. 상대 강도 5단계와 A/B/행운의 경기별 출현표.
3. 이명·수식언·풍문·전적·정탐의 의미 키와 첫 문장 사전.
4. 균형형·핵심무공 올인형·심법 연계형·다무공형 투자 알고리즘.
5. 제약 후보 수, 선택 상한, 위험도와 추가 보상.
6. 행동력·기력·내공의 카드 비용과 라운드 회복량.
7. 1수 결과로 체력 0이 됐을 때 2수 처리.
8. 결투 라운드 상한과 제한 종료 승패 판정.
9. 신규 습득 무공의 시작 성급과 습득 비용.
10. 결승 뒤 수련 포인트의 명망·점수·보상 전환 방식.

### 종료 기준

- [ ] 구현 필수 값과 플레이테스트 후 조정 값을 구분한다.
- [ ] 임시 UI·연출 숫자가 기준 문서보다 우선하지 않는다.
- [ ] 미확정 항목은 구현 Plan에 임의 값으로 확정되지 않는다.

## 단계 1A — T0 전투 규칙 PoC

목적: 10칸 전장·10타이밍·2수 동시 선택·합의 핵심 판단과 AI 공정성을 검증한다.

### 범위

- 10칸 위치, 거리, 중첩, 교차와 자리 교환.
- 2수 비공개 잠금, 동시 공개, 1수→2수 순차 해상.
- AI의 플레이어 선택 전 사전 잠금.
- 행동력·기력·내공과 라운드 종료 회복.
- 접근, 후퇴, 공격, 막기, 회피, 밀치기, 교체, 재장전, 명상.
- 일반 공격, 공격 대 방어, 합과 차이 피해.
- 무효 행동 원인과 구조화 결과 로그.

### 승인 기준

- [ ] 양측 행동 두 개가 잠금 전에 유출되지 않는다.
- [ ] 1수 결과 뒤 2수를 변경할 수 없다.
- [ ] 이동 후 거리로 공격 성립 여부를 판정한다.
- [ ] 합과 일반 공격의 결과·로그가 구분된다.
- [ ] 같은 시드와 입력에서 결정론적 결과를 재현한다.

## 단계 1B — T0 전투 연출 PoC

목적: 핵심 규칙이 실제 화면에서 빠르고 공정하게 읽히는지 검증한다.

### 범위

- 전장·거리·자원·타이밍·1수·2수 슬롯.
- 카드 상시 정보, 호버 팝업과 클릭 상세.
- AI 잠금 표시와 양측 2수 동시 공개.
- 이동, 일반 공격, 무효 행동, 합, 피해, 자원 변화.
- 합의 구성 원인과 차이 피해 로그.
- 화면 흔들림·점멸 감소·자산 누락 폴백.

### 승인 기준

- [ ] 카드만 보고 3초 안에 문파/공용, 이름, 종류, 사거리와 주요 비용을 말할 수 있다.
- [ ] 플레이어가 한 행동 묶음의 성공·실패 이유를 설명한다.
- [ ] 합과 일반 공격을 혼동하지 않는다.
- [ ] 연출을 줄여도 같은 결과와 저장 상태가 유지된다.

## 단계 2A — 예선 결승 데모 시스템

목적: 5전 데모에서 수련·행운·상대 성장·제약·절초를 전투와 연결한다.

### 데모 범위

- 1~2전: 내부전.
- 3~4전: 예선.
- 5전: 예선 결승·데모 최종전.
- 시작 핵심무공과 심법은 각각 3성.
- 3전부터 공용·외문 무공 또는 보조장비 보상 후보 제공.
- 승리 기본 5 + 등급 보너스(C1/B3/A5/S7) + 비공개 행운 앞면 0~3.
- 첫 4전 A 4회 약 50%, A3+S1 약 60%, A3+B1 약 30%의 절초 개방 목표.
- 상대 등급·행운·수련 스타일은 회차 생성 시 고정.
- 상대 정보는 개발용 태그가 아니라 이명·풍문·전적·정탐으로 표시.
- 상대 정보를 보고 삼수양보 등 제약을 선택.

### 승인 기준

- [ ] 시작 문파를 선택하고 5전 예선 결승까지 진행할 수 있다.
- [ ] 3~5전이 하나의 예선 구간으로 표시된다.
- [ ] 성과 등급과 행운 결과가 `A++` 형식과 실제 포인트 정산에서 일치한다.
- [ ] 첫 4전 A 4회 핵심무공 올인 시 절초 개방률이 목표 범위에 가깝다.
- [ ] 절초 없이도 예선 결승을 시작하고 완료할 수 있다.
- [ ] 상대 생성 결과와 표현 키가 재도전·저장에서 유지된다.

## 단계 2B — 예선 결승 데모 연출

목적: 상대 읽기→제약→전투→성과·행운→수련→절초→데모 종료의 감정 곡선을 완성한다.

### 범위

- 경기·단계와 상대 이명·풍문·전적·정탐.
- 제약 선택 전 불이익·보상 미리보기.
- 기본·등급·행운·제약 보상의 분리 정산.
- 수련 분배, 성급 상승과 3·6·9·10성 해금.
- 4전 종료 절초 신규 해금과 5전 준비.
- 승리, 패배, 재도전, 예선 우승과 데모 종료.
- 절초 보유·미보유 양쪽 데모 경로.

### 승인 기준

- [ ] 플레이어가 상대 문구를 근거로 제약 선택 이유를 설명한다.
- [ ] 정탐은 수치표를 그대로 노출하지 않고 표현의 구체성을 높인다.
- [ ] 절초 해금과 사용이 하이라이트지만 미보유 경로를 막지 않는다.
- [ ] 데모 종료 뒤 전체판 6~10전 목표를 이해한다.

## 단계 3 — 시작 6세력 수직 슬라이스

목적: 같은 공용 전투 엔진에서 문파별 핵심무공·심법·진의가 다른 판단을 만드는지 검증한다.

### 범위

- 화산, 개방, 당문, 소림, 무당, 마교.
- 문파 전용 핵심무공·심법과 각 무공 종류의 공용 무공 2종.
- 3·6·9성 기술, 10성 절초와 진의.
- 공용 비용·합·피해 경감·사거리·상태·명상·장전 수정자.
- 문파 기믹 의미 키, 아이콘, VFX·음향의 T1 표현.

### 승인 기준

- [ ] 시작 6세력 각각으로 데모를 완주한다.
- [ ] 공용 엔진에 문파 전용 해상 순서를 추가하지 않는다.
- [ ] 각 문파의 핵심 판단을 플레이어가 설명한다.
- [ ] 자산이 없어도 공용 폴백으로 기능이 유지된다.

## 단계 4 — 밸런스와 콘텐츠 밀도

데모 검증 후 다음 지표로 수치·문구·보상 풀을 조정한다.

- 문파별 승률과 합 발생률.
- 절초 개방 경기와 데모 절초 체험률.
- C/B/A/S 비율과 행운 `+` 분포.
- 상대 강도 체감과 이명·풍문 해석 정확도.
- 수련 스타일별 승률과 제약 선택률.
- 재도전율과 보상 선택 편중.
- 일반 공격·합·절초 연출의 반복 피로.
- 결과 화면과 전환에 걸리는 대기 시간.
- 최소 해상도와 긴 문구의 가독성.

필요할 때만 공용 무공, 이명·풍문 문구, 장비, 상대 프로필과 전용 연출을 늘린다.

## 단계 5 — 12세력 10전 전체판 수직 슬라이스

목적: 시작 문파 선택부터 내부전·예선·본선·8강·준결승·결승까지의 10전 회차를 같은 데이터·연출 모듈로 확장한다.

- 6~7전 본선, 8전 8강, 9전 준결승, 10전 결승.
- 12세력 상대 풀과 반복 방지.
- 전체판 수련·보상·정탐·재도전·저장.
- 결승 단계의 연출 밀도와 회차 종료.

### 승인 기준

- [ ] 추가 6세력이 적 또는 습득 무공으로 실제 전투에서 작동한다.
- [ ] 일반적인 전체판 진행에서 핵심무공 10성·심법 8성 전후가 가능하다.
- [ ] `10칸·10타이밍·10전` 구조가 UI·저장·테스트에서 동일하다.
- [ ] 데모용 화면과 코드를 복제하지 않고 데이터와 모듈로 10전까지 확장한다.
