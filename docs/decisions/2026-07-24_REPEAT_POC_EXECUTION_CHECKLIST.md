# REPEAT_POC 실행 체크리스트

## 시작 전

- [ ] `AGENTS.md`, Active Context, Documentation Map을 읽었다.
- [ ] `plans/CODEX_GOAL_REPEAT_POC.md`를 읽었다.
- [ ] 현재 branch·HEAD·원격 추적 상태를 확인했다.
- [ ] 사용자 기존 변경과 작업 범위를 분리했다.
- [ ] 계획의 현재 Task만 `in_progress`로 표시했다.

## 각 Task

- [ ] Red 반례를 먼저 작성했다.
- [ ] Red가 예상 이유로 실패했다.
- [ ] 최소 Green만 구현했다.
- [ ] 관련 기존 회귀를 실행했다.
- [ ] Refactor가 동작·계약을 바꾸지 않았다.
- [ ] 정본·Registry·freshness 소비자를 확인했다.
- [ ] 독립 커밋을 만들었다.
- [ ] PR 본문에 실제 실행 결과를 기록했다.

## 전체 Goal

- [ ] PR-A0~A3 전체 기술 검증 통과.
- [ ] 고정 build SHA 기록.
- [ ] STEP 14 프로토콜 상태를 `DO_NOT_RUN`에서 실행 가능으로 전환.
- [ ] 참가자 5명 결과 기록.
- [ ] KEEP/AMPLIFY/CHANGE/REMOVE/DEFER/RETEST 판정.
- [ ] T1_GREENLIGHT_REVIEW 또는 REPEAT_POC 결정.
- [ ] 남은 미검증을 완료로 표시하지 않았다.
