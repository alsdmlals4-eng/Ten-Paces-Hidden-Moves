# 12세력 대회 전투 UI·아트 명세

> 목적: 플레이어가 소속·기능·사거리·비용·3수/3수/4수 행동 배치·합·수련을 빠르게 읽고, 상대의 무림식 정보로 제약 위험을 판단하게 한다.

이 파일은 **화면 영역, 카드 정보 배치, 상호작용 상태와 카드 아트**의 현재 기준이다. 화면 간 시선·감정 흐름, 공개·해상·합·성장·결말 연출, 카메라·VFX·오디오와 접근성 폴백은 `10_COMBAT_PRESENTATION_PLAN.md`가 책임진다. `docs/[백업]/`과 `docs/[보류]/`는 구현 기준으로 사용하지 않는다.

## 1. 회차·대진 화면

| 영역 | 표시 내용 |
|---|---|
| 대진 진행 | 내부전 / 예선 / 예선 결승 / 본선 / 8강 / 준결승 / 결승, 현재 경기와 10전 진행도 |
| 데모 진행 | 1~2전 내부전, 3~4전 예선, 5전 예선 결승 |
| 다음 상대 | 이름·문파·이명·풍문·최근 전적·알려진 무기·선호 거리·정탐 정보 |
| 제약 선택 | 삼수양보 등 제약명, 불이익, 위험도, 승리 보상 |
| 보상 선택 | 무공서, 수련 보조, 장비, 회복, 정탐 효과 |
| 수련 패널 | 핵심무공·심법 성급, 다음 필요 포인트, 3·6·9·10성 해금 |

- `전장 10칸 / 라운드 행동 타이밍 10개 / 전체 대회 10전`을 서로 다른 정보로 표시한다.
- 개발용 상대 태그 `[강적] [A++] [올인형]`은 화면에 직접 노출하지 않는다.

## 2. 상대 정보 표현

| 표현 | 암시 대상 |
|---|---|
| 이명·수식언 | 종합 강도와 대표 전법 |
| 풍문 | 행운과 최근 성장세 |
| 전적 | 실제 강도와 안정성 |
| 무공 평가 문장 | 균형형·올인형·심법 연계형·다무공형 |
| 정탐 | 위 정보를 더 구체적으로 표현 |

- 같은 내부 의미에는 같은 어휘 계열을 반복 사용한다.
- 정탐은 개발용 수치보다 무림식 표현으로 구체화한다.
- 표시가 실제 상대 데이터와 모순되어서는 안 된다.
- 상대 정보는 제약 선택 전에 제공한다.

| 영역 | 표시 내용 |
|---|---|
| 상단 좌측 | 플레이어 초상·체력·기력·내력·상태 |
| 상단 중앙 좌측 | 플레이어 절초 기세 |
| 상단 중앙 | 라운드·행동 묶음 `1/3~3/3`·현재 수·해상 순서 |
| 상단 중앙 우측 | 상대 절초 기세 |
| 상단 우측 | 상대 초상·체력·기력·내력·공개 상태 |
| 10칸 전장 | 1~10번 칸, 플레이어 3 / 상대 8 POC 배치, 현재 거리 |
| 행동 슬롯 | 첫 묶음 3수, 둘째 묶음 3수, 셋째 묶음 4수 |
| 보유 기술 | 기초 행동 7종과 습득 무공. 덱·손패가 아님 |
| 카드 상세 | 대상, 사거리, 피해, 조건, 효과, 태그, 행동 슬롯·기력·내력 |
| 전투 로그 | 대응 → 속공 → 이동 → 일반 공격 순서와 결과 근거 |
| 진행 버튼 | 현재 묶음의 배치 검증·확정·해상 시작 |

- 좌우 세로 캐릭터 패널을 사용하지 않고 상단 HUD로 올려 전투판 너비를 확보한다.
- 덱·손패·남은 카드·버린 카드 수를 표시하지 않는다.
- AI 행동은 플레이어의 비공개 선택을 읽지 않는다.

## 4. 카드 템플릿

### 분위기

- 먹색·흑청색 중심의 어두운 무협 배경.
- 오래된 화선지 같은 미색 카드 본체.
- 얇은 금속성 금장과 전통 문양.
- 반실사 무협 인물과 수묵화의 중간 톤.
- 카드마다 데이터가 달라도 골격과 정보 위치는 바꾸지 않는다.

### 정보 계층

| 위치 | 표시 정보 |
|---|---|
| 좌측 상단 | 소속 배지: `[기초]` 또는 문파·세력 배지 |
| 상단 중앙 | 사거리: `-`, `0`, `1`, `2`, `2~3` 등 |
| 우측 상단 | 기능 배지: 이동·공격·대응·회복·강화 |
| 상단 제목부 | 카드 이름 |
| 중앙 | 카드 원화 |
| 하단 | 행동 슬롯·기력·내력 |

- `[강호낭인]` 소속은 별도 배지로 사용하지 않고 기초 행동은 `[기초]`로 표시한다.
- 검법·보법·심법·속공·집중·강건 등은 기능 배지가 아니라 상세 태그다.
- 행동력은 사용하지 않는다.
- 사용하지 않는 기력·내력 비용도 `0`으로 표시한다.
- 카드 본체와 상세 패널에서 `막기 경감`을 공통 항목으로 사용하지 않는다.

기본 읽기 순서는 `소속 → 사거리 → 기능 → 카드명 → 원화 → 행동 슬롯·기력·내력`이다.

## 5. 실제 UI 컴포넌트 계약

- `CardView`는 카드 앞면만 표현한다.
- `CardDetailPanel`은 클릭된 카드의 상세 데이터를 표현한다.
- `CardCatalog`는 JSON을 읽고 필수·금지 필드를 검증한다.
- 원화·소속 배지·기능 배지·비용 아이콘은 독립 자산이다.
- UI 컴포넌트는 효과와 수치를 계산하지 않는다.
- 향후 사용자 편집 기능은 같은 JSON/Resource 필드를 수정하는 별도 단계다.

## 8. 3수·3수·4수 선택과 해상 표시

- 라운드는 첫 묶음 3수, 둘째 묶음 3수, 셋째 묶음 4수다.
- 행동 슬롯 1인 카드는 한 수를 점유한다.
- 행동 슬롯 2인 카드는 배치 시작 위치부터 인접한 두 수를 점유하며 앞 수는 준비, 마지막 수는 실행이다.
- 행동 슬롯 3인 카드는 준비·전개·실행의 세 수를 점유한다.
- 카드 본체에는 특정 `2수→3수`를 고정 표기하지 않고 행동 슬롯 수만 표시한다.
- 방향·대상·비용을 선택한 뒤 `[진행]`으로 묶음을 확정한다.
- 확정 뒤 해당 묶음은 변경할 수 없다.
- 동타이밍 기본 해상은 `대응 → 속공 → 이동 → 일반 공격`이다.
- 실제 피해를 받으면 아직 실행되지 않은 이동·공격 등 후속 행동이 중단된다.
- `[집중]`과 `[강건]`은 중단 방지 계약이다.
- 합은 같은 공격 단계의 유효한 쌍방 공격에만 발생한다.

## 9. 수련 결과 화면

1. 승리 기본 `+5`.
2. 성과 등급 보너스.
3. 비공개 행운 결과.
4. 제약 승리 보상.
5. 최종 수련 포인트.
6. 무공별 포인트 분배.

- 개별 행운 다이스 면은 보여주지 않는다.
- `+` 개수와 실제 행운 포인트가 일치해야 한다.
- 다음 성급 비용과 해금을 미리 보여준다.
- 분배 확정 전 되돌리기를 제공한다.
- 절초가 없어도 예선 결승 진입이 가능함을 분명히 한다.

## 10. 제약 선택 화면

- 상대 이명·풍문·전적·정탐 정보를 제약 카드보다 먼저 읽을 수 있게 한다.
- 제약명은 `삼수양보`, `단전봉맥`, `기혈역행`, `일장지양`처럼 무협식으로 표시한다.
- 아래에 현대적인 규칙 설명, 위험도, 승리 보상을 함께 표시한다.
- 제약을 선택하면 시작 자원·거리·행동 제한 변화를 미리 보여준다.
- 문파 핵심 기믹 전체를 막는 제약은 생성하지 않는다.

## 12. 완료 기준

- 카드만 보고 3초 안에 소속, 기능, 사거리, 행동 슬롯·기력·내력을 말할 수 있다.
- `[기초]`와 문파 배지, 기능 배지 5종을 혼동하지 않는다.
- 3수·3수·4수와 다중 슬롯 점유를 혼동하지 않는다.
- 카드 클릭 시 상세 패널 데이터가 카드 정의와 일치한다.
- 덱·손패·행동력·공통 `막기 경감`이 노출되지 않는다.
- 합과 일반 공격, 속공과 일반 공격의 차이를 로그로 설명할 수 있다.
- 최소 해상도·긴 문구·자산 누락·모션 감소 상태에서도 같은 정보가 전달된다.
