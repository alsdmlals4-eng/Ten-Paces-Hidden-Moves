# 12세력 대회 전투 UI·아트 명세

> 목적: 플레이어가 문파·무공·사거리·비용·2수 선택·합·수련을 빠르게 읽고, 상대의 무림식 정보로 제약 위험을 판단하게 한다.

이 파일은 **화면 영역, 카드 정보 배치, 상호작용 상태와 카드 아트**의 현재 기준이다. 화면 간 시선·감정 흐름, 공개·해상·합·성장·결말 연출, 카메라·VFX·오디오와 접근성 폴백은 `10_COMBAT_PRESENTATION_PLAN.md`가 책임진다. `docs/[백업]/`과 `docs/[보류]/`는 구현 기준으로 사용하지 않는다.

## 1. 회차·대진 화면

| 영역 | 표시 내용 |
|---|---|
| 대진 진행 | 내부전 / 예선 / 예선 결승 / 본선 / 8강 / 준결승 / 결승, 현재 경기와 10전 진행도 |
| 데모 진행 | 1~2전 내부전, 3~4전 예선, 5전 예선 결승 |
| 다음 상대 | 이름·문파·이명·풍문·최근 전적·알려진 무기·선호 거리·정탐 정보 |
| 제약 선택 | 삼수양보 등 제약명, 불이익, 위험도, 승리 보상 |
| 보상 선택 | 무공서, 수련 보조, 장비, 회복, 정탐 효과 |
| 수련 패널 | 핵심무공·심법 성급, 다음 필요 포인트, 3·6·9·10성 해금 |

- `전장 10칸 / 라운드 행동 타이밍 10개 / 전체 대회 10전`을 서로 다른 정보로 표시한다.
- 개발용 상대 태그 `[강적] [A++] [올인형]`은 화면에 직접 노출하지 않는다.
- 상대 정보가 제약 카드보다 먼저 읽히는 시선 순서는 `10_COMBAT_PRESENTATION_PLAN.md`를 따른다.

## 2. 상대 정보 표현

### 정보 층

| 표현 | 암시 대상 |
|---|---|
| 이명·수식언 | 종합 강도와 대표 전법 |
| 풍문 | 행운과 최근 성장세 |
| 전적 | 실제 강도와 안정성 |
| 무공 평가 문장 | 균형형·올인형·심법 연계형·다무공형 |
| 정탐 | 위 정보를 더 구체적으로 표현 |

예시:

```text
[매화일섬] 낙유진
화산파 후기지수

“검 한 자루만을 벼려 예선의 이름난 고수들을 연달아 꺾었다.”
“최근 두 차례의 비무에서 기연을 얻은 뒤 검세가 크게 달라졌다고 한다.”
```

- 같은 내부 의미에는 같은 어휘 계열을 반복 사용한다.
- 정탐은 `핵심무공 9성` 같은 개발용 수치보다 `절초의 문턱에 이르렀다`처럼 무림식으로 구체화한다.
- 표시가 실제 상대 데이터와 모순되어서는 안 된다.
- 상대 정보는 제약 선택 전에 제공한다.
- 문구 키와 내부 상대 프로필의 매핑은 `09_COMBAT_SYSTEM_ARCHITECTURE.md`를 따른다.

## 3. 전투 화면

| 영역 | 표시 내용 |
|---|---|
| 적 상태 | 체력·행동력·기력·내공, 무기, 장전, 방패, 상태, 전투 중 드러난 기믹 |
| 10칸 전장 | 1~10번 칸, A=4 / B=6 초기 배치, 현재 거리 |
| 플레이어 상태 | 체력·행동력·기력·내공, 무기·보조장비, 연격·반격 등 상태 |
| 타이밍 진행 | 1~10 진행도와 현재 2수 묶음 |
| 첫 번째 행동 | 현재 묶음의 1수 |
| 두 번째 행동 | 현재 묶음의 2수, 첫 결과 뒤 변경 불가 |
| 무공 카드 | 문파·공용, 종류, 사거리, 비용, 성급, 사용 불가 이유 |
| 상세 효과 | 대상, 사거리, 합, 피해, 효과, 비용, 조건, 성급 해금 |
| 결과 로그 | 이동 → 방어 → 공격 성립 → 합/일반 공격 → 피해·상태 → 회복 |

AI의 두 행동이 플레이어 선택 전에 잠겼음을 잠금 아이콘과 설명으로 표시한다. 선택 완료 전에는 AI 행동을 공개하지 않는다.

## 4. 무공 카드 아트

### 분위기

- 먹색·흑청색 중심의 어두운 무협 배경.
- 오래된 화선지 같은 미색 카드 본체.
- 얇은 금속성 금장과 전통 문양.
- 수묵화와 동양 무협 일러스트의 중간 톤.
- 성급이 올라가도 카드 골격과 정보 위치를 바꾸지 않는다.

### 정보 계층

| 위치 | 표시 정보 |
|---|---|
| 좌측 상단 | 문파 배지 또는 `공용` 배지 |
| 상단 중앙 | 사거리 |
| 우측 상단 | 권법·검법·심법·창법 등 종류 배지 |
| 상단 제목부 | 무공 이름 |
| 중앙 | 무공 삽화 |
| 하단 | 타이밍·행동력·기력·내공 비용 |

기본 읽기 순서는 `문파/공용 → 무공명 → 종류 → 사거리 → 비용 → 효과`다.

### 사거리

- `0`: 같은 칸.
- `1`: 인접 칸.
- `2~4`: 최소·최대 사거리.
- `자신`: 심법·명상·호신 효과.
- `전체`: 전장 전체.
- `-`: 거리 개념 없음.

### 비용

카드 하단은 타이밍 → 행동력 → 기력 → 내공 순서다. 사용하지 않는 비용도 `0`으로 표시한다.

## 5. 카드 상태

| 상태 | 표현 |
|---|---|
| 마우스 오버 | 작은 상세 팝업 |
| 선택 | 테두리 강조, 상세 패널 고정 |
| 1수·2수 배치 | 해당 행동 번호 표시 |
| 잠금 | 자물쇠와 수정 불가 표시 |
| 사용 불가 | 어둡게 표시하고 부족 자원·사거리·장전 이유 제공 |
| 합 예상 | `합 가능`, 상대 행동 공개 전 수치 비공개 |
| 실행 중·완료 | 현재 카드 강조와 결과 요약 |

## 6. 상세 효과 패널

표시 순서:

1. 무공 이름과 성급.
2. 문파 또는 공용.
3. 대상과 사거리.
4. 합 수치와 일반 피해.
5. 핵심 효과와 비용.
6. 사용 조건.
7. 3·6·9성 기술과 10성 절초·진의.
8. 관련 심법·기믹 수정.

카드의 상세 효과는 마우스 오버 시 작은 팝업으로 읽을 수 있고, 클릭 시 상세 패널에 고정한다.

정보 역할:

- 카드 본체: 즉시 선택에 필요한 고정 정보.
- 호버 팝업: 이번 행동에서 판단할 핵심 효과와 사용 불가 이유.
- 클릭 상세: 성급 해금, 계산 구성, 심법·기믹과 장기 비교.

같은 전문을 세 영역에 반복하지 않는다.

## 7. 2수 선택과 합 표시

- 첫 번째와 두 번째 행동을 모두 선택해야 잠글 수 있다.
- 둘째 행동의 예상 사거리와 자원은 첫 행동이 정상 실행된다는 가정으로 미리보기 한다.
- 첫 행동 실패 가능성이 있으면 둘째 행동에 조건부 경고를 표시한다.
- 양측 잠금 후 행동 두 개를 한꺼번에 공개한다.
- 1수 해상 뒤 2수를 재생하며 중간 변경은 불가능하다.
- 합은 짧은 공방 연출 뒤 최종 결과를 표시한다.
- 로그는 `기본 합 → 무기 → 성급 → 심법·기믹 → 반격 → 최종 합 → 차이 피해` 순서다.
- 동률이면 `합 무승부 / 체력 피해 없음`을 표시한다.
- 실제 공개·해상 타이밍과 접근성 폴백은 `10_COMBAT_PRESENTATION_PLAN.md`를 따른다.

## 8. 수련 결과 화면

표시 순서:

1. 승리 기본 `+5`.
2. 성과 등급 보너스: C `+1`, B `+3`, A `+5`, S `+7`.
3. 비공개 행운 결과: 앞면 수 `0~3`.
4. 제약 승리 보상.
5. 최종 수련 포인트.
6. 무공별 포인트 분배.

결과 표기 예시:

```text
성과: B++
승리 기본       +5
B등급 보너스    +3
행운 보너스     +2
최종 수련       10
```

- 개별 행운 다이스 면은 보여주지 않는다.
- `+` 개수와 실제 행운 포인트가 일치해야 한다.
- 다음 성급 비용과 3·6·9·10성 해금을 미리 보여준다.
- 분배 확정 전 되돌리기를 제공한다.

### 데모 절초 표시

- 5전 시작 전까지 남은 수련 횟수와 핵심무공 10성까지 남은 포인트를 표시한다.
- 절초 개방 확률 자체를 숫자로 보장 표시하지 않는다.
- 4전 종료 후 10성이 되면 5전 전투 준비 화면에서 절초 신규 해금을 강조한다.
- 절초가 없어도 예선 결승 진입이 가능함을 분명히 한다.

### 제한

- 핵심무공 10성 슬롯 1개.
- 심법 10성 슬롯 1개.
- 심법 보유 슬롯 2개.

## 9. 제약 선택 화면

- 상대 이명·풍문·전적·정탐 정보를 제약 카드보다 먼저 읽을 수 있게 한다.
- 제약명은 `삼수양보`, `단전봉맥`, `기혈역행`, `일장지양`처럼 무협식으로 표시한다.
- 아래에 현대적인 규칙 설명, 위험도, 승리 보상을 함께 표시한다.
- 제약을 선택하면 시작 자원·거리·행동 제한의 변화를 미리 보여준다.
- 문파 핵심 기믹 전체를 막는 제약은 생성하지 않는다.

## 10. 구현 전 UI 산출물

- 대진·상대 정보·제약 화면의 저충실도 와이어프레임.
- 전투 화면의 전장·상태·타이밍·2수 슬롯·카드·로그 배치.
- 카드 기본·호버·클릭·잠금·사용 불가 상태 시안.
- 성과·행운·수련·절초 해금 결과 화면 흐름.
- 최소 해상도와 최대 문구 길이 레이아웃.
- 키보드·마우스 포커스 순서와 모션 감소 상태.

## 11. 완료 기준

- 카드만 보고 3초 안에 문파/공용, 무공명, 종류, 사거리, 주요 비용을 말할 수 있다.
- 첫 수·둘째 수·잠금·현재 타이밍을 혼동하지 않는다.
- 합과 일반 공격의 차이를 로그와 연출로 설명할 수 있다.
- 기본·등급·행운·제약 보상이 분리되어 보인다.
- `A++` 표기와 실제 수련 정산이 일치한다.
- 상대의 개발용 태그가 노출되지 않는다.
- 이명·풍문·정탐이 실제 상대 성장과 모순되지 않는다.
- 플레이어가 상대 정보를 근거로 제약 위험을 조절할 수 있다.
- 데모 5전과 전체 대회 10전 진행도를 혼동하지 않는다.
- 최소 해상도와 긴 문구에서도 첫 행동과 잠금 버튼이 화면 밖으로 밀리지 않는다.
- 모션·점멸 감소와 자산 누락 상태에서도 같은 정보와 결과가 전달된다.
