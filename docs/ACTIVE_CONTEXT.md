# 십보강호 활성 컨텍스트

> 목적: 새로운 채팅이나 작업자가 과거 대화 없이 저장소만 읽고 현재 방향, 확정 규칙, 미확정 범위, 실제 검증 상태와 다음 작업을 복원하게 한다. 사용자와 작업자는 먼저 다섯 활성 기획서를 읽고, 이 문서는 상세 인수 부록으로 사용한다.

## 1. 현재 상태

- 장르: 1대1 무협 로그라이트 비무 대회.
- 현재 단계: **다섯 활성 기획서 체계 정비 + T0 전투 프로토타입 실행 준비 + 문파·무공·아트 기준 확정**.
- 엔진: Godot 4.7.1 stable.
- 작업 브랜치: `agent/playtest-ready-combat-prototype`.
- Draft PR: `#2 feat: prepare T0 combat playtest and integrated planning docs`.
- 정적 검사와 문서 동기화 검사는 구성되어 있다.
- Godot headless와 Windows 수동 플레이테스트는 아직 완료로 확정하지 않는다.
- 최신 승인 이미지 원본 일부는 대화에서만 존재하며 저장소 반입이 필요하다.

## 3. 제품 핵심 경험

- 전장 10칸.
- 라운드 행동 타이밍 10개.
- 전체 대회 10전.
- 데모는 1~5전 예선 결승까지.
- AI는 플레이어 선택 전에 두 수를 잠근다.
- 플레이어도 두 수를 선택해 잠근다.
- 양측 두 수를 동시에 공개하고 1수→2수 순서로 해상한다.
- 첫 수 결과를 보고 둘째 수를 변경할 수 없다.
- 같은 타이밍에 양측 공격이 모두 유효하면 합을 판정한다.
- 낮은 최종 합 수치 쪽이 차이만큼 피해를 받고 동률은 피해가 없다.
- 플레이어는 상대의 거리·자원·성향을 읽어 두 수를 계획한다.

## 4. 전투 표현 원칙

- 전투 도메인이 위치, 자원, 합, 피해, 성장과 저장 결과를 소유한다.
- UI·애니메이션·VFX·사운드는 구조화 이벤트를 표현한다.
- 플레이어에게 개발용 강도·투자 태그를 직접 보여주지 않는다.
- 상대의 강함과 성장 성향은 이명·풍문·전적·정탐 문장으로 암시한다.
- 같은 표현은 반복 플레이에서 같은 의미를 가져야 한다.

## 5. 공식 무공 체계

### 핵심무공 8종

`검법 / 도법 / 권법 / 장법 / 창법 / 곤봉법 / 궁술 / 암기술`

### 보조무공 8종

`심법 / 호신공 / 독공 / 마공 / 보법 / 방패술 / 외공 / 의술`

- 부법은 공식 성장 계통에서 제외한다.
- 도끼는 개별 무기·연출 태그로만 사용한다.
- 점혈은 의술의 세부 기술·상태 태그다.
- 신법·경공은 보법의 세부 태그다.
- 한 회차 보조무공 보유는 최대 2개다.
- 핵심무공 10성 최대 1개, 보조무공 10성 최대 1개다.

## 6. 숙련도 구조

| 성급 | 결과 |
|---:|---|
| 3성 | 새로운 기술 1 해금 |
| 5성 | 기술 1 강화 — 신규 카드 없음 |
| 7성 | 새로운 기술 2 해금 |
| 9성 | 기술 2 강화 — 신규 카드 없음 |
| 10성·대성 | 핵심무공은 절초, 보조무공은 진의 해금 |

무공서 하나의 사용 가능 카드는 3성·7성·10성 총 3장이다. 문파 전용 무공서 28개 기준 카드 목표는 84장이다.

## 7. 10문파와 무공서

### 무당파

- 태극검(검법): 삼환투월 / 연자초수 / 절초·진무태극검
- 태극권(권법): 람작미 / 운수 / 절초·사량발천근
- 순양무극공(심법): 포원수일 / 자오주천 / 진의·순양귀일
- 제운종(보법): 제운답 / 종운등천 / 진의·운외천행

### 화산파

- 매화검법(검법): 백운출수 / 매화난영 / 절초·태악삼청봉
- 자하신공(심법): 자하초현 / 자기동래 / 진의·자하만천

### 소림사

- 나한권(권법): 나한출수 / 복호나한 / 절초·금강나한인
- 달마봉법(곤봉법): 반야봉 / 횡소천군 / 절초·달마복마진
- 금강불괴신공(호신공): 철포삼 / 금종조 / 진의·금강불괴

### 당문

- 당문암기비전(암기술): 추혼비도 / 폭우이화침 / 절초·만천화우
- 오독심경(독공): 오독침맥 / 만독식심 / 진의·만독귀원
- 금침의경(의술): 금침도혈 / 회양구침 / 진의·생사환맥
- 귀영보(보법): 잠영환위 / 귀영천변 / 진의·무흔무영

### 개방

- 항룡십팔장(장법): 항룡유회 / 비룡재천 / 절초·시승육룡
- 타구봉법(곤봉법): 악견난로 / 오구탈장 / 절초·천하무구
- 취팔선보(보법): 취선망월 / 팔선과해 / 진의·대취무흔

### 마교

- 혈염도법(도법): 혈영참 / 수라난무 / 절초·혈해멸도
- 역혈마공(마공): 역맥폭기 / 혈기연소 / 진의·마혈귀원

일반 마교 교도 무공에는 `천마` 명칭을 사용하지 않는다.

### 양가장

- 양가창법(창법): 이화분영창 / 회마반창 / 절초·칠성연환창
- 양문철벽패술(방패술): 금탕고수 / 철벽추진 / 진의·양문불퇴

### 백우궁

- 백우궁술(궁술): 백보천양 / 연주구성 / 절초·천우낙성
- 유운보법(보법): 비연답 / 답설무흔 / 진의·풍운무정

### 사파

- 오호단문도(도법): 맹호하산 / 오호단문 / 절초·군호파악
- 추혼암기술(암기술): 유엽비도 / 추혼십이표 / 절초·천라추혼
- 철골외공(외공): 동피철골 / 횡련강신 / 진의·십삼태보

사파 전용 계통에는 검법·장법·보법을 배치하지 않는다.

### 녹림

- 녹림창법(창법): 독룡출동 / 횡소천군 / 절초·회마칠성창
- 산림궁술(궁술): 관목잠시 / 연주산우 / 절초·만산비우
- 산림잠행보(보법): 초상비 / 답엽무흔 / 진의·천산무영

## 8. 무공 카드 앞면

- 좌상단: 문파 문양 이미지. 공용은 무(無) 상징 이미지.
- 최상단 중앙: 사거리 슬롯. 숫자는 UI 데이터 레이어.
- 상단 중앙: 기술 이름.
- 우상단: 무공 종류 아이콘 이미지. 검법·권법 같은 텍스트 직접 표기 금지.
- 중앙: 기술 삽화.
- 하단: 비용·효과 없이 무공의 묘리 한 소절.
- 묘리는 큰 글자로 최대 두 줄.
- 10성 핵심무공은 우측 세로 `절초`, 보조무공은 `진의` 표찰.

상세 팝업은 후속 단계에서 이름, 대상, 효과, 특수효과, 비용, 설명·키워드를 표시한다.

## 9. 문파 종합 시트와 아트 방향

- 공통 그림체: 선명한 반실사 무협 인물 + 수묵·한지 배경 + 먹색·금색 UI.
- 남녀 대표 제자는 겹치지 않는 독립 세로 패널에 배치한다.
- 한 명은 핵심무공, 다른 한 명은 보조무공 또는 두 번째 핵심무공의 성격을 보여준다.
- 중앙에 무공서, 우측에 3성·7성·10성 카드 이미지를 배치한다.
- 무공서 한 권과 카드 세 장을 같은 가로 행에 정렬한다.
- 생성 이미지의 한글·수치·분류는 기준 데이터가 아니다.
- 정확한 명칭과 분류는 편집 가능한 UI 레이어와 아이콘 자산으로 합성한다.

## 10. 절초 기세

절초는 두 조건을 가진다.

```text
핵심무공 10성 → 절초 해금
전투 중 기세 충전 → 해당 전투에서 절초 사용 가능
```

승인 방향:

- 양측 상단 HUD에 칸형 기세 게이지를 표시한다.
- 잠금, 축적 중, 발동 가능, 예약, 소모 상태를 구분한다.
- 증가량뿐 아니라 합·회피·막기 같은 증가 이유를 표시한다.
- UI·VFX·사운드는 획득·상한·소모를 계산하지 않는다.

미확정:

- 자원 명칭.
- 최대치와 발동 요구치.
- 행동별 획득량과 중복 상한.
- 전투 시작값과 초기화 규칙.
- 사용 실패·취소 시 소모 규칙.
- 상대 게이지 공개 범위와 AI 사용 기준.

## 12. 현재 위험

- 기존 부록에 12세력·3·6·9성 표현이 남아 있을 수 있다.
- Godot 실행 환경에서 실제 런타임 검증이 완료되지 않았다.
- 승인된 최신 이미지 원본 일부가 저장소에 없다.
- 기술 효과·비용·밸런스는 아직 미확정이다.
- 생성 이미지에서 분류 한글이 잘못 표기될 수 있다.

대응:

- 다섯 활성 기획서를 우선 기준으로 사용한다.
- 구형 부록은 검색·동기화한다.
- 텍스트와 아이콘을 이미지 후합성한다.
- 이미지 인덱스에서 원본 반입 상태를 관리한다.
