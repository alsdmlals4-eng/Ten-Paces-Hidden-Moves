# 준비 강화·전조 표시·자동 배치 설계

## 목표

전투 계획 조작을 카드 1회 선택 중심으로 단순화하고, `basic_stance`의 사용자 표시명을 `[준비]`로 바꾸며, 준비 강화의 수명과 명상 보상을 명확히 한다. 다중 슬롯 행동의 실행 전 점유 단계는 카드 `[준비]`와 혼동되지 않도록 `[전조]`로 표시한다.

## 기준 브랜치

- Base: `agent/repeat-poc-a3-review-ui-closeout`
- 신규 스택: `agent/prepare-momentum-and-auto-placement`
- 기존 A3 review gate·가설 snapshot·결정적 복기 흐름은 보존한다.

## 확정 규칙

### 카드 명칭

- 내부 ID `basic_stance`는 유지한다.
- 사용자 표시명 `태세`를 `준비`로 변경한다.
- 카드 설명·로그·상태 설명의 사용자 노출 표현은 `[준비]`로 통일한다.

### 다중 슬롯 행동

- `action_stage = preparation` 내부 식별자는 유지한다.
- 2슬롯·3슬롯 행동의 최종 실행 수 이전 점유 수는 사용자에게 `[전조]`로 표시한다.
- 예: 2슬롯 공격은 1수 `[전조]`, 2수 `[공격]`이다.
- 예: 3슬롯 절초는 1수·2수 `[전조]`, 3수 `[절초]`이다.

### 준비 강화 상태

- `[준비]` 실행 시 다음 비이동 실행 행동을 강화 대상으로 보관한다.
- `이동`과 `보법`은 준비 상태를 소비하지 않는다. 이동 실패·제자리 이동도 이동 행동이므로 소비하지 않는다.
- 명상이 실제 실행되면 기존 기력·내력 회복에 절초 기세 +1을 추가하고 준비 상태를 소비한다.
- 공격이 실제 실행되면 기존 공격력 보너스와 `[강건]`을 적용하고 준비 상태를 소비한다.
- 막기·회피·기타 비이동 행동은 해당 행동 처리 후 준비 상태를 소비한다.
- 자원 부족·중단 등으로 비이동 행동이 실제 실행되지 않아도 그 행동 시도에서 준비 상태를 소비한다.
- 절초 기세는 최대값을 넘지 않는다.

### 자동 배치

- 기초 카드와 절초 모두 선택 즉시 현재 묶음에서 가장 앞의 연속된 빈 구간에 배치한다.
- 1슬롯 카드는 가장 앞의 빈 1수, 2슬롯 카드는 가장 앞의 연속된 빈 2수, 3슬롯 카드는 가장 앞의 연속된 빈 3수에 배치한다.
- 유효 구간이 없으면 배치하지 않고 자원도 소비하지 않는다.
- 이동·공격 카드가 자동 배치되면 기존 대상 선택 흐름으로 즉시 진입한다.
- 자동 배치 이후에도 배치된 슬롯 클릭으로 기존 해제 동작을 유지한다.

### 절초 예약·환불

- 절초 선택 시 기세가 정확히 5이고 연속 빈 구간이 있을 때만 자동 예약한다.
- 자동 예약 성공 시 기세 5를 즉시 소비한다.
- 판정 진행 전 예약 슬롯을 클릭해 취소할 수 있다.
- 예약 취소 시 기세 5를 반환한다.
- 판정 진행 이후 중단·방향 실패·사거리 실패에는 환불하지 않는다.

## 아키텍처

### `ActionTimingPanel`

- `find_earliest_open_anchor(span: int) -> int`를 제공한다.
- 현재 묶음의 actionable index만 순회하며 `place_card()`와 동일한 연속 점유 조건을 사용한다.
- `0`은 배치 가능한 구간 없음이다.

### `CombatBoardPreview`

- 카드 선택 handler가 `find_earliest_open_anchor()`를 호출하고 즉시 `place_card()`를 수행한다.
- 기초 카드와 절초가 같은 자동 배치 경로를 사용하되, 절초는 예약 가능성 검사와 기세 소비·환불을 기존 메서드로 유지한다.
- 슬롯 클릭은 선택 카드 배치가 아니라 기존 배치 해제만 담당한다.

### `CombatResolutionEngine`

- actor 상태에 준비 강화 여부를 명시적으로 저장한다.
- 이동 phase는 준비 상태를 변경하지 않는다.
- 비이동 행동 처리 시작 시 준비 상태의 소비 여부를 결정한다.
- 명상 실행 시 준비 상태가 있었다면 momentum +1을 적용한다.
- 공격 보너스·강건은 준비 상태가 있는 다음 공격에 적용한다.
- 사용자 로그의 다중 슬롯 실행 전 표시는 `[전조]`로 바꾼다.

### 카드 데이터

- `basic_stance`의 `name`, `effect_text`, `tags`, `flavor`에서 사용자 표현을 `준비`로 동기화한다.
- 내부 ID와 category는 변경하지 않는다.

## 실패 처리

- 자동 배치 가능한 연속 구간이 없으면 `[배치 불가] 연속된 빈 행동 슬롯이 부족합니다.`를 기록한다.
- 절초 기세가 5가 아니면 예약하지 않는다.
- 절초 자동 배치 실패 시 기세를 먼저 소비하지 않는다.
- 예약 성공 후 사용자가 슬롯을 클릭해 취소하면 전체 점유 구간을 제거하고 기세 5를 복원한다.

## 테스트

1. 정적 계약
   - `basic_stance` 표시명은 `준비`다.
   - `ActionTimingPanel`에 `find_earliest_open_anchor`가 있다.
   - 카드 선택 handler는 자동 배치를 사용한다.
   - 다중 슬롯 preparation 로그는 `[전조]`다.

2. Godot 준비 상태 회귀
   - 준비→명상은 momentum +1, 준비 상태 소모.
   - 준비→이동→명상은 이동 후 준비 유지, 명상 후 momentum +1 및 소모.
   - 준비→이동→공격은 이동 후 유지, 공격 보너스·강건 적용 후 소모.
   - 준비→비이동 실패는 준비 상태 소모.
   - momentum 5에서 준비→명상은 5를 넘지 않음.

3. Godot 자동 배치 회귀
   - 1슬롯 카드가 가장 앞 빈 수에 배치됨.
   - 앞 수가 차 있으면 다음 빈 수에 배치됨.
   - 2슬롯 카드는 가장 앞 연속 빈 2수에 배치됨.
   - 연속 구간이 없으면 실패함.
   - 절초는 가장 앞 연속 구간에 자동 예약되고 기세 5 소비.
   - 예약 슬롯 클릭 취소 시 기세 5 반환.

4. 전체 회귀
   - 기존 전투 판정·A1 AI·A2 summary·A3 review gate·접근성 verifier 전부 통과.
