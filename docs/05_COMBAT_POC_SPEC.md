# 십보강호 전투 POC·세로 슬라이스 명세

> 책임: T0 전투 POC의 목적·범위·구현 상태·증거·완료 기준과 T1 이후 진입 게이트  
> 판정 원본: `docs/02_COMBAT_RULES.md`  
> 검증 원본: `docs/08_TEST_CHECKLIST.md`  
> 구현 기준: PR #7 `agent/t0-combat-poc-board@147a031c75e96bff170d7f99016beb9e85b12066`, Issue #13

## 1. 목적

T0는 12세력·10전 전체판을 만드는 단계가 아니다. 가장 위험한 가설을 단일 결투로 검증한다.

> **10칸 전장에서 거리·방향·자원·상대의 공개 상태를 읽고 `3수 → 3수 → 4수`로 계획한 결과가 합·대응·이동·중단·절초로 설명 가능하게 이어질 때, 플레이어가 자신의 예측과 수정 판단에서 반복 플레이 동기를 느끼는가?**

현재 구현은 이 질문을 기술적으로 실행할 수 있는 상태다. 실제 사용자에게 재미·이해·재도전 의향이 확인됐다는 뜻은 아니다.

## 2. 단계 구분

### T0 — 단일 전투 POC

현재 PR #7의 제품 범위다.

- `[강호낭인]` 대 공개 상태 기반 최소 AI의 1대1 전투.
- 전장 10칸, 플레이어 4번·상대 7번, 시작 거리 3.
- 같은 칸 `[밀착]`, 공동 목적지 허용, 자리 교환·통과 금지.
- 라운드 `3수 → 3수 → 4수`, 총 10수.
- 기초 행동 8종과 절초 3종.
- 이동 목적지·공격 방향·다중 슬롯 준비/실행.
- 기력·내력 계획 미리보기.
- 합·순차 방어·회피·필중·중단·강건.
- 기세 예약·취소·소모.
- 공개 상태 기반 결정적 최소 AI.
- 승리·패배·무승부·4/7 완전 재시작.
- 순차 결과 연출, 로그, 키보드 포커스, 모션 감소, 음향 제어.

T0에 포함하지 않는다.

- 세력 선택과 문파별 판정 규칙.
- 무공·심법·성급·수련.
- 대회 진행·보상·정탐·제약.
- 회차 저장·불러오기·재도전 자원.
- T1용 성향 다양화·실수율·정식 난이도 AI.
- 전체판 콘텐츠 수량 확정.

### T1 — 최소 세로 슬라이스

T0의 실제 사용자 플레이 증거가 통과한 뒤 진행한다.

- 플레이 스타일 2개.
- 상대 3명 내외.
- 전투 3회 내외.
- 전투 전 상대 정보.
- 전투 후 보상 선택.
- 무공 또는 능력 강화 1회 이상.
- 최종 강적 1회.

목적은 5전 데모 제작 전에 `정보 → 계획 → 전투 → 보상 → 다음 전투`의 연결이 반복 동기를 만드는지 확인하는 것이다.

### T2 — 5전 데모

- 1~2전 내부전.
- 3~4전 예선.
- 5전 예선 결승.
- 시작 무공·심법과 수련.
- 상대 정보·제약·성과·보상.
- 조건부 절초 조기 체험.

절초가 없어도 5전을 정상 시작·완료해야 한다.

### 전체판 방향

- AI 싱글플레이 1대1 대회 10전.
- 8전 8강, 9전 준결승, 10전 결승.
- 시작 6세력과 추가 6세력의 12세력 후보.
- 무공·심법 1~10성, 절초·진의.
- 정탐·제약·재도전·저장·제한적 영구 해금.

T1·T2 결과에 따라 수량·명칭·경제·회차 길이를 줄이거나 합칠 수 있다.

## 4. 현행 T0 계약

세부 수치와 예외는 `docs/02_COMBAT_RULES.md`가 소유한다.

### 전장·라운드

- 10칸, 4/7 시작, 거리 3.
- `[밀착]` 거리 0과 한 칸 최대 2인.
- `[3,3,4]`, 10수 뒤 다음 라운드.
- 현재 묶음 완전 배치·대상·자원 유효성 후 진행.

### 시작 상태

| 자원 | 양측 시작 |
|---|---:|
| 체력 | 30/30 |
| 공격력 | 8 |
| 기력 | 5/5 |
| 내력 | 4/4 |
| 절초 기세 | 0/5 |

### 현재 판단 요소

- 이동·보법으로 다음 공격 거리와 밀착 여부를 만든다.
- 속공의 이동 전 우선 위치와 일반 공격의 실행 위치를 읽는다.
- 같은 수 공격의 합 공격력 차이를 예측한다.
- 막기·회피·태세 결합과 필중의 상호작용을 계산한다.
- 기력·내력과 명상 순서를 계획한다.
- 중단 위험과 강건 보호를 고려해 다중 슬롯 행동을 배치한다.
- 기세 5를 절초 예약에 즉시 투자할지 결정한다.
- 공개 상태만 보는 최소 AI의 반복 결정을 읽는다.

## 6. STEP 14 플레이테스트 계약

### 6.1 검증 질문

1. 플레이어가 3수·3수·4수 묶음의 역할 차이를 설명하는가?
2. 첫 묶음에서 거리 3이 접근·대응·자원 선택을 만드는가?
3. 이동 목적지와 공격 방향을 혼동하지 않는가?
4. 같은 수 공격이 왜 합이 됐고 피해가 어떻게 정해졌는지 설명하는가?
5. 방어도 차감→같은 수 반감→필중/회피 순서를 이해하는가?
6. 어떤 행동이 왜 중단됐고 강건이 무엇을 보호했는지 이해하는가?
7. 기세 획득·절초 예약·취소·소모 이유를 설명하는가?
8. 상대가 비공개 계획을 보지 않는다고 신뢰하는가?
9. 실패 뒤 다른 계획을 시도하려는가?
10. 한 판 후 다시 플레이하고 싶은가?

## 7. T0 완료 기준

다음을 모두 만족해야 T1 진입을 검토한다.

### 기능·정합성

- 전장·카드·절초·판정·AI·종료·재시작 계약이 데이터·코드·테스트·문서에서 일치한다.
- 정상·실패·경계·반례·회귀가 자동 또는 런타임에서 재현된다.
- 비공개 계획 누출과 재시작 상태 누적이 없다.
- 활성 문서·Skill·PR이 구형 계약을 현재로 설명하지 않는다.

### 플레이어 증거

- 치명적 입력·판정 오류 없이 한 판을 완료한다.
- 핵심 실패 이유를 대체로 설명한다.
- 최소한 일부 상대 행동 규칙을 발견한다.
- 계획을 바꿔 재시도하려는 행동이 관찰된다.
- 핵심 정보가 색·모션·음향 하나에만 의존하지 않는다.

### 생산 판단

- 코어를 강화하지 않는 기능은 T1에서 제외한다.
- 새로운 세력·무공이 공용 전장·수·거리·자원·판정 계약을 재사용할 수 있다.
- 접근성 장벽과 목표 플랫폼 성능 위험을 기록한다.
