## 1. 현재 한 줄 약속

> **상대의 공개된 상태와 반복 습관을 읽고, 서로의 현재 계획을 모른 채 10칸 전장에 `3수 → 3수 → 4수`로 수를 걸어, 거리·`[합]`·대응·중단으로 한 수를 파훼하고 그 이유를 복기해 다음 계획을 바꾸는 1대1 무협 심리 전술 로그라이트.**

### 시장 차별화 문장

> **미래를 미리 보는 무협 게임이 아니라, 보이지 않는 상대의 수를 읽고 파훼하는 무협 결투.**

### 플레이어 약속

> **어려운 실시간 콤보를 외워 이기는 것이 아니라, 상대의 습관을 읽고 한 수 앞에서 파훼한 무인이 된다.**

### 코어 우선순위

```text
대전 격투식 수읽기·파훼
> 전술 퍼즐식 거리·순서
> 로그라이트 성장
```

로그라이트 성장은 다음 결투를 더 흥미롭게 만드는 보조 구조다. 전투의 수읽기와 파훼를 원시 능력치 상승이나 무작위 손패로 대체하지 않는다.

## 2. 현재 핵심 경험 가설과 검증 상태

프로젝트 코어는 확정됐지만, 실제 플레이어가 이 경험을 재미있고 이해 가능하다고 느끼는지는 STEP 14에서 검증한다.

```text
READ      공개 위치·거리·자원·상태·이전 행동을 읽는다
→ HYPOTHESIZE  상대 성향과 가능한 수를 좁힌다
→ PLAN         현재 3/3/4 묶음에 행동·대상·자원을 배치한다
→ COMMIT       틀릴 위험을 받아들이고 계획을 확정한다
→ RESOLVE      대응 → 속공 → 이동 → 일반 공격을 본다
→ REVIEW       예상과 실제의 결정적 차이를 이해한다
→ ADAPT        다음 묶음 또는 재시작에서 계획을 바꾼다
```

### 뾰족한 재미

> **간파한 위험에 수를 걸고, 공개 순간 거리·합·대응이 맞아 상대의 계획을 끊은 뒤, 그 근거를 복기해 다음 수에서 더 정확해지는 쾌감.**

### 핵심 감정

- `간파`: 상대가 무엇을 노리는지 짐작했다.
- `결단`: 틀릴 위험을 감수하고 현재 묶음을 확정했다.
- `파훼`: 거리·합·대응·중단으로 상대 계획을 무너뜨렸다.
- `납득`: 성공·실패의 결정적 원인을 이해했다.
- `적응`: 다음 계획을 실제로 바꿨다.

### 증거 상태

```yaml
project_core: CORE_CONFIRMED
step_0_to_13: IMPLEMENTED
mechanical_step14: RECORDED
human_step14: NOT_RUN
production_gate: REPEAT_POC
t1_greenlight: NOT_GRANTED
```

## 3. 대상 플레이어와 제약

### 대상 플레이어

- 격투 게임의 상대 읽기와 심리전을 좋아하지만 실시간 입력·콤보 암기 부담은 줄이고 싶은 플레이어.
- 적은 규칙을 반복해서 익히며 위치·순서·대응 숙련을 높이는 전술 플레이어.
- 실패 원인을 이해하고 다음 시도에서 계획을 바꾸는 학습형 반복 플레이를 선호하는 플레이어.
- 싱글플레이에서 개성이 읽히는 라이벌과 겨루고 싶은 플레이어.

### 제약

- 1대1 싱글플레이 무협 결투가 우선이다.
- 전장은 10칸 일자형이며 기본 시작은 플레이어 4번·상대 7번이다.
- 현재 묶음의 행동은 서로에게 비공개다.
- AI는 공개 상태와 기록만 사용하고 플레이어의 미확정 계획을 읽지 않는다.
- 기초 행동은 항상 사용할 수 있는 소수 공용 어휘다.
- 덱·손패·행동력·내공은 코어 경제가 아니다.
- 결과는 결정론적으로 계산되고 이유를 설명할 수 있어야 한다.
- 색·모션·음향 하나에만 핵심 정보를 의존하지 않는다.
- 소규모 Godot 제작에서 상대·무공·세력을 데이터 기반으로 반복 생산할 수 있어야 한다.

## 4. 설계 원칙

### 4.1 수읽기가 피해량보다 우선한다

- 높은 피해 하나보다 상대의 의도·거리·순서·대응을 읽는 판단을 우선한다.
- 공격 실패와 피해 변화는 방향·사거리·합·방어·회피·필중·중단 중 무엇 때문인지 설명한다.
- AI 난이도는 비공개 정보 치팅이 아니라 성향·후보 선택·예측 깊이·제한된 실수로 만든다.

### 4.2 적은 공용 행동에서 깊이를 만든다

현행 기초 행동 8종은 이동·보법·막기·회피·속공·강공·명상·태세다.

새 행동을 늘리기 전에 다음 관계를 검증한다.

- 거리 0 `[밀착]`과 거리 1 이상.
- 속공·이동·일반 공격의 실행 순서.
- 같은 실행 수 공격의 `[합]`.
- 방어도·같은 수 반감·회피·`[필중]`.
- 다중 슬롯 준비·실행과 같은 수 중단.
- 기력·내력·명상·기세·절초 투자.

### 4.3 불완전 정보와 공정성을 함께 지킨다

- 상대의 현재 계획은 완전히 보여 주지 않는다.
- 플레이어가 읽을 공개 상태·행동 이력·성향 단서는 제공한다.
- AI는 플레이어의 미확정 슬롯·대상·절초 예약을 입력받지 않는다.
- 완전 랜덤 AI와 단일 고정 스크립트 사이에서 읽을 수 있는 복수의 합리적 후보를 설계한다.

### 4.4 계산·표현·복기를 분리한다

- 전투 도메인은 위치·비용·합·피해·방어·상태·기세·중단·승패를 계산한다.
- UI는 선택 가능 상태·예상 자원·대상·결과와 이유를 표현한다.
- VFX·애니메이션·오디오는 확정 이벤트를 재생하며 결과를 다시 계산하지 않는다.
- 복기 화면은 `내 예상 / 상대 실제 / 결정적 교차점 / 다음에 바꿀 선택`을 상세 로그보다 먼저 보여 준다.

### 4.5 모든 요소에 기획의 필연성을 요구한다

```text
WHY  : 전달할 경험과 유도할 행동은 무엇인가?
HOW  : 그 경험을 만들 전략과 제약은 무엇인가?
WHAT : 플레이어가 실제로 보는 규칙·화면·콘텐츠는 무엇인가?
```

WHY와 연결되지 않는 WHAT은 `CHANGE / REMOVE / DEFER / RETEST` 후보다.

## 5. 핵심 루프와 보상

| 단계 | 행동 | 선택 | 피드백 |
|---|---|---|---|
| READ | 공개 상태·행동 이력을 읽음 | 어떤 위험이 가장 가능한가 | 거리·자원·상태·이력 |
| HYPOTHESIZE | 상대 성향을 가정 | 어떤 역수가 여러 후보를 견디는가 | 공개 단서·이전 결과 |
| PLAN | 행동·대상·자원을 배치 | 선점·대응·자원 중 무엇에 걸 것인가 | 슬롯·대상·자원 미리보기 |
| COMMIT | 계획 확정 | 틀릴 위험을 받아들일 것인가 | 입력 잠금·계획 공개 |
| RESOLVE | 판정 관찰 | 어느 교차점이 승부를 갈랐는가 | 합·대응·중단·기세 |
| REVIEW | 예상과 실제 비교 | 무엇을 잘못 읽었는가 | 결정적 원인 요약 |
| ADAPT | 다음 계획 변경 | 다음에는 무엇을 바꿀 것인가 | 새 가설·재도전 |

### 보상 사다리

- `Micro`: 합 승리, 정확한 대응, 중단, 거리 선점, 기세 획득.
- `Session`: 한 라이벌의 성향을 발견하고 결투 승리.
- `Meta`: 원시 능력치가 아니라 새로운 전투 관점·행동 변형·상대 정보 접근 해금.

## 6. 프로젝트 코어 보호 경계

### INVARIANT

- 1대1 무협 라이벌 결투.
- 10칸 선형 전장.

### CHANGEABLE

- 체력·피해·기력·내력·기세의 정확한 수치.

- HUD·아트·음향·애니메이션 외피.

### REQUIRES_REAPPROVAL

- 10칸 전장을 다른 공간 구조로 교체.
- 3/3/4 계획 구조 제거 또는 본질 변경.
- 상대 계획을 완전히 미리 보여 주기.
- 덱·손패·행동력 경제를 핵심으로 도입.
- 실시간 입력 숙련을 승부의 주요 기준으로 전환.
- PvP 또는 원시 능력치 성장을 제품의 1차 구조로 전환.

### OUT_OF_SCOPE — T1

- 12세력 전체 제작.
- 무공·심법 1~10성 전체 성장.
- 10전 대회 전체 캠페인.
- 대형 시장·정탐·세력 관계·연구 트리.
- 대규모 서사 분량과 진영 전용 판정 규칙.

## 9. 제품 범위

### T0 — 단일 전투 POC

현재 STEP 0~13 구현과 STEP 14 사람 검증 단계다. 성장·대회·저장·정식 라이벌 성향 시스템은 포함하지 않는다.

### T1 — 라이벌 최소 세로 슬라이스

T0 사람 증거 뒤 다음으로 제한한다.

- 플레이 스타일 2개.
- 읽을 수 있는 성향이 다른 상대 3명.
- 전투 3회.
- 전투 전 상대 정보.
- 수평적 보상 선택 1회 이상.
- 최종 라이벌 1회.

T1의 질문은 `정보 → 가설 → 계획 → 전투 → 복기 → 보상 → 다음 라이벌`이 반복 동기를 만드는가이다.

### T2 — 5전 데모

- 1~2전 내부전, 3~4전 예선, 5전 예선 결승.
- 상대 정보·제약·성과·수평 성장 연결.
- 절초가 없어도 정상 완주.

### 전체판 가설

10전 대회·12세력·무공·심법 1~10성·정탐·저장·영구 해금은 확정 수량이 아니다. T1·T2 증거와 제작 비용에 따라 축소·통합·삭제한다.

## 10. 유지·강화·변경·삭제·보류

### KEEP

10칸, 3/3/4, 기초 행동 8종, 덱 없음, 비공개 계획, 합·대응·중단, AI 비치팅 금지, 설명 가능한 엔진, 완전 재시작.

### AMPLIFY

라이벌 성향, 계획 대비 결과, 거리·순서·파훼, 묵직한 무협 연출, 실패 뒤 계획 변경.

### CHANGE

결정적 최소 AI를 읽을 수 있는 성향 정책으로 확장하고, 상세 로그보다 결정적 복기 요약을 우선하며, 성장은 원시 능력치보다 수평적 행동 변형으로 설계한다.

### REMOVE

12세력·10성·10전을 현재 확정 수량으로 취급하는 전제와, 코어 검증 전에 시스템을 계속 추가하는 관성을 제거한다.

### DEFER

전체 대회·대형 경제·정탐·세력 관계·대규모 저장·서사 확장.

### RETEST

3/3/4 인지 부하, 합 공식 지배, 기세·절초 눈덩이, 8개 행동 온보딩, 자발적 재도전과 세션 길이.

## 11. 프로젝트 성공 조건

### 코어 경험

- 플레이어가 상대 성향을 근거로 가설을 세운다.
- 계획 공개 전에 의미 있는 위험과 결단을 느낀다.
- 합·대응·중단의 결정적 원인을 자기 말로 설명한다.
- 실패 뒤 다음 묶음 또는 재시작 계획을 실제로 바꾼다.
- 수읽기·파훼가 원시 능력치·무작위 보상보다 승부를 지배한다.

### 공정성·접근성

- AI가 미확정 계획을 읽지 않는다.
- 결과가 로그·텍스트·형태를 통해 설명 가능하다.
- 핵심 정보가 색·모션·음향 하나에만 의존하지 않는다.
- 포인터 외 키보드와 필요한 대체 입력 경로를 제공한다.

### 제작성

- 새 상대는 성향·후보 수·위험 선호·공개 단서·카운터 창의 공통 문법을 재사용한다.
- 새 무공은 읽기·거리·순서·대응 중 하나의 질문을 바꾸며 단순 피해 상향에 그치지 않는다.
- 신규 콘텐츠가 독립 판정 엔진이나 별도 UI 문법을 요구하지 않는다.

## 12. 다음 기획 게이트

### STEP 14 사람 플레이

신규 플레이어 5명을 대상으로 한 발견형 테스트를 먼저 실행한다.

- 4명 이상이 치명적 차단 없이 한 판 완료.
- 4명 이상이 3/3/4와 결정적 판정 원인 하나를 설명.
- 3명 이상이 안내 없이 상대 성향 하나를 발견.
- 3명 이상이 재도전에서 계획을 실질적으로 변경.
- 3명 이상이 자발적으로 다시 하거나 구체적인 다음 수를 제시.
- 핵심 정보가 색·모션·음향 하나에만 의존한 참가자가 없음.

이 수치는 통계적 시장 증명이 아니라 T1 진입 발견 신호다. 실패하면 성공 기준을 바꾸지 않고 `REPEAT_POC`한다.

### T1 진입 판정

```text
CORE_CONFIRMED
+ STEP 14 사람 증거
+ 결정적 복기 이해
+ 최소 라이벌 성향 발견
+ 접근성 치명 장벽 없음
= T1_GREENLIGHT_REVIEW
```

현재 판정은 `CORE_CONFIRMED / REPEAT_POC / T1_NOT_GRANTED`다.
