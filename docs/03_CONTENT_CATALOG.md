# 십보강호 콘텐츠 카탈로그

> 책임: 현재 구현 콘텐츠, T1 계획, T2·전체판 가설과 HOLD 경계를 구분한다.  
> 전투 규칙: `docs/02_COMBAT_RULES.md`  
> 단계 범위: `docs/05_COMBAT_POC_SPEC.md`

## 1. 상태 구분

- `CURRENT_T0`: PR #7 데이터·코드·씬·테스트에 연결된 콘텐츠.
- `PLANNED_T1`: T0 사람 검증 통과 뒤 최소 세로 슬라이스에서 제작할 후보.
- `HYPOTHESIS_T2_PLUS`: 5전 데모 또는 전체판 방향 후보.
- `HOLD`: 사용자 재개 승인 전 활성 기획·구현·테스트에 넣지 않는 요소.

콘텐츠 수나 문서 존재만으로 구현 완료를 표시하지 않는다.

## 2. CURRENT_T0 — 단일 전투 POC

### 2.1 전투원·전장

| 역할 | 현재 콘텐츠 |
|---|---|
| 플레이어 | `[강호낭인]`, 4번 시작 |
| 상대 | `[배외검객]`, 공개 상태 기반 최소 AI, 7번 시작 |
| 전장 | 1~10번 일자형 10칸 |
| 시작 거리 | 3 |
| 밀착 | 같은 칸 최대 2인, 거리 0 `[밀착]` |

- 정지 상대 칸 진입과 같은 빈 칸 공동 목적지를 허용한다.
- 자리 교환과 상대 통과는 금지한다.
- 상대 AI는 플레이어의 미확정 계획·대상·절초 예약을 읽지 않는다.
- 고정 행동 묶음은 AI를 끈 독립 회귀 fixture에서만 사용한다.

- 예약 즉시 기세를 소비하고 `[진행]` 전 취소만 환불한다.
- 절초 데이터 원본은 `data/cards/ultimate_cards.json`이다.

### 2.4 현재 규칙 콘텐츠

- 라운드 `3수 → 3수 → 4수`.
- 이동 목적지·공격 방향.
- 자원 계획 미리보기.
- 같은 실행 수 공격의 `[합]`과 동점 상쇄.
- 방어도 차감→같은 수 반감.
- 회피와 파공검기 `[필중]`.
- 같은 수 미실행 행동 중단.
- 태세 기반 `[강건]`.
- 승리·패배·무승부·4/7 완전 재시작.

### 2.5 현재 UI·아트·사운드

- 프로젝트 전용 수묵 석양 전장 배경.
- 플레이어·상대 전신 수묵 캐릭터와 초상.
- 상단 체력·기력·내력·기세·상태 HUD.
- `3수 → 3수 → 4수` 슬롯과 기초 카드 8종.
- 절초 목록 3종.
- 카드 상세·전투 로그·진행·재시작.
- 이동·공격·선택·잠금·중단·합·필중 결과 표시.
- 순차 `timing_results` 연출, 빠른 재생·즉시 완료·모션 감소.
- 절차적 SFX, 음소거·음량 조절.
- 명시적 키보드 포커스와 Windows UI Automation 메타데이터.

현재 자산은 프로젝트 POC 승인 자산이며 최종 상용 품질 확정은 아니다.

## 3. PLANNED_T1 — 최소 세로 슬라이스

T0의 실제 사용자 STEP 14와 코어 재정의가 통과한 뒤 최소 범위로 제작한다.

| 영역 | 최소 범위 |
|---|---|
| 플레이 스타일 | 2개 |
| 상대 | 3명 내외 |
| 전투 | 3회 내외 |
| 전투 전 정보 | 상대 성향·대표 행동·거리 선호 |
| 전투 후 | 보상 선택 1회 이상 |
| 성장 | 무공·능력 강화 1회 이상 |
| 최종 검증 | 강적 1회 |

### 3.1 플레이 스타일 후보

정확한 세력명을 확정하기 전에 전투 역할을 검증한다.

- 근접 압박형: 접근·밀착·속공·강공·태세 연계.
- 거리 운용형: 보법·사거리·대응·자원 관리.

기초 행동·절초만으로 역할이 충분히 구분되지 않을 때 고유 무공을 추가한다.

### 3.2 상대 후보

- 접근 선호.
- 거리 유지.
- 대응·자원 관리.

각 상대는 공개 정보와 관찰 가능한 반복 성향만 사용한다.

## 5. 무공·심법 추가 원칙

고유 콘텐츠는 가능한 한 현행 공용 계약을 조합한다.

- 행동 슬롯과 준비/실행.
- 기력·내력 비용.
- 거리·이동·방향.
- 피해·회복·방어·회피.
- 합·필중·중단·강건.
- 기세와 절초 예약.
- 상태 지속과 태그.

세력 하나를 위해 별도 라운드·전장·판정 엔진을 만들지 않는다.

T1 권장 콘텐츠 단위:

```text
대표 고유 행동 1~2개
+ 안정형 공용 변형 1개
+ 조건형 고위험 행동 1개
```

## 6. HOLD

다음은 사용자 재개와 별도 규칙 승인 전 데이터·코드·테스트에 넣지 않는다.

- 행동력·내공.
- Focus/집중 상태.
- 무작위 손패·드로·버린 카드.
- 자유 통과·자리 교환.
- 세력별 독립 판정 순서.
- 별도 장전 시스템.
- 현재 공용 상태 계약에 없는 독·점혈·밀치기 세부 수치.
- 저장·영구 성장·재도전 경제의 확정값.

## 7. 콘텐츠 추가 게이트

새 콘텐츠는 다음 질문을 통과해야 한다.

1. 프로젝트 코어의 어떤 판단·감정을 강화하는가?
2. 기존 행동·절초와 다른 선택을 만드는가?
3. 공용 슬롯·거리·자원·합·대응 계약으로 표현 가능한가?
4. AI가 비공개 정보 없이 사용할 수 있는가?
5. 실패 이유를 로그와 결과 UI로 설명할 수 있는가?
6. 색·모션·음향 없이도 정보를 전달하는가?
7. 정상·실패·경계·반례 테스트를 만들 수 있는가?
8. T1 제작 비용 대비 반복 플레이 가치를 높이는가?

답하지 못하는 콘텐츠는 `REMOVE / DEFER / TEST` 후보로 둔다.
