# 십보강호 콘텐츠 카탈로그

> 책임: 현재 구현된 콘텐츠, 다음 세로 슬라이스 후보, 전체판 가설과 HOLD 경계를 구분한다.  
> 현행 전투 규칙: `docs/02_COMBAT_RULES.md`  
> 단계 범위와 진입 게이트: `docs/05_COMBAT_POC_SPEC.md`

## 1. 상태 구분

- `CURRENT_T0`: 현재 데이터·코드·씬·테스트에 연결된 콘텐츠.
- `PLANNED_T1`: T0 통과 후 최소 세로 슬라이스에서 검증할 콘텐츠.
- `HYPOTHESIS_T2_PLUS`: 5전 데모 또는 전체판 후보.

콘텐츠 수가 많다는 이유로 현재 구현 완료로 표시하지 않는다.

## 2. CURRENT_T0 — 단일 전투 POC

### 2.1 전투원

| 역할 | 현재 콘텐츠 |
|---|---|
| 플레이어 | `[강호낭인]`, 4번 칸 시작 |
| 상대 | 공개 상태 최소 AI 상대, 7번 칸 시작 |

- 전장은 1~10번의 일자형 10칸이다.
- 시작 거리는 3칸이다.
- 현재 상대는 플레이어의 비공개 계획을 읽지 않는 공개 상태 최소 AI다. 재현용 fixture는 테스트에만 사용한다.

### 2.2 기초 행동 8종

| ID | 이름 | 기능 | 현재 역할 |
|---|---|---|---|
| `basic_move` | 이동 | 이동 | 좌·우 1칸 위치 조정 |
| `basic_footwork` | 보법 | 이동 | 내력 1로 좌·우 1~2칸 위치 조정 |
| `basic_guard` | 막기 | 대응 | 방어도 4를 먼저 차감하고 같은 수면 남은 피해를 반감 |
| `basic_evade` | 회피 | 대응 | 같은 수 공격 완전 회피 |
| `basic_quick_attack` | 속공 | 공격 | 기력 1, 거리 1, 이동 전 피해 6 |
| `basic_heavy_attack` | 강공 | 공격 | 2슬롯, 기력 1·내력 1, 거리 1~2, 피해 8 |
| `basic_meditate` | 명상 | 회복 | 기력 2·내력 1 회복 |
| `basic_stance` | 태세 | 강화 | 내력 1, 다음 공격 +2 또는 대응 결합 |

원본 데이터는 `data/cards/basic_cards.json`이다.

### 2.3 대응 결합

- 태세+막기: 한 슬롯, 현재 묶음 전체 방어, 방어도 6.
- 태세+회피: 한 슬롯, 현재 묶음 전체 완전 회피.
- 결합 비용은 두 행동의 기력·내력 비용을 합산한다.

### 2.4 현재 전투 fixture

| 공개 상태 | 상대 최소 AI 우선순위 |
|---|---|
| 거리 3 | 이동으로 접근, 기세 5면 파공검기 검토 |
| 거리 1~2 | 속공·강공 또는 절초 사거리 판단 |
| 저체력 근거리 | 막기 또는 회피 |
| 자원 부족 | 명상 |

목적은 거리·대상·대응·자원·라운드 판정을 같은 입력에서 반복 검증하는 것이다.

### 2.5 현재 UI·표현 콘텐츠

- 수묵 산성 배경.
- 10칸 전투판과 양측 placeholder 캐릭터.
- 상단 체력·기력·내력·절초 기세·라운드 HUD.
- `3수 → 3수 → 4수` 행동 슬롯.
- 기초 행동 카드 8종.
- 카드 상세 패널.
- 접이식 전투 기록.
- 진행 버튼.
- 이동 가능·공격 가능 칸 상태.

현재 카드 원화와 캐릭터는 최종 상용 아트가 아니다.

## 3. PLANNED_T1 — 최소 세로 슬라이스

T0의 STEP 0~14가 통과한 뒤 아래 최소 범위에서 콘텐츠 연결을 검증한다.

| 영역 | 권장 최소 범위 |
|---|---|
| 플레이 스타일 | 2개 |
| 상대 | 3명 내외 |
| 전투 | 3회 내외 |
| 고유 행동 | 스타일별 3~5개 |
| 전투 전 정보 | 상대 성향·대표 행동·거리 선호 |
| 성장 | 강화 선택 1회 이상 |
| 보상 | 전투 후 선택 1회 이상 |
| 최종 검증 | 강적 1회 |

### 3.1 플레이 스타일 후보

정확한 세력명보다 전투 역할을 먼저 검증한다.

- 근접 압박형: 접근·속공·강공·태세 연계.
- 거리 운용형: 보법·대응·사거리 변화·자원 관리.

이 두 역할이 기초 행동만으로 충분히 구분되지 않을 때 고유 무공을 추가한다.

### 3.2 상대 후보

- 접근 선호 상대.
- 거리 유지 상대.
- 대응·자원 관리 상대.

상대는 플레이어의 현재 미확정 계획을 읽지 않고, 공개 정보와 반복 가능한 성향만 사용한다.

## 4. HYPOTHESIS_T2_PLUS — 세력 후보

다음 12세력은 전체판 정체성 후보이며 현재 T0 구현 목록이 아니다.

| 세력 후보 | 대표 계통 | 장기 역할 가설 |
|---|---|---|
| 화산파 | 검법 | 연속 검법과 공격 흐름 유지 |
| 개방 | 곤법 | 방어 후 거리 장악 |
| 당문 | 암기술 | 원거리 견제·상태 누적 후보 |
| 소림 | 권법 | 근접 압박과 피해 버팀 |
| 무당파 | 태극권·검법 | 대응 후 반격 |
| 마교 | 도법 | 체력 대가의 고위험 공격 |
| 양가장 | 창법 | 거리 2 유지와 추적 |
| 백우궁 | 궁술 | 장거리 이동·사격 연계 |
| 관군 | 창법·방패술 | 방어 후 전진 압박 |
| 아미 | 점혈·검법 | 자원 압박과 회복 |
| 오독 | 독장 | 상태 누적과 자원 순환 |
| 녹림채 | 도법 | 기력 탈취와 난전 가속 |

- 세력 수·명칭·무기·기믹은 T1·T2 결과에 따라 줄이거나 합칠 수 있다.
- 현재 T0 판정 엔진에 세력 전용 해상 절차를 넣지 않는다.
- 거리 0, 장전, 독, 밀치기, 보조 무기 같은 미지원 요소는 규칙 승인을 거치기 전 데이터로 확정하지 않는다.

## 5. 무공 콘텐츠 단위 가설

T1 이후 고유 행동을 추가할 때 다음 단위를 우선 검토한다.

```text
플레이 스타일의 대표 무공 1종
+ 안정적인 공용 무공 1종
+ 조건부 고위험 무공 1종
```

모든 무공은 가능한 한 현행 공용 계약을 재사용한다.

- 행동 슬롯.
- 기력·내력 비용.
- 사거리·이동 거리.
- 피해·회복.
- 막기·회피·태세 결합.
- 절초 기세.
- `[속공]`, `[강건]`, `[검법]`, `[보법]` 같은 태그.

새 무공 하나 때문에 별도 라운드·별도 전장·별도 판정 순서를 만들지 않는다.

## 6. 심법·성장 콘텐츠 가설

- 심법은 패시브·자원 순환·기초 행동 변형을 담당한다.
- 한 회차 최대 2개 보유 방향은 가설로 유지한다.
- 핵심무공·심법의 성급과 절초·진의는 T1 전투→보상→성장 연결이 통과한 뒤 확정한다.
- 현재 T0에는 세력 선택, 무공 성급, 심법과 성장 해금형 절초가 연결되지 않았다. 공용 절초 3종은 현재 전투에 구현돼 있다.

세부 가설은 `docs/06_STARTING_FACTION_MASTERY_DATA.md`가 책임진다.

## 7. 콘텐츠 추가 게이트

새 콘텐츠는 다음 질문을 통과해야 한다.

1. T0의 어떤 핵심 판단을 강화하는가?
2. 기존 8개 행동과 다른 선택을 만드는가?
3. 행동 슬롯·기력·내력·거리·대응 계약으로 표현 가능한가?
4. AI가 비공개 정보 치팅 없이 사용할 수 있는가?
5. 실패 이유를 로그로 설명할 수 있는가?
6. 접근성 폴백에서 같은 정보를 전달할 수 있는가?
7. 정상·실패·경계 테스트를 추가할 수 있는가?

위 질문에 답하지 못하는 콘텐츠는 T1 범위에 넣지 않는다.
