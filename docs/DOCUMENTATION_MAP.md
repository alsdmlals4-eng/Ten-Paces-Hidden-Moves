## 작업 시작 순서

```text
최신 사용자 지시
→ AGENTS.md
→ 위 다섯 활성 기획서
→ 현재 작업과 연결된 상세 명세·부록
→ 실제 코드·씬·데이터·자산·테스트
```

- 과거 대화 없이 저장소만으로 현재 방향을 복원한다.
- `[백업]`은 과거 기록, `[보류]`는 재승인 전 작업 금지 영역이다.
- 구현·자산·테스트를 확인하지 않은 상태는 완료로 추정하지 않는다.

## 문서 책임 원칙

- 다섯 활성 기획서는 사용자와 작업자의 진입점이다.
- 상세 명세는 특정 수치·데이터·구현 계약·테스트를 증명한다.
- 같은 내용을 여러 활성 문서에 복사하지 않고 책임 문서를 링크한다.
- 기존 `APPROVED_*` 또는 세부 문서는 별도 활성 기획서가 아니라 부록으로 취급한다.
- 중복 내용은 다섯 활성 기획서로 흡수하고, 고유 계약이 없는 문서만 참조 검사 후 아카이브한다.

## 모든 작업의 갱신 규칙

- 규칙·콘텐츠·UI/UX·밸런스 변경 → 게임 기획서.
- 엔진·구현·데이터·로드맵·MVP 변경 → 프로그래밍 기획서.
- 캐릭터·배경·카드·아이콘·VFX·이미지 승인 변경 → 아트 기획서와 이미지 인덱스.
- BGM·SFX·보이스·오디오 이벤트 변경 → 사운드 기획서.
- 테스트·일정·범위·위험·승인·인수인계 변경 → QA·PM 기획서.

복수 분야에 영향이 있으면 같은 작업과 같은 PR에서 관련 문서를 모두 갱신한다.

## 문서 품질 기준

- 사용자가 빠르게 이해할 수 있도록 제목·표·짧은 문단을 사용한다.
- 새 채팅·새 작업자가 문서만 보고 같은 방향과 품질로 결과물을 만들 수 있을 만큼 상세하게 작성한다.
- 확정·진행 중·검증 필요·보류를 구분한다.
- 각 활성 기획서에는 현재 상태, 품질 기준, 금지 사항, 다음 작업, 상세 부록 링크가 있어야 한다.
- 최신 이미지의 책임 원본은 아트 기획서와 이미지 인덱스다.

## 완료 검사

```bash
python scripts/check_planning_sync.py
python tests/static_check.py
godot --headless --path . --script res://tests/run_all.gd
```

Godot 실행 환경이 없으면 앞의 두 검사를 수행하고 런타임 미검증을 기록한다.
