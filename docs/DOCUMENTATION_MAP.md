## 우선순위

문서 간 충돌 시 다음 순서를 따른다.

1. 사용자 최신 확정 결정
2. 저장소 루트의 `AGENTS.md`
3. `docs/BASE_RULES_VERSION.md`가 존재하면 해당 버전 정보
4. 이 문서 지도
5. 전투 규칙과 전투 시스템 아키텍처
6. 대회 세로 슬라이스 명세
7. 핵심무공·심법 데이터와 테스트 체크리스트
8. UI·아트 명세와 실제 구현 파일

## 기준 문서 갱신 규칙

- 확정된 기획 변경은 관련 분야의 기존 기준 파일에 반영한다.
- 활성 문서를 `v2`, `final`, `latest` 또는 날짜가 붙은 새 이름으로 복제하지 않는다.
- 이전 내용은 Git 이력으로 보존하며 별도 백업 파일을 새로 만들지 않는다.
- 기존 `docs/[백업]/`은 과거 기록 확인 외에는 읽거나 수정하지 않는다.
- 새 기준 문서에 통합된 이전 버전·중복 파일은 참조 경로를 확인한 뒤 제거한다.
- 기준 파일의 제목·경로·역할이 바뀌면 이 문서와 `README.md`의 연결을 확인한다.

## 보류 문서 규칙

- [보류 항목 인덱스]([보류]/README.md)는 아이디어 보존용이며 활성 기준이 아니다.
- Codex와 구현 에이전트는 사용자가 재개를 명시하지 않은 보류 항목을 구현하지 않는다.
- 재개가 확정된 항목은 관련 기준 문서에 통합한 뒤 별도 Plan으로 작업한다.

## 현재 핵심 동기화 범위

- 라운드: 10타이밍, 행동 두 개씩 비공개 잠금·동시 공개
- 공격 대 공격: 합 수치 비교와 차이 피해
- 핵심무공: 여러 개 보유, 3·6·9성 기술, 10성 절초, 10성 최대 1개
- 심법: 최대 2개 보유, 패시브·기믹·명상 강화, 10성 진의, 10성 최대 1개
- 수련: 경기 승리 기본 5포인트 + 성과 등급 확정·확률 보너스
- 제약: 삼수양보 등 무협식 선택 난이도와 승리 추가 보상
- 문파별 기믹 수행도 성과 평가는 `[보류]`
