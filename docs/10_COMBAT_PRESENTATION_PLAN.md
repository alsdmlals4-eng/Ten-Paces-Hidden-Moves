## 1. 연출 목표

십보강호의 연출은 화려함보다 다음 네 가지를 우선한다.

1. 지금 무엇을 보고 무엇을 선택해야 하는지 즉시 이해한다.
2. AI의 2수가 공정하게 잠겼고 결과가 규칙대로 해상됐음을 신뢰한다.
3. 거리·자원·합·성장 결과의 원인을 다음 판단에 사용할 수 있다.
4. 이명·풍문·초식·절초가 내부 데이터와 모순되지 않는 무협 경험으로 남는다.

```text
상대 읽기
→ 위험 선택
→ 두 수 예측
→ 동시 공개
→ 1수·2수 해상
→ 원인 확인
→ 성과·행운 정산
→ 수련과 해금
→ 다음 상대 또는 데모 결말
```

## 2. 데모 감정 곡선

| 구간 | 플레이어 감정 | 연출 역할 |
|---|---|---|
| 1전 내부전 | 낯섦보다 이해 | 전장·거리·카드·2수 잠금의 최소 규칙을 명확히 보여준다. |
| 2전 내부전 | 예측 성공의 쾌감 | 상대의 반복 성향과 합 결과를 읽게 한다. |
| 3전 예선 | 선택 폭의 확장 | 첫 공용·외문 무공 또는 보조장비와 상대 풍문을 연결한다. |
| 4전 예선 | 성장 압박과 기대 | 예선 결승 전 남은 수련과 절초 가능성을 강조한다. |
| 5전 예선 결승 | 축적한 판단의 하이라이트 | 상대 정보·제약·빌드·합·절초를 종합하고 데모의 끝을 보상한다. |

절초는 데모 진행 필수가 아니다. 개방한 회차에서는 5전의 하이라이트가 되고, 개방하지 못한 회차도 정상적인 빌드와 판단으로 결승을 완주할 수 있어야 한다.

## 3. 공통 연출 원칙

- 한 화면에는 하나의 첫 시선과 하나의 중심 질문을 둔다.
- 첫 1초에는 현재 단계와 가장 중요한 대상이 읽혀야 한다.
- 선택 전에 필요한 정보는 즉시 보이고 계산 상세는 호버·클릭·로그로 단계화한다.
- 연출은 전투·성장 결과를 표현하지만 새로 계산하거나 변경하지 않는다.
- 같은 내부 의미는 같은 아이콘·문구·음향 계열을 반복한다.
- 강한 상대를 약한 문구로 속이지 않는다.
- 연출을 줄이거나 자산이 누락돼도 입력, 판정, 저장 결과는 유지된다.
- 정확한 지속 시간은 실제 플레이에서 검증하며 연출 때문에 다음 입력이 불필요하게 지연되지 않게 한다.
- 화면 흔들림·점멸·색상·음향은 보조 신호이며 텍스트·아이콘·형태 신호를 함께 제공한다.

## 4. 상태 소유 경계

| 계층 | 책임 | 금지 |
|---|---|---|
| 전투·회차 도메인 | 위치, 자원, 행동 잠금, 공격 성립, 합, 피해, 성과, 행운, 수련, 저장 | 화면 재생 완료 여부로 결과를 바꾸지 않음 |
| 구조화 이벤트 | `pair_locked`, `pair_revealed`, `move`, `guard`, `attack`, `clash`, `hit`, `resource`, `invalid_action` 등 원인과 결과 전달 | 장면 전용 임의 수치 추가 금지 |
| 프레젠테이션 | 자산 선택, 카드 강조, 로그, 카메라·VFX·음향, 입력 잠금, 폴백 | 피해·보상·수련·저장 직접 변경 금지 |
| UI 입력 | 카드 선택, 1수·2수 배치, 잠금, 상세 열기, 확인·되돌리기 | AI 행동 열람, 잠금 뒤 행동 변경 금지 |

컷인 종료 콜백이나 애니메이션 이벤트에서 피해·보상·수련을 지급하지 않는다. 도메인이 확정한 결과를 구조화 이벤트로 받아 표현한다.

## 5. 대회·상대 정보 모듈

### 5.1 중심 질문

> 이 상대가 어떤 전법과 성장 상태를 가졌으며, 어느 정도의 제약을 감수할 것인가?

### 5.2 시선 순서

```text
현재 경기·대회 단계
→ 상대 이름·문파·이명
→ 풍문·최근 전적
→ 알려진 무기·선호 거리·무공 평가
→ 정탐 정보
→ 제약 후보와 보상
→ 전투 준비 확정
```

### 5.3 표시 계약

- 개발용 태그인 `[강적]`, `[A++]`, `[핵심무공 올인형]`은 기본 UI에 노출하지 않는다.
- 이명은 종합 강도와 대표 전법을 암시한다.
- 풍문·전적은 성장세와 행운을 암시한다.
- 무공 평가는 균형형·핵심무공 올인형·심법 연계형·다무공형을 암시한다.
- 정탐 단계가 높아질수록 문장이 구체화되지만 원시 수치표를 그대로 출력하지 않는다.
- 문구는 내부 프로필의 의미 키에서 선택하며 실제 데이터와 모순될 수 없다.
- 제약 카드는 상대 정보보다 먼저 시선을 빼앗지 않는다.
- 제약의 무협식 이름 아래에 현대적인 규칙 설명, 위험도, 승리 보상을 함께 표시한다.

### 5.4 즉시 반응

- 정탐 정보가 추가되면 새 문장만 강조하고 전체 패널을 반복해서 흔들지 않는다.
- 제약 선택 시 전투 시작 상태에서 달라질 자원·거리·행동 제한을 미리 보여준다.
- 선택한 제약이 문파 핵심 기믹 전체를 봉쇄할 가능성이 있으면 확정할 수 없게 한다.
- 상대 프로필은 회차 최초 생성 시 고정하며 재도전과 저장·불러오기에서 같은 문구·성장이 유지된다.

## 6. 전투 기본 화면 모듈

### 6.1 중심 질문

> 공개된 거리와 자원에서 어떤 두 수를 잠가야 하는가?

### 6.2 첫 시선과 정보 순서

```text
양측 위치와 현재 거리
→ 체력·행동력·기력·내공
→ 현재 타이밍과 2수 묶음
→ 1수·2수 슬롯
→ 사용 가능한 무공 카드
→ 상세 효과와 결과 로그
```

### 6.3 카드 정보 층

#### 카드에 상시 표시

- 문파 또는 `공용` 배지.
- 무공 이름.
- 권법·검법·심법·창법 등 종류 배지.
- 사거리.
- 타이밍·행동력·기력·내공 비용.
- 선택·잠금·사용 불가 상태.

#### 마우스 오버 팝업

- 대상과 실제 사거리 설명.
- 합 수치와 일반 피해의 핵심 요약.
- 조건, 상태, 장전·회수·연격 등 즉시 판단 정보.
- 사용 불가의 구체적 이유.

#### 클릭 고정 상세

- 전체 효과와 계산 구성.
- 3·6·9성 기술 및 10성 절초·진의.
- 관련 심법·기믹 수정.
- 다음 행동 또는 2수 묶음에 미치는 영향.

호버 팝업은 빠른 판단, 클릭 상세는 학습과 비교를 담당한다. 같은 전문을 카드 본체와 두 상세 영역에 반복하지 않는다.

## 7. 2수 선택·잠금·공개 모듈

### 7.1 선택 단계

1. 플레이어가 1수 슬롯에 행동을 배치한다.
2. 첫 수가 정상 실행된다는 가정으로 위치·자원 변화를 미리 계산한다.
3. 플레이어가 2수 슬롯에 행동을 배치한다.
4. 첫 수 실패 시 둘째 수가 무효가 될 수 있으면 조건부 경고를 표시한다.
5. 두 슬롯이 모두 형식상 유효할 때만 잠금 버튼을 활성화한다.

### 7.2 AI 공정성 표시

- AI는 플레이어 선택 전에 현재 공개 정보만으로 두 수를 잠근다.
- 잠금 완료 사실은 자물쇠 아이콘과 짧은 설명으로 표시한다.
- AI 카드의 내용은 양측 잠금이 완료되기 전까지 공개하지 않는다.
- 플레이어 잠금 뒤에는 양측 모두 행동을 변경할 수 없다.
- 재도전에서도 상대 성장 프로필은 유지하지만 각 2수는 현재 공개 상태로 새로 잠근다.

### 7.3 공개 단계

```text
양측 잠금 확인
→ AI·플레이어 1수·2수 동시 공개
→ 1수 강조
→ 1수 해상과 결과 요약
→ 2수 강조
→ 2수 해상과 결과 요약
→ 다음 묶음으로 이동
```

두 행동을 먼저 함께 공개해 예측의 적중과 실패를 확인하게 하고, 해상은 1수와 2수를 분리해 원인을 읽게 한다.

## 8. 행동 해상 모듈

한 타이밍의 표현 순서는 규칙의 해상 순서와 일치한다.

```text
행동 가능 여부·비용
→ 준비·보조장비
→ 이동
→ 막기·회피·호신
→ 공격·밀치기 성립
→ 합 또는 일반 공격
→ 피해·상태
→ 장전·회수·명상·자원 회복
```

- 무효 행동은 사라지지 않고 `왜 실패했는지`를 카드와 로그에 표시한다.
- 이동 후 사거리로 공격 성립 여부를 판정했음을 위치 변화와 함께 보여준다.
- 행동력·기력·내공 차감은 해당 행동이 강조될 때 함께 반영한다.
- 상태 부여와 해제는 이름, 중첩, 지속 조건을 아이콘과 로그로 동시에 제공한다.
- 음향이나 색만으로 성공·실패를 전달하지 않는다.

## 9. 합 연출 모듈

### 9.1 발생 조건

같은 타이밍에 양측 공격이 모두 유효하고 서로 사거리 안일 때만 합 연출을 사용한다. 한쪽 공격이 무효이거나 공격 대 막기·회피인 경우 합처럼 보이게 하지 않는다.

### 9.2 연출 흐름

```text
양측 공격 진입
→ 짧은 공방 또는 무기 충돌
→ 최종 합 구성 표시
→ 승패 또는 동률
→ 차이 피해 적용
→ 다음 상태 복귀
```

- 여러 차례 공방은 한 번의 계산을 표현하는 시각 연출이다.
- 로그는 `기본 합 → 무기 → 성급 → 심법·기믹 → 반격 → 최종 합 → 차이 피해` 순서다.
- 무당 태극권 반격은 상대 공격 수치가 실제로 참조됐음을 별도 항목으로 표시한다.
- 동률은 `합 무승부 / 체력 피해 없음`으로 명시한다.
- 일반 공격과 합은 충돌음, 결과 표제, 로그 구조가 구분돼야 한다.

### 9.3 접근성 폴백

- 화면 흔들림을 끄면 충돌 지점 테두리, 짧은 정지 프레임, 결과 문구로 대체한다.
- 점멸 감소 시 강한 플래시 대신 명도 변화와 아이콘을 사용한다.
- 연출 건너뛰기에서도 최종 합 구성과 피해 원인은 확인할 수 있다.

## 10. 절초·진의·문파 기믹 모듈

- 절초는 신규 해금, 선택, 발동을 서로 다른 순간으로 표시한다.
- 일반 초식보다 긴 연출을 사용할 수 있지만 결과 확인과 다음 입력을 과도하게 늦추지 않는다.
- 진의는 추가 카드나 별도 버튼이 아니라 상시 운용 변화이므로 발동 조건을 만족한 순간에 짧은 표식과 로그로 설명한다.
- 연격·기세·독·반격·마화 등은 같은 의미 키와 아이콘을 반복해 학습 가능하게 한다.
- 문파 기믹 연출은 공용 이벤트를 꾸미며 전용 해상 순서를 만들지 않는다.
- 자산이 없으면 공용 공격·상태 표현과 텍스트 로그로 폴백한다.

## 11. 성과·행운·수련 모듈

### 11.1 중심 질문

> 이번 승리로 무엇을 얻었고, 어느 무공에 투자할 것인가?

### 11.2 정산 순서

```text
승리 기본 +5
→ 성과 등급 보너스
→ 비공개 행운 앞면 수
→ 제약 승리 보상
→ 최종 수련 포인트
→ 무공·심법 분배
→ 성급 상승과 해금
```

- 결과는 `B++`처럼 등급과 행운 `+`를 함께 표시한다.
- 개별 행운 다이스 면은 보여주지 않는다.
- 기본, 등급, 행운, 제약 보상을 한 숫자로 합치기 전에 각각 설명한다.
- 분배 전 현재 성급, 다음 필요 포인트, 부분 수련, 3·6·9·10성 해금을 미리 보여준다.
- 분배 확정 전 되돌리기를 제공한다.
- 10성 제한 때문에 투자할 수 없으면 제한 종류와 가능한 대안을 설명한다.

### 11.3 절초 해금

- 4전 종료 후 핵심무공이 10성이 되면 5전 준비 화면에서 신규 절초를 강조한다.
- 절초를 얻지 못했어도 예선 결승 진입이 정상 경로임을 명확히 한다.
- 절초 해금은 보상 순간을 강조하되 즉시 강제 전투로 넘어가지 않고 카드 상세와 비용을 확인할 시간을 준다.

## 12. 승리·패배·데모 종료 모듈

### 12.1 일반 승리

```text
마지막 판정
→ 승리 상태 확정
→ 짧은 승리 정체성 연출
→ 성과·행운 정산
→ 수련
→ 다음 상대 예고
```

작은 성공마다 전체 흐름을 멈추지 않고, 결투 종료와 대회 단계 종료를 구분한다.

### 12.2 패배와 재도전

- 패배 원인 요약은 마지막 피해만이 아니라 거리 실패, 자원 고갈, 무효 행동, 합 패배의 주요 원인을 보여준다.
- 재도전 시 같은 상대의 성장·이명·풍문·수련 스타일이 유지됨을 안내한다.
- 재기 보상과 명망 소모 여부를 전투 재진입 전에 확인한다.
- 재도전을 선택하지 않아도 결과와 배운 정보를 확인할 수 있다.

### 12.3 5전 예선 결승

- 경기 전에는 지금까지의 대회 경로와 상대 정보를 짧게 회수한다.
- 절초 보유 시 신규 하이라이트를 강조하되 사용을 강제하지 않는다.
- 결투 종료 뒤 `예선 우승`과 `데모 종료`를 분리해 표시한다.
- 전체판의 6~10전은 기능 목록을 과도하게 나열하지 않고 본선·8강·준결승·결승으로 이어지는 목표를 보여준다.

## 13. 카메라·VFX·오디오 예산

- 기본 전투 카메라는 10칸 전장과 양측 상태를 읽는 것을 우선한다.
- 일반 행동은 카메라 이동보다 대상 강조와 짧은 접촉 반응을 우선한다.
- 전체 화면 흔들림은 합, 강한 피격, 절초 같은 높은 우선순위에 제한한다.
- 같은 타이밍에 여러 효과가 겹치면 판정 중요도 순으로 VFX와 음향을 줄인다.
- 공격 전조, 합, 상태이상, 해금은 서로 다른 형태·리듬을 사용한다.
- 배경음은 대회 단계와 결투 긴장을 보조하되 카드 선택과 결과 음향을 가리지 않는다.
- 음향 자산이 없어도 텍스트·아이콘·모션만으로 상태를 이해할 수 있어야 한다.

## 14. 접근성·폴백

- 모션 감소: 큰 이동·확대 대신 대상 테두리와 단계 전환을 사용한다.
- 점멸 감소: 강한 플래시 대신 명도 변화·아이콘·짧은 정지로 대체한다.
- 색상 독립: 모든 상태는 아이콘·문구·형태 차이를 함께 가진다.
- 읽기 시간: 자동 진행되는 결과 문구는 사용자가 확인하거나 로그에서 재열람할 수 있다.
- 입력: 키보드·마우스 포커스가 카드, 1수, 2수, 잠금, 상세, 확인 순서로 이동한다.
- 자산 누락: 기본 초상·기본 공격·기본 상태 아이콘·텍스트 로그로 결정론적으로 폴백한다.

## 15. 데이터·자산 계약

연출 데이터는 원시 수치 대신 의미 키와 구조화 이벤트를 사용한다.

```text
presentation_event_id
source_actor_id
target_actor_id
action_id
result_type
reason_keys
magnitude_summary
priority
camera_request
vfx_key
audio_key
text_key
fallback_key
```

- 실제 필드명과 Resource 구조는 구현 Plan Mode에서 저장소 구조를 확인한 뒤 확정한다.
- `result_type`, `reason_keys`, 수치 요약은 전투 도메인이 제공한다.
- 카메라·VFX·오디오 키는 결과를 변경하지 않는다.
- 자산 레지스트리는 누락 시 폴백 키를 가진다.
- 이명·풍문·정탐 문구 키는 상대 성장 프로필과 함께 저장한다.

## 16. 제작 우선순위

| 티어 | 대상 | 목적 |
|---|---|---|
| T0 | 2수 잠금·공개, 이동, 일반 공격, 합, 피해, 자원, 결과 로그 | 핵심 규칙 이해와 공정성 검증 |
| T1 | 상대 정보·제약, 성과·행운, 수련, 절초 해금, 승패·데모 종료 | 데모 전체 흐름과 보상 검증 |
| T2 | 문파 기믹별 VFX·아이콘·음향, 결승 전용 변주 | 문파 정체성과 반복성 강화 |
| T3 | 고급 컷인, 추가 카메라, 다수 배경·관중 변주 | 핵심 검증 뒤 품질 확장 |

T0·T1이 작동하지 않은 상태에서 T2·T3 자산을 대량 제작하지 않는다.

## 17. 구현 전 남은 결정

다음 항목은 구조는 승인됐지만 정확한 값이 미확정이다.

1. 일반 행동·합·절초의 목표 지속 시간과 건너뛰기 규칙.
2. 성과 C/B/A/S의 공용 점수 지표와 임계값.
3. 상대 강도 5단계별 출현 분포와 어휘 사전.
4. 이명·풍문·전적·정탐의 의미 키와 문장 후보.
5. 수련 스타일별 투자 알고리즘과 성급 경계 우선순위.
6. 제약 후보 수, 선택 상한, 위험도와 추가 보상.
7. 행동력·기력·내공의 정확한 카드 비용과 라운드 회복량.
8. 1수 결과로 전투원이 쓰러졌을 때 2수 취소·표현 규칙.
9. 라운드 상한과 제한 시간 종료 시 승패 판정.
10. 최소 해상도, 목표 UI 배율, 키보드·패드 입력 범위.
11. 절초·문파 기믹별 전용 자산 수와 공용 폴백 범위.
12. 데모 종료 뒤 제공할 전체판 예고와 피드백 수집 항목.

미확정 값을 연출 시안의 임시 수치로 확정하지 않는다.

## 18. 향후 로드맵

### 단계 P0 — 연출 계약 고정

- 이 문서, 전투 규칙, UI 명세, 아키텍처의 책임 경계를 일치시킨다.
- 구조화 이벤트와 폴백 요구를 Codex Plan에 반영한다.
- 이명·풍문·전적·정탐의 첫 의미 키 표본을 작성한다.
- 일반 행동, 합, 절초의 저충실도 스토리보드를 만든다.

**종료 기준**

- 구현자가 결과를 다시 계산하지 않고 이벤트만 표현하는 구조를 설명할 수 있다.
- 5전 데모의 모든 화면이 한 개 이상의 중심 질문과 다음 입력을 가진다.

### 단계 P1 — T0 전투 연출 PoC

- 10칸 전장, 2수 잠금·동시 공개, 1수→2수 해상 UI를 구현한다.
- 이동, 일반 공격, 무효 행동, 합, 피해, 자원 변화와 로그를 연결한다.
- 화면 흔들림·점멸 감소 폴백을 함께 검증한다.

**종료 기준**

- 플레이어가 한 묶음의 예측 성공·실패 이유를 로그 없이도 말할 수 있다.
- 로그를 열면 합과 피해 수치의 구성 원인을 확인할 수 있다.

### 단계 P2 — T1 대회·성장 연출 PoC

- 상대 정보→제약→전투→성과·행운→수련 흐름을 연결한다.
- 상대 내부 프로필과 무림식 문구의 일치 여부를 검증한다.
- 4전 종료 절초 해금과 5전 예선 결승 진입을 구현한다.
- 절초 유무 양쪽 경로로 데모를 완주한다.

**종료 기준**

- 플레이어가 상대 문구를 근거로 제약 선택 이유를 설명한다.
- `A++` 표기와 실제 포인트 정산이 일치한다.
- 절초가 없어도 데모가 막히지 않는다.

### 단계 P3 — 문파 정체성·품질 확장

- 시작 6세력의 대표 기믹 의미 키, VFX, 아이콘, 음향을 제작한다.
- 합·절초의 반복 피로와 카메라 예산을 조정한다.
- 추가 6세력과 공용 무공에 같은 계약을 확장한다.

**종료 기준**

- 문파 전용 해상 분기 없이 공용 이벤트와 수정자로 표현된다.
- 자산을 제거해도 공용 폴백으로 전투와 저장이 유지된다.

### 단계 P4 — 전체판 10전 확장

- 6~7전 본선, 8강, 준결승, 10전 결승의 연출 밀도를 단계화한다.
- 장기 반복에서 합·절초·결과 화면의 피로를 측정한다.
- 전체판 결승과 회차 종료 연출을 추가한다.

**종료 기준**

- 데모용 연출을 복제하지 않고 같은 모듈과 데이터로 10전까지 확장한다.
- 결승이 일반전과 구분되지만 결과 규칙과 입력 계약은 유지된다.
