# 십보강호 전투 연출 기획서

> 책임: 현재 T0 전투의 시선 흐름·계획·수별 판정·합·중단·절초·종료 연출과 접근성 폴백  
> 규칙 원본: `docs/02_COMBAT_RULES.md`  
> UI 원본: `docs/07_COMBAT_UI_SPEC.md`  
> 검증: `docs/08_TEST_CHECKLIST.md`

## 1. 연출 목표

전투 연출은 화려함보다 다음을 우선한다.

1. 지금 무엇을 선택해야 하는지 이해한다.
2. 10칸·4/7·거리 3과 `[밀착]`을 즉시 읽는다.
3. 자신이 배치한 수와 실제 판정의 관계를 신뢰한다.
4. 합·방어·회피·필중·중단·기세의 이유를 다음 계획에 사용한다.
5. 계산을 방해하지 않으면서 무협 공방의 손맛을 제공한다.
6. 자산·모션·음향이 없거나 줄어도 같은 결과를 읽는다.
7. 빠른 재생·즉시 완료가 결과를 바꾸지 않는다.

## 2. 상태 구분

- `CURRENT`: PR #7에서 실제 표현한다.
- `TECHNICALLY_VERIFIED`: 자동·Godot·Windows 기술 증거가 있다.
- `HUMAN_NOT_RUN`: 주관적 읽기성·이해·선호 검증 전이다.
- `T1_LATER`: 현재 T0에 넣지 않는다.

## 3. 첫 시선

전투 진입 뒤 권장 시선 순서:

```text
중앙 10칸 전장과 전투원
→ 4번·7번과 거리 3
→ 양측 체력·기력·내력·기세·상태
→ 플레이어 절초 목록
→ 라운드·묶음·판정 순서
→ 현재 수 슬롯
→ 기초 행동 8종
```

- 전장과 전투원이 카드 상세·로그보다 먼저 읽힌다.
- 발 앵커와 타일 번호가 위치 판독을 책임진다.
- 같은 칸에서는 좌우로 약간 분리하고 `[밀착]` 상태와 로그를 함께 표시한다.
- 수묵 석양 배경은 타일·전투원·금빛 절초 강조보다 낮은 대비를 유지한다.

## 4. 현재 전투 흐름

```text
4/7·거리 3과 자원 확인
→ 현재 묶음 행동·절초 배치
→ 이동 목적지·공격 방향 지정
→ 예상 기력·내력·기세 예약 확인
→ 진행
→ 대응 준비
→ 각 수의 준비·속공·합·이동·일반 행동 순차 재생
→ 피해·자원·기세·중단·상태 반영
→ 다음 묶음 또는 combat_ended
→ 필요 시 결전 다시 시작
```

엔진은 묶음을 한 번 계산한다. 화면은 반환된 `timing_results`와 `presentation_events`만 재생한다.

## 5. 상태 소유 경계

| 계층 | 책임 | 금지 |
|---|---|---|
| 전투 엔진 | 위치·비용·합·피해·방어·상태·기세·중단 | 모션 완료 여부로 결과 변경 |
| AI | 공개 상태 기반 상대 행동 | 플레이어 미확정 계획 읽기 |
| 판정 반환 | 확정 state·수별 snapshot·event·log | 장면용 임의 피해 추가 |
| UI | 입력·예상값·확정 결과 표현 | 피해·승패·보상 재계산 |
| VFX·애니메이션·오디오 | 확정 event 재생 | 상태·기세·보상 직접 지급 |

## 6. 계획 단계 표현

### 6.1 카드·절초 선택

- 카드 호버는 임시 상세, 클릭은 고정 상세와 선택을 제공한다.
- 절초 3종은 기세 게이지 아래 planning 중 항상 보인다.
- 기세·빈 슬롯·입력 잠금에 따른 비활성 이유를 텍스트로 표시한다.
- 포커스·선택·고정 상세를 서로 다른 테두리·상태로 구분한다.

### 6.2 슬롯 배치

- 1슬롯 행동은 한 수에 이름을 표시한다.
- 다중 슬롯 앞 수는 `[준비]`, 마지막 수는 `[실행]`이다.
- 절초는 `[절초 예약]`과 즉시 소비된 기세를 표시한다.
- 중단·전투 불능은 점유 슬롯과 원인을 함께 표시한다.
- 묶음 경계·점유 충돌·자원 부족은 배치하지 않거나 진행을 막고 이유를 남긴다.

### 6.3 대상 지정

- 이동·보법: 녹색 목적지와 타일 번호.
- 공격·절초: 붉은 좌·우 방향과 문구.
- `[밀착]`: 방향과 무관한 유효 사거리임을 결과에서 설명한다.
- 정지 상대 칸과 공동 목적지는 유효 후보가 될 수 있다.
- 자리 교환·통과는 비활성 이유를 제공한다.

### 6.4 자원·기세 예상

- 배치 즉시 예상 기력·내력을 갱신한다.
- 명상 회복은 실행 수 순서로 반영한다.
- 절초 예약은 기세 5→0을 즉시 표시한다.
- `[진행]` 전 예약 취소는 0→5 복원을 즉시 표시한다.
- 예상값과 확정값을 색·문구·상태로 구분한다.

## 7. 진행과 입력 잠금

표현 상태:

```text
planning
→ committed
→ resolving
→ presenting_result
→ next_bundle_ready
```

종료 시:

```text
combat_ended
```

- resolving 동안 카드·절초·슬롯·대상·진행을 잠근다.
- 현재 수의 state를 적용한 뒤 해당 event를 재생한다.
- 다음 묶음 준비 전 입력을 열지 않는다.
- 종료 뒤 재시작 외 계획 입력을 잠근다.

## 8. 수별 판정 표현

표시 순서:

```text
대응 준비
→ 현재 수 준비 이벤트
→ 속공 위치·유효성
→ 같은 수 공격군의 합 또는 단독 공격
→ 회피·필중·방어·최종 피해
→ 중단·전투 불능
→ 이동
→ 명상·단독 태세 등 일반 행동
→ 자원·기세·상태 결과
```

UI가 이 순서를 이용해 판정을 다시 계산하지 않는다.

## 9. 합 연출

합은 한쪽 선공이 상대를 취소한 장면처럼 보이면 안 된다.

표시 요소:

- 같은 실행 수의 양측 행동 이름.
- 양측 원공격력.
- 합 승리·패배·동점 상쇄.
- 차이 원피해.
- 회피·필중·방어 적용.
- 최종 피해와 기세 획득.
- 정산 뒤 중단·전투 불능.

권장 요약:

```text
합 상쇄 · 양측 피해 없음
합 승리 · 차이 피해 2
필중 · 회피 불가 · 방어 후 피해 3
```

## 10. 대응·필중 연출

### 막기

```text
원피해
→ 방어도 4 또는 6 차감
→ 같은 수면 남은 피해 반감
→ 최종 피해
```

감소량 하나만 보여 주지 않고 적용 순서를 설명한다.

### 회피

- 같은 수 회피 또는 태세+회피의 묶음 범위를 표시한다.
- 피해 0과 성공 기세 획득을 구분한다.

### 필중

- 파공검기의 `[필중]` 표식을 표시한다.
- 회피만 무시하며 방어·합·거리·중단은 유지됨을 결과 문구로 설명한다.

## 11. 중단·강건 연출

중단 표현 순서:

1. 해당 수의 준비·실행 예정 행동.
2. 실제 피해.
3. 같은 수의 아직 실행 전 행동 중단.
4. 중단 대상 슬롯과 이유.
5. 이후 수 계획 유지 또는 전투 불능 취소.

강건:

- 태세 실행 뒤 금빛 `[강건]` 상태를 표시한다.
- 속공 단계 중단을 막았으면 `피해는 받음 / 중단만 방지`를 분리한다.
- 보호 소비 뒤 상태를 즉시 제거한다.
- 2·3슬롯 행동과 전투 불능을 보호하지 않는다는 결과를 설명한다.

## 12. 절초 연출

- 절초 목록은 planning 중 상시 표시한다.
- 예약은 슬롯 연결·이름·방향·기세 0으로 표현한다.
- `[진행]` 전 취소는 예약 해제·기세 5 복원으로 표현한다.
- committed 이후 실패·중단에는 환불되지 않았음을 로그에 남긴다.
- 수묵·금빛 atlas를 사용하고 일반 공격보다 긴 최대 0.70초 강조를 제공한다.
- 파공검기의 필중은 별도 텍스트·아이콘으로 표시한다.
- 모션 감소에서도 결과 라벨·로그·상태는 유지한다.

## 13. 이동·밀착 연출

- 출발·도착 칸을 먼저 강조한다.
- 유효 이동만 실제 위치로 애니메이션한다.
- 실패 이동을 이동했다가 되돌리는 방식으로 표현하지 않는다.
- 공동 목적지는 양측이 같은 수에 같은 칸으로 이동했음을 한 묶음으로 보여 준다.
- 밀착 시 전투원 이미지를 약간 좌우 분리해 발 앵커·공동 타일을 동시에 읽게 한다.
- 자리 교환·통과 실패는 원인 문구를 제공한다.

## 14. 종료·재시작 연출

- 체력 0 뒤 `combat_ended`와 승리·패배·무승부를 표시한다.
- 추가 계획 입력을 잠근다.
- `결전 다시 시작`을 표시한다.
- 재시작은 4/7·30/5/4/0~5·라운드 1·planning으로 복원한다.
- 태세·강건·예약·로그·VFX·SFX·입력 잠금이 남지 않아야 한다.
- 숨겨진 재시작 버튼은 포커스 순서를 가로채지 않는다.

## 15. 재생 옵션

### 빠른 재생

각 event의 대기를 줄이되 결과 순서를 유지한다.

### 즉시 완료

남은 event 대기를 다음 프레임 수준으로 끝내고 결과 state·로그를 그대로 적용한다.

### 모션 감소

- 이동·공격 tween과 scale pulse를 생략한다.
- 상태·텍스트·로그·VFX 의미를 유지한다.
- 입력 복구 시점과 결과는 일반 모드와 같다.

### 음향

- 합·피해·막기·회피·기세·절초·패배의 확정 event를 재생한다.
- 음소거 시 현재 재생을 즉시 정지한다.
- 음향 없이도 텍스트·아이콘으로 같은 정보를 전달한다.

## 16. 접근성 폴백

현재 기술 계약:

- 색 외에 텍스트·형태·아이콘·테두리를 사용한다.
- 카드→절초→슬롯→타일→진행→재생→재시작→옵션의 명시적 포커스 순서를 제공한다.
- Enter는 포인터와 같은 선택·배치·대상·취소 동작을 한다.
- 한국어 접근성 이름·설명을 제공한다.
- 긴 결과 문구와 960×640을 회귀 검사한다.
- 모션·음향이 없어도 결과를 읽을 수 있다.

실제 보조기기 사용자·주관적 음향/모션·색각·저시력 검증은 `HUMAN_NOT_RUN`이다.

## 17. 성능 원칙

- 연출은 판정 실행이나 입력 복구를 무기한 차단하지 않는다.
- 임시 효과 노드·로그·오디오 스트림을 무제한 누적하지 않는다.
- 일반·빠른·즉시·모션 감소 모드의 최종 결과가 같다.
- 대표 장면과 합·절초·다수 로그의 최악 장면을 분리한다.
- 평균 FPS만이 아니라 frame time·CPU·GPU·메모리·로딩을 목표 장치에서 측정한다.

DEBUG 표본은 있으나 Release 목표 사양은 `NOT_RUN`이다.
