# 십보강호 전투 연출 기획서

> 책임: 현재 T0 전투의 시선 흐름·행동 계획·판정·로그·접근성 폴백  
> 규칙 원본: `docs/02_COMBAT_RULES.md`  
> UI 원본: `docs/07_COMBAT_UI_SPEC.md`  
> 범위·검증: `docs/05_COMBAT_POC_SPEC.md`, `docs/08_TEST_CHECKLIST.md`

## 1. 연출 목표

십보강호의 전투 연출은 화려함보다 다음을 우선한다.

1. 지금 무엇을 보고 무엇을 선택해야 하는지 이해한다.
2. 자신이 배치한 수와 실제 판정의 관계를 신뢰한다.
3. 거리·방향·자원·대응·실패 원인을 다음 계획에 사용한다.
4. 전투 계산을 방해하지 않으면서 무협 공방의 손맛을 제공한다.
5. 자산·모션·음향이 없어도 같은 정보를 읽을 수 있다.

현재 T0 전투 흐름:

```text
양측 상태와 거리 확인
→ 현재 묶음의 행동 배치
→ 이동 목적지·공격 방향 지정
→ 예상 자원 확인
→ 진행
→ 대응 → 속공 → 이동 → 일반 공격
→ 수별 결과와 원인 확인
→ 다음 묶음 또는 다음 라운드
```

## 2. 상태 소유 경계

| 계층 | 책임 | 금지 |
|---|---|---|
| 전투 도메인 | 위치·비용·대응·피해·자원·라운드 | 애니메이션 완료 여부로 결과 변경 |
| 구조화 결과 | 행동·원인·전후 값·로그 키 | 장면 전용 임의 수치 추가 |
| UI | 입력·선택 가능 상태·예상 자원·로그 | 피해·회복·승패 재계산 |
| VFX·애니메이션·오디오 | 확정 결과의 시각·청각 표현 | 판정·보상·저장 직접 변경 |

연출 콜백에서 피해·기력·내력·기세를 지급하지 않는다.

## 3. 첫 시선

전투 화면 진입 후 첫 시선 순서:

```text
중앙 10칸 전장과 양측 위치
→ 상단 체력·기력·내력·절초 기세
→ 현재 라운드
→ 하단 현재 행동 묶음
→ 기초 행동 8종
```

- 전장과 캐릭터가 카드·로그보다 먼저 읽혀야 한다.
- 현재 편집 가능한 수와 미래 잠금 수를 즉시 구분한다.
- 로그와 상세 패널은 기본 전장을 가리지 않는 오버레이다.

## 4. 계획 단계

### 4.1 카드 선택

- 호버: 상세를 임시 표시한다.
- 클릭: 상세를 고정하고 배치할 행동으로 선택한다.
- 선택 카드는 테두리·명도·문구로 표시한다.
- 대응 결합 대기 상태는 일반 카드 선택과 구분한다.

### 4.2 슬롯 배치

- 빈 슬롯을 클릭하면 카드가 배치된다.
- 다중 슬롯은 연결된 점유 형태를 사용한다.
- 묶음 경계 초과는 배치하지 않고 이유를 기록한다.
- 점유 슬롯을 클릭하면 행동 전체를 해제한다.

### 4.3 대상 지정

이동:

```text
카드 배치
→ 녹색 이동 가능 칸 표시
→ 도착 칸 클릭
→ 슬롯에 방향·칸 번호 표시
```

공격:

```text
카드 배치
→ 붉은 좌·우 공격 방향 표시
→ 방향 클릭
→ 슬롯에 공격 방향 표시
```

- 대상 지정 중 다른 카드 선택으로 입력 상태가 꼬이지 않게 한다.
- 지정 완료 뒤 다음 미지정 행동으로 이동할 수 있다.
- 색만 사용하지 않고 아이콘·테두리·문구를 함께 쓴다.

### 4.4 자원 예상

- 카드 배치 즉시 기력·내력 예상값을 갱신한다.
- 명상 회복은 실제 실행 수 순서에 맞춰 예상한다.
- 부족 행동은 슬롯과 HUD 모두에 표시한다.
- 계획 예상값은 실제 자원과 시각적으로 구분한다.
- 행동 해제 시 즉시 복원한다.

## 5. 진행·입력 잠금

진행 가능 조건이 충족되기 전:

- 버튼 비활성
- `판정 전`
- 빈 슬롯·대상 미지정·자원 부족 이유 표시

진행 요청 후:

- `진행 요청됨`
- 현재 묶음 카드·슬롯·대상 입력 잠금
- 판정 중인 수 강조
- 중복 클릭 방지

판정 완료 후:

- 현재 묶음을 완료 상태로 표시
- 다음 묶음을 열거나 다음 라운드로 이동
- 새로운 계획 입력 허용

## 6. 판정 연출 순서

규칙 순서와 표현 순서를 일치시킨다.

```text
대응 준비
→ 속공
→ 이동
→ 일반 공격·일반 행동
→ 피해·자원·기세 결과
```

### 대응

- 막기: 방어 자세·푸른 표식·감소량 문구
- 회피: 잔상 또는 측면 이동 표현·`회피` 문구
- 태세+막기: 묶음 전체를 감싸는 강화 방어 표식
- 태세+회피: 묶음 전체 회피 상태 표식

### 속공

- 이동 전에 카드와 공격 방향을 강조한다.
- 적중·빗나감 이유를 짧게 표시한다.
- 속공 뒤 이동이 이어질 때 단계가 섞여 보이지 않게 한다.

### 이동

- 출발 칸과 도착 칸을 먼저 강조한다.
- 이동 실패 시 캐릭터를 억지로 이동시켰다가 되돌리지 않는다.
- 동일 목적지 충돌은 양측 목적지 표식과 실패 문구를 함께 보여준다.

### 일반 공격

- 이동 후 실제 위치를 기준으로 방향·사거리를 다시 보여준다.
- 같은 단계 쌍방 공격은 어느 한쪽이 먼저 취소된 것처럼 연출하지 않는다.
- 피해 적용을 동시에 표시하거나 최소한 동일 단계임을 명확히 한다.

현재 POC에는 별도 합 수치 비교·차이 피해가 없다. 합 전용 연출은 사용하지 않는다.

## 7. 결과 피드백

한 수의 요약은 다음 질문에 답해야 한다.

- 누가 어떤 행동을 했는가?
- 어느 판정 단계였는가?
- 목표 방향과 거리는 유효했는가?
- 막기·회피 중 무엇이 적용됐는가?
- 원피해와 최종 피해는 얼마인가?
- 위치·기력·내력·기세가 어떻게 변했는가?

권장 로그:

```text
[1라운드 1묶음 · 2수]
강호낭인이 오른쪽으로 속공했다.
상대는 거리 1에 있어 공격이 적중했다.
상대는 막기를 준비해 피해 6 중 4를 막았다.
최종 피해 2.
```

피할 로그:

```text
priority=20, event emitted, damage_result=2
```

내부 상세는 개발 로그로 분리한다.

## 8. 전투 기록 패널

- 기본적으로 접는다.
- `기록` 버튼으로 펼치고 접는다.
- 최신 수가 보이도록 자동 스크롤하되 사용자가 과거 로그를 읽는 동안 강제 이동하지 않는다.
- 라운드→묶음→수 구조로 묶는다.
- 행동 배치·해제 같은 계획 로그와 판정 로그를 구분한다.
- 원인 문구는 같은 규칙에 같은 용어를 반복한다.

향후 읽기 성공 표식 후보:

- 간파: 정확한 회피·대응
- 파훼: 준비 행동 중단
- 선점: 이동 후 위치 예측 공격
- 받아냄: 강한 공격을 막기로 감소
- 허초: 상대 대응을 낭비시킴

이 표식은 현재 구현 완료 항목이 아니며 STEP 14에서 필요성을 검증한다.

## 9. 카드·상세 연출

카드 본체:

- `[기초]`
- 이름
- 기능 배지
- 사거리
- 행동 슬롯·기력·내력
- 원화

상세 패널:

- 대상·방향
- 피해·이동량·방어도·회복
- 조건·태그
- 전체 효과
- 대응 결합 설명

카드 본체·호버·클릭 상세에 같은 전문을 반복하지 않는다.

## 10. STEP 11 중단 연출 예정

현재는 `PLANNED`다.

필요한 표현:

1. 준비 중인 행동과 남은 점유 수
2. 실제 피해 발생
3. 중단 대상 행동 강조
4. 취소 표식과 이유
5. 집중·강건에 의한 중단 방지
6. 사망·강제 제어 우선 처리

중단 연출은 이미 같은 단계에서 성립한 공격을 취소한 것처럼 보이면 안 된다.

## 11. 승패·재시작 예정

STEP 13에서 구현한다.

- 체력 0 직후 추가 계획 입력 잠금
- 승자·패자·종료 원인
- 마지막 수 요약
- 재시작 버튼
- 재시작 시 상태·슬롯·대상·로그 초기화

전체 대회 결과·보상·수련 화면은 T1 이후다.

## 12. 접근성 폴백

### 색

- 이동·공격·선택·잠금·부족을 색 외 아이콘·형태·문구로 구분한다.

### 모션

- 화면 흔들림과 섬광을 줄이는 경로를 둔다.
- 결과 이해에 필요하지 않은 긴 대기 시간을 제거한다.
- 모션이 꺼져도 판정 단계와 결과 문구가 유지된다.

### 입력

- 키보드·마우스 모두 지원하는 흐름을 목표로 한다.
- 포커스 위치를 항상 시각적으로 표시한다.
- 카드→슬롯→대상→진행→상세→로그를 순환할 수 있게 한다.

### 자산 누락

- 카드 원화가 없으면 기능 아이콘과 기본 도형을 표시한다.
- 배경이 없으면 전장과 캐릭터의 대비를 유지하는 단색 폴백을 사용한다.
- 음향이 없어도 텍스트·아이콘으로 같은 결과를 전달한다.

## 13. 성능 원칙

- UI 애니메이션이 판정 실행을 차단하지 않는다.
- 수별 효과 노드를 무제한 누적하지 않는다.
- 로그 행과 임시 강조 노드를 정리한다.
- 대표 장면과 동시 공격·다수 로그가 있는 최악 장면을 분리 측정한다.
- 목표 Windows 사양과 frame time 예산은 실제 프로파일 전까지 `NOT_RUN`이다.

## 15. T1 이후 대회 연출

T0 통과 뒤 별도 범위로 진행한다.

```text
상대 정보
→ 제약 선택
→ 전투
→ 성과·보상
→ 수련
→ 다음 상대
```

이명·풍문·정탐·절초·대회 결말은 현재 T0 화면의 완료 조건이 아니다.
