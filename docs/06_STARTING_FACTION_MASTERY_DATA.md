# 유파 무공·대성 데이터 구조

> 현재는 무공을 대량으로 만들지 않는다. PoC에서는 소림과 당문의 유파 규칙만 검증하고, 대성은 다른 유파 무공을 단계별로 해금하는 장기 성장 구조로 문서화한다.

## 1. PoC 유파 규칙

| 세력 | 유파 무공 | 바꾸는 공용 행동 | 효과 |
|---|---|---|---|
| 소림 | 금강호신 | 막기 | 막기 경감률 +15% |
| 당문 | 독침 | 공격, 암기벨트 | 암기 적중 시 독 1중첩 |

두 규칙 모두 새로운 행동 버튼을 만들지 않는다. 기존 공용 행동의 결과만 변형한다.

## 2. 대성 단계의 역할

| 단계 | 해금 원칙 | 예시 |
|---:|---|---|
| 1성 | 해당 유파의 기본 규칙 | 소림 막기 강화, 당문 공격 독 부여 |
| 2성 | 공용 행동 하나의 조건 완화 또는 비용 개선 | 막기 기력 비용 감소, 암기 재장전 비용 감소 |
| 3성 | 같은 유파의 보조 규칙 선택 | 권법 공격 뒤 기력 회복, 독 상태 적 기력 압박 |
| 4성 | 다른 공용 행동과의 연계 | 막기 뒤 접근 강화, 후퇴 뒤 암기 강화 |
| 5성·대성 | 다른 유파 무공 또는 고급 변형 선택지 해금 | 무당 보법, 오독 독공 등에서 하나 선택 |

대성은 피해 숫자를 크게 올리는 시스템보다 ‘사용 가능한 유파 규칙의 조합’을 넓히는 시스템이다.

## 3. 시작 6세력의 제품 방향

| 세력 | 공용 행동을 주로 바꾸는 영역 | 현재 상태 |
|---|---|---|
| 화산파 | 공격, 접근·후퇴, 회피 | 후속 구현 |
| 개방 | 밀치기, 막기, 공격 | 후속 구현 |
| 당문 | 공격, 재장전, 보조장비 | PoC 구현 |
| 소림 | 막기, 접근, 거리 0 공격 | PoC 구현 |
| 무당파 | 회피, 막기, 내공 사용 | 후속 구현 |
| 마교 | 공격, 접근, 체력 대가 | 후속 구현 |

## 4. 보류 원칙

- PoC가 끝나기 전에는 각 유파에 새로운 무공 3개 이상을 추가하지 않는다.
- 새 유파를 추가할 때는 먼저 ‘어느 공용 행동을 어떻게 바꾸는가’를 한 문장으로 정의한다.
- 새 무공이 기존 공용 행동을 취소하거나 예외 단계를 만들면 보류한다.
- 소림 대 당문 재대결의 재미가 확인되기 전에는 20개 이상의 무공 목록을 만들지 않는다.
