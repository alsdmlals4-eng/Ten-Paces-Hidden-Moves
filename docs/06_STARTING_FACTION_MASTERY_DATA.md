# 십보강호 세력·핵심무공·심법 성장 가설

> 책임: T1 이후 세력 정체성, 핵심무공·심법 성장 구조와 현행 전투 계약에 연결 가능한 수정자 후보  
> 현재 전투 규칙: `docs/02_COMBAT_RULES.md`  
> 콘텐츠 범위: `docs/03_CONTENT_CATALOG.md`  
> 단계 진입 조건: `docs/05_COMBAT_POC_SPEC.md`

## 1. 현재 상태

이 문서는 **T1 이후 가설 원본**이다.

- 현재 T0에는 세력 선택·핵심무공·심법·성급·진의와 성장 해금형 절초가 구현되지 않았다. 공용 절초 3종은 전투 POC에 구현돼 있다.
- 아래 수치·해금 단계·세력 수는 콘텐츠 제작 확정값이 아니다.
- T0 STEP 0~14가 통과하기 전 대규모 무공 데이터를 작성하지 않는다.
- 현행 전투 판정에 없는 자원·판정·행동 절차를 이미 구현된 것처럼 사용하지 않는다.

상태 표기:

- `HYPOTHESIS`: 제품 방향 후보.
- `PLANNED_T1`: 최소 세로 슬라이스에서 검증할 후보.
- `HOLD`: 재설계 승인 전 사용 금지.

## 2. 현행 전투 계약과 연결 원칙

세력·무공·심법은 우선 다음 공용 필드와 상태를 조합한다.

```text
action_slots
stamina_cost
internal_cost
range
move_range
damage
block
recovery
momentum
tags
status duration
response profile
next attack bonus
```

현행 용어:

- 행동 슬롯.
- 기력.
- 내력.
- 체력.
- 절초 기세.
- 이동·보법·막기·회피·속공·강공·명상·태세.

원칙:

- UI는 카드·상태·예상 자원을 표시하고 판정을 확정하지 않는다.
- 무공은 공용 `대응 → 속공 → 이동 → 일반 공격` 순서를 재사용한다.
- 세력 하나를 위해 별도 라운드 구조나 독립 판정 엔진을 만들지 않는다.
- 새 필드는 데이터·저장·AI·로그·UI·테스트 소비자를 함께 설계한 뒤 도입한다.

## 3. T1 최소 성장 실험

T1에서는 전체 성급 체계를 먼저 구현하지 않는다.

권장 최소 실험:

| 영역 | 최소 범위 |
|---|---|
| 플레이 스타일 | 2개 |
| 대표 고유 행동 | 스타일별 3~5개 |
| 강화 기회 | 전투 후 1회 이상 |
| 강화 유형 | 비용·사거리·피해·대응·회복 중 1~2축 |
| 심법 역할 | 스타일별 대표 패시브 1개 |
| 절초 | 필수 아님 |

검증 질문:

- 성장 선택이 다음 전투 계획을 실제로 바꾸는가?
- 단순 피해 상승보다 거리·대응·자원 운용을 강화하는가?
- 같은 8개 기초 행동과 명확하게 연결되는가?
- 카드·로그만 보고 변경 효과를 이해하는가?
- AI가 같은 공용 규칙으로 사용할 수 있는가?

## 4. 장기 보유 구조 가설

### 4.1 핵심무공

`HYPOTHESIS`:

- 한 회차에 여러 핵심무공을 익힐 수 있다.
- 각 계통은 대표 세력 무공과 공용 무공 후보를 가질 수 있다.
- 한 회차에서 최고 단계 핵심무공은 제한할 수 있다.
- 핵심무공은 고유 행동·태그·공용 수정자를 제공한다.

### 4.2 심법

`HYPOTHESIS`:

- 한 회차에 최대 2개 보유.
- 자원 순환·대응·상태 유지·명상 변형을 담당.
- 관련 핵심무공을 단방향으로 강화.
- 핵심무공 숙련도가 심법을 자동 강화하지 않음.
- 최고 단계 심법 수를 제한할 수 있음.

정확한 보유 한도는 T1에서 조합 부담과 UI 복잡도를 확인한 뒤 확정한다.

## 5. 1~10성 장기 구조 후보

아래 표는 전체판 방향을 비교하기 위한 `HYPOTHESIS`다.

| 성급 | 핵심무공 후보 | 심법 후보 |
|---:|---|---|
| 1성 | 기본 패시브 | 기초 자원·방어 패시브 |
| 2성 | 기본 수치·조건 강화 | 기초 패시브 강화 |
| 3성 | 기술 1 | 대표 기믹 또는 패시브 |
| 4성 | 기술 1 효율 강화 | 기믹·명상 효율 강화 |
| 5성 | 비용·사거리·조건 강화 | 자원 순환·조건 강화 |
| 6성 | 기술 2 | 기술 2 또는 명상 변형 |
| 7성 | 연계·대응 강화 | 기믹 유지·중첩 강화 |
| 8성 | 상위 효율 강화 | 대표 무공 연계 강화 |
| 9성 | 기술 3 | 기술 3 또는 상위 패시브 |
| 10성 | 절초 후보 | 진의 후보 |

기존 방향:

- 시작 세력의 핵심무공과 심법을 각각 3성에서 시작하는 안.
- 성급 `n` 도달 비용을 `n` 수련 포인트로 두는 안.
- 핵심무공 10성 최대 1개, 심법 10성 최대 1개 안.

이 수치는 T1 경제·선택 빈도·회차 길이 검증 전 확정하지 않는다.

## 6. 세력 정체성 후보

아래 12세력은 전체판 후보이며 현재 구현 데이터가 아니다.

| 세력 후보 | 핵심무공 후보 | 심법 역할 후보 | 현행 계약에 연결할 수 있는 방향 |
|---|---|---|---|
| 화산파 | 검법 | 연격 | 다음 검법의 기력·내력 비용 또는 태세 보너스 조정 |
| 개방 | 곤법 | 호신·기세 | 막기 성공·위치 유지로 기세 또는 다음 이동 강화 |
| 당문 | 암기술 | 독공 | 상태 태그·지속 피해 후보, T1 전 별도 규칙 필요 |
| 소림 | 권법 | 호신심법 | 근접 대응·방어도·강건 후보 |
| 무당파 | 태극권·검법 | 내력 순환 | 회피·막기 후 다음 공격 보너스 |
| 마교 | 도법 | 체력 대가 | 체력 비용과 공격 강화의 명시적 교환 |
| 양가장 | 창법 | 거리 유지 | 사거리 2·보법·추적 조건 |
| 백우궁 | 궁술 | 보법 연계 | 장거리 방향 공격 후보, 별도 장전은 HOLD |
| 관군 | 창법·방패술 | 군진 | 막기 후 이동·공격 강화 |
| 아미 | 점혈·검법 | 의술 | 기력·내력 압박과 회복 |
| 오독 | 독장 | 독맥 | 상태 누적과 내력 회복 후보 |
| 녹림채 | 도법 | 외공 | 기력 압박과 연속 공격 |

세력 후보는 다음 이유로 변경될 수 있다.

- 역할이 서로 중복됨.
- 현행 거리 1 이상 규칙과 맞지 않음.
- 새 자원·상태·UI 비용이 과도함.
- AI가 공정하게 운용하기 어려움.
- T1 플레이 스타일 2개만으로 충분한 재미가 확인됨.

## 7. 화산 연격 가설 재정의

`PLANNED_T1` 후보:

### 7.1 생성

- 검법 태그 행동이 정상 실행되면 `연격` 1회를 얻는다.
- 다음 검법의 기력 또는 내력 비용을 줄이거나 다음 검법에 조건부 보너스를 준다.
- 다른 무공 계통에는 적용하지 않는다.
- 최소 비용·최대 중첩·지속 범위를 데이터로 제한한다.

### 7.2 현행 묶음과의 연결

- 지속은 `다음 검법 실행` 또는 `현재 행동 묶음 종료` 같은 현행 단위를 사용한다.
- 2슬롯 행동의 준비 수와 실행 수를 구분한다.
- 피격 중단이 추가되면 중단된 검법이 연격을 생성·소모하는지 명시한다.
- UI는 연격 보유·적용 대상·예상 비용을 표시한다.

### 7.3 검증

- 연격이 검법 연타를 강제해 다른 행동을 무의미하게 만들지 않는가?
- 명상·태세·보법과 섞을 이유가 있는가?
- 자원 감소가 무제한 무료 행동으로 이어지지 않는가?
- 로그에서 생성·적용·소모를 구분하는가?

## 8. 공용 무공 콘텐츠 단위

T1 이후 계통 하나를 만들 때 권장 단위:

| 유형 | 역할 |
|---|---|
| 대표 무공 | 플레이 스타일 정체성 |
| 안정형 공용 무공 | 낮은 비용·명확한 조건 |
| 조건형 공용 무공 | 높은 비용·거리·대응·상태 조건 |

공용 무공은 세력 고유 기믹을 자동 생성하지 않되, 이미 생성된 공용 상태·태그의 혜택을 받을 수 있다.

## 9. 절초·진의 가설

### 절초

- 핵심무공의 최고 단계 보상 후보.
- 전투 중 절초 기세와 별도 해금 조건을 모두 요구할 수 있다.
- 카드 또는 행동 슬롯을 통해 발동 예약 상태를 표시한다.
- 실제 기세 소모·피해·상태는 전투 도메인이 계산한다.

### 진의

- 심법의 최고 단계 보상 후보.
- 대표 기믹의 생성·유지·변환을 강화한다.
- 무한 자원·무한 회피 같은 영구 안전 상태를 만들지 않는다.

성장 해금형 절초와 진의는 현재 T0에 구현하지 않는다. 공용 절초 3종의 현재 규칙은 `docs/02_COMBAT_RULES.md`가 책임진다.

## 11. 데이터 작성 계약

실제 T1 데이터로 승격할 때 각 무공·심법은 다음을 가진다.

```text
stable id
version or schema
source faction or common
category
tags
action slot cost
stamina/internal cost
targeting mode
range or move range
damage/recovery/defense modifier
status application and duration
momentum interaction
player-facing text
AI evaluation hints
save default and migration
```

금지:

- 표시 이름을 저장 키로 사용.
- UI 문자열을 판정 조건으로 사용.
- 테스트 없는 문파 전용 예외.
- 기존 카드 필드와 의미가 겹치는 새 필드.
- 현재 지원하지 않는 효과를 문구만으로 구현 완료 처리.

## 12. 성장 진입 게이트

핵심무공·심법 구현은 다음을 만족한 뒤 시작한다.

1. T0 STEP 0~13 런타임 통과.
2. STEP 14에서 거리·대응·자원·중단의 재미 확인.
3. T1 플레이 스타일 2개와 상대 3명 범위 확정.
4. 보상 선택이 다음 전투에 미치는 최소 효과 정의.
5. 저장·ID·Schema·migration 범위 합의.
6. AI가 같은 행동 계약을 사용하도록 계획.
7. 접근성·로그 표현 수용 기준 정의.

## 13. 검증 기준

- 현재 T0 미구현 성장 요소를 완료로 표시하지 않는다.
- 모든 신규 효과가 현행 공용 판정 또는 승인된 새 계약에 연결된다.
- 세력별 정체성이 단순 피해 배율만으로 구분되지 않는다.
- 비용·거리·상태·지속의 정상·실패·경계 테스트가 있다.
- 저장·AI·UI·로그 소비자를 함께 갱신한다.
- T1 결과가 좋지 않으면 세력 수·성급 수·콘텐츠 단위를 축소할 수 있다.
