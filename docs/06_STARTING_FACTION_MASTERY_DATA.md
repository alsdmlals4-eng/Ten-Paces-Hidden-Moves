# 십보강호 세력·핵심무공·심법 성장 가설

> 책임: T1 이후 세력 정체성·무공·심법·성장 구조와 현행 전투 계약에 연결 가능한 후보  
> 현재 전투 규칙: `docs/02_COMBAT_RULES.md`  
> 콘텐츠 범위: `docs/03_CONTENT_CATALOG.md`  
> 단계 진입: `docs/05_COMBAT_POC_SPEC.md`

## 1. 현재 상태

이 문서는 **T1 이후 가설 원본**이다.

- 현재 T0에는 세력 선택·핵심무공·심법·성급·진의·수련 경제가 연결되지 않았다.
- 공용 절초 3종은 T0 전투 POC에 구현돼 있으나 성장 해금형 절초 체계가 확정된 것은 아니다.
- 아래 수치·해금 단계·세력 수는 제품 확정값이 아니다.
- T0 실제 사용자 STEP 14와 프로젝트 코어 승인을 통과하기 전 대규모 무공 데이터를 제작하지 않는다.

상태:

- `PLANNED_T1`: 최소 세로 슬라이스에서 검증할 후보.
- `HYPOTHESIS`: 장기 방향 비교용.
- `HOLD`: 재설계 승인 전 사용 금지.

## 2. 현행 전투 계약과 연결 원칙

세력·무공·심법은 우선 다음 공용 필드와 상태를 조합한다.

```text
action_slots
resolution_phase
stamina_cost
internal_cost
range
move_range
damage
attack_power_coefficient
block
recovery
momentum
tags
status duration
response profile
next attack bonus
interruptible
fortitude
sure_hit
```

현행 공용 단위:

- 전장 10칸과 거리 0 `[밀착]`.
- 라운드 `3수 → 3수 → 4수`.
- 이동·보법·막기·회피·속공·강공·명상·태세.
- 합·방어·회피·필중·중단·강건.
- 기력·내력·절초 기세.
- 준비·실행·예약·취소·전투 불능.

원칙:

- UI는 상태·예상값·결과를 표현하고 판정을 확정하지 않는다.
- 무공은 현행 판정 파이프라인과 같은 실행 수 합 계약을 재사용한다.
- 세력 하나를 위해 별도 라운드·전장·판정 엔진을 만들지 않는다.
- 새 필드는 데이터·저장·AI·로그·UI·테스트 소비자를 함께 설계한 뒤 도입한다.
- 독·점혈·밀치기처럼 현행 상태 계약에 없는 효과는 별도 승인 전 수치 데이터로 확정하지 않는다.

## 3. T1 최소 성장 실험

T1에서는 전체 1~10성 체계를 먼저 구현하지 않는다.

| 영역 | 최소 범위 |
|---|---|
| 플레이 스타일 | 2개 |
| 대표 고유 행동 | 스타일별 3~5개 이하 |
| 강화 기회 | 전투 후 1회 이상 |
| 강화 축 | 비용·거리·피해·대응·기세 중 1~2개 |
| 심법 역할 | 스타일별 대표 패시브 1개 |
| 절초 | 필수 아님 |

검증 질문:

- 성장 선택이 다음 전투의 행동 순서와 거리 계획을 바꾸는가?
- 단순 피해 상승보다 합·대응·자원·기세 판단을 강화하는가?
- 기초 행동·절초와 명확하게 연결되는가?
- 카드·HUD·로그만 보고 효과를 이해하는가?
- AI가 비공개 정보 없이 같은 규칙으로 사용할 수 있는가?
- 강화가 한 가지 정답 빌드를 강제하지 않는가?

## 4. 장기 보유 구조 가설

### 4.1 핵심무공

`HYPOTHESIS`:

- 한 회차에 여러 핵심무공을 익힐 수 있다.
- 각 계통은 대표 세력 무공과 공용 무공 후보를 가질 수 있다.
- 한 회차 최고 단계 핵심무공 수를 제한할 수 있다.
- 핵심무공은 고유 행동·태그·공용 수정자를 제공한다.

### 4.2 심법

`HYPOTHESIS`:

- 한 회차 최대 2개 보유 후보.
- 자원 순환·대응·상태 유지·명상 변형을 담당.
- 관련 핵심무공을 단방향으로 강화.
- 핵심무공 숙련도가 심법을 자동 강화하지 않음.
- 최고 단계 심법 수 제한 후보.

정확한 보유 한도는 T1에서 조합 부담·UI 복잡도·선택 밀도를 확인한 뒤 정한다.

## 5. 1~10성 장기 구조 후보

| 성급 | 핵심무공 후보 | 심법 후보 |
|---:|---|---|
| 1성 | 기본 패시브 | 기초 자원·방어 패시브 |
| 2성 | 기초 효율 강화 | 패시브 강화 |
| 3성 | 기술 1 | 대표 기믹 |
| 4성 | 기술 1 효율 | 기믹·명상 효율 |
| 5성 | 비용·거리·조건 | 자원 순환 |
| 6성 | 기술 2 | 기술 2 또는 명상 변형 |
| 7성 | 연계·대응 | 상태 유지·중첩 |
| 8성 | 상위 효율 | 대표 무공 연계 |
| 9성 | 기술 3 | 상위 패시브 |
| 10성 | 절초 후보 | 진의 후보 |

기존 후보:

- 시작 세력 핵심무공과 심법을 각각 3성에서 시작.
- 성급 `n` 도달 비용을 `n` 수련 포인트로 설정.
- 한 회차 핵심무공 10성 최대 1개, 심법 10성 최대 1개.

T1 경제·회차 길이·선택 빈도 검증 전 확정하지 않는다.

## 6. 세력 정체성 후보

| 세력 | 핵심무공 후보 | 심법 역할 | 현행 계약 연결 방향 |
|---|---|---|---|
| 화산파 | 검법 | 연격 | 다음 검법 비용·태세·합 보너스 |
| 개방 | 곤법 | 호신·기세 | 막기·위치 유지·기세 순환 |
| 당문 | 암기술 | 독공 | 상태 태그 후보, 별도 승인 필요 |
| 소림 | 권법 | 호신 | 밀착·방어도·강건 |
| 무당파 | 태극권·검법 | 내력 순환 | 회피·막기 후 반격 |
| 마교 | 도법 | 체력 대가 | 체력 비용과 공격 강화 교환 |
| 양가장 | 창법 | 거리 유지 | 사거리 2·보법·추적 |
| 백우궁 | 궁술 | 보법 연계 | 장거리 방향 공격, 장전은 HOLD |
| 관군 | 창법·방패술 | 군진 | 막기 후 이동·공격 |
| 아미 | 점혈·검법 | 의술 | 기력·내력 압박과 회복 |
| 오독 | 독장 | 독맥 | 상태 누적·내력 회복 후보 |
| 녹림채 | 도법 | 외공 | 기력 압박·연속 공격 |

세력 후보는 역할 중복·새 시스템 비용·AI 공정성·프로젝트 코어 기여도에 따라 줄이거나 합친다.

## 7. 화산 연격 최소 후보

`PLANNED_T1` 후보:

- 검법 태그 행동이 정상 실행되면 `연격` 1회를 얻는다.
- 다음 검법의 기력·내력 비용 또는 합 원공격력에 조건부 보너스를 줄 수 있다.
- 다른 무공 계통에는 자동 적용하지 않는다.
- 최소 비용·최대 중첩·지속을 데이터로 제한한다.
- 지속 단위는 `다음 검법 실행` 또는 `현재 행동 묶음 종료`처럼 현행 단위를 사용한다.
- 준비만 하고 중단된 검법이 연격을 생성·소모하는지 명시해야 한다.

검증:

- 검법 연타가 이동·명상·대응을 무의미하게 만들지 않는가?
- 합·중단·기세 판단을 강화하는가?
- 무제한 무료 행동으로 이어지지 않는가?
- 생성·적용·소모가 로그에서 구분되는가?

## 8. 절초·진의 가설

### 절초

- 핵심무공 최고 단계 보상 후보.
- 전투 중 기세와 성장 해금 조건을 함께 요구할 수 있다.
- T0 공용 절초 예약·취소·소모 계약을 재사용한다.
- 피해·상태는 전투 도메인이 계산한다.

### 진의

- 심법 최고 단계 패시브 후보.
- 전투 시작·명상·대응·자원 순환을 변형할 수 있다.
- 화면에 항상 보이는 상태와 로그 원인을 제공해야 한다.

T0의 공용 절초 3종을 특정 세력의 10성 절초로 자동 귀속하지 않는다.

## 9. 수련 경제 가설

기존 후보:

```text
최종 수련 포인트 = 승리 기본 5 + 성과 등급 보너스 + 행운 결과
```

- C/B/A/S 성과 등급 후보.
- 낮은 성과일수록 큰 행운 보정 후보.
- 성급 `n` 비용 `n`포인트 후보.

위험:

- 높은 성과가 행운 감소로 느껴질 수 있다.
- 성과 보상과 재기 보정이 섞일 수 있다.
- 최적 투자 경로가 지나치게 고정될 수 있다.

대안은 성과·행운·재기 보정을 별도 시스템으로 분리하는 것이다. 아직 확정하지 않는다.

## 10. 성장 진입 게이트

T1 성장 구현 전에 다음을 확인한다.

- [ ] T0 실제 사용자 STEP 14 결과가 있다.
- [ ] 프로젝트 코어가 사용자 승인으로 확정됐다.
- [ ] 성장 선택이 강화해야 할 핵심 판단이 명시됐다.
- [ ] 플레이 스타일 2개의 차이가 공용 계약으로 설명된다.
- [ ] 저장·재시도·결정론 경계가 설계됐다.
- [ ] 카드·HUD·로그·AI·테스트 소비자가 정의됐다.
- [ ] 5전 데모 전체 경제를 먼저 만들지 않는다.

## 11. 검증 기준

- 현재 T0 구현과 T1 가설을 혼동하지 않는다.
- 세력·무공이 공용 판정 계약을 재사용한다.
- 새 상태·자원·태그는 소유자·표시·저장·AI·테스트를 가진다.
- 성장 선택이 다음 전투 행동을 바꾸는지 행동 증거로 확인한다.
- 수치가 확정되지 않았으면 `HYPOTHESIS`로 유지한다.
