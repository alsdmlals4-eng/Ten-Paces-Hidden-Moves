# 십보강호 전투 규칙서

> 책임: 전장·행동 배치·대상·비용·판정·대응·자원·라운드 진행 규칙  
> 현재 기준: STEP 0~10 + TARGETING 10.5 + RESPONSE/RESOURCE PREVIEW 10.6  
> 구현 계약: `data/combat/`, `data/cards/basic_cards.json`, `src/combat/combat_resolution_engine.gd`

## 1. 상태 구분

이 문서는 규칙의 상태를 다음처럼 구분한다.

- `CONFIRMED`: 사용자 확정값과 데이터·코드·정적 테스트가 일치한다.
- `IMPLEMENTED_FOR_REVIEW`: 구현됐지만 최신 사용자 Windows 확인이 남았다.
- `PLANNED`: 다음 STEP에서 구현한다.
- `HOLD`: 과거 규칙 또는 미래 후보이며 현재 판정에 사용하지 않는다.

## 2. 전장과 위치

### 2.1 기본 전장

- 전장은 1번부터 10번까지의 일자형 10칸이다.
- 플레이어 기본 시작 위치는 4번 칸이다.
- 상대 기본 시작 위치는 7번 칸이다.
- 시작 거리는 3칸이다.
- 전장 카메라는 전체 10칸을 한 화면에서 읽을 수 있는 고정 와이드 구도다.
- 한 칸에는 한 전투원만 위치한다.
- 전장 밖으로 이동할 수 없다.
- 상대가 현재 점유한 칸을 이동 목적지로 선택할 수 없다.

### 2.2 동시 이동 충돌

- 양측이 같은 빈 칸을 동시에 목적지로 선택하면 둘 다 이동하지 않는다.
- 현재 POC에서는 상대 칸으로의 이동이 금지되므로 서로 통과하거나 자리를 교환하지 않는다.
- 한 칸 중첩·통과·자리 교환은 `HOLD`다.

### 2.3 거리

```text
거리 = 두 전투원의 칸 번호 차이의 절댓값
```

- 거리 1은 인접 칸이다.
- 현재 POC에서는 같은 칸에 위치할 수 없으므로 정상 전투 거리는 1 이상이다.
- 공격은 선택한 방향이 상대 방향과 일치하고 실제 거리가 카드 사거리 이내일 때 적중 가능하다.

## 3. 라운드와 행동 묶음

### 3.1 라운드 구성

한 라운드는 총 10수다.

```text
1묶음: 1수 · 2수 · 3수
2묶음: 4수 · 5수 · 6수
3묶음: 7수 · 8수 · 9수 · 10수
```

표기:

```text
3수 → 3수 → 4수
```

- 전투는 1라운드 1묶음 1수에서 시작한다.
- 현재 묶음만 편집할 수 있다.
- 지난 묶음은 통과 상태다.
- 미래 묶음은 잠금 상태다.
- 한 묶음을 판정한 뒤에만 다음 묶음이 열린다.
- 3묶음의 10수 판정이 끝나면 다음 라운드 1묶음 1수로 돌아간다.
- 10수 완료는 전투 종료가 아니다.

### 3.2 행동 배치 완료 조건

`[진행]` 버튼은 다음 조건을 모두 만족할 때만 활성화한다.

1. 현재 묶음의 모든 행동 슬롯이 점유됐다.
2. 이동·공격 행동의 대상이나 방향이 모두 지정됐다.
3. 실행 순서대로 계산한 기력·내력 예상값이 유효하다.
4. 다중 슬롯 행동이 현재 묶음 안의 연속 슬롯을 정상 점유한다.

빈 슬롯, 미지정 대상, 자원 부족 중 하나라도 있으면 진행할 수 없다.

### 3.3 다중 슬롯 행동

- `action_slots`가 2인 행동은 연속된 두 슬롯을 차지한다.
- 현재 묶음의 끝을 넘어 배치할 수 없다.
- 점유한 두 슬롯 중 어느 칸을 클릭해도 행동 전체를 해제한다.
- 실행 시점은 마지막 점유 수다.

예:

```text
강공을 1수부터 배치
→ 1수·2수 점유
→ 실제 강공 판정은 2수
```

다중 슬롯은 준비 시간과 계획 공간을 함께 나타낸다. 2슬롯 행동은 앞 수가 `[준비]`, 마지막 수가 `[실행]`이며, 3슬롯 행동은 앞 두 수가 `[준비]`, 마지막 수가 `[실행]`이다. 준비 수는 비용·이동·피해를 적용하지 않지만 확정 `presentation_event`와 전투 로그로 한 수씩 재생한다. 피격 중단은 피해를 받은 그 수의 아직 실행 전 행동에만 적용하므로, 앞선 수의 피격은 다중 슬롯 행동의 마지막 실행 수를 취소하지 않는다.

## 4. 행동 선택과 대상 지정

### 4.1 기본 입력 흐름

```text
기초 행동 선택
→ 현재 묶음의 빈 슬롯 선택
→ 필요한 경우 이동 칸 또는 공격 방향 선택
→ 예상 자원과 슬롯 상태 확인
→ 묶음 완료 후 진행
```

카드를 다시 선택하거나 점유 슬롯을 해제하면 계획을 수정할 수 있다. `[진행]`으로 판정을 시작한 뒤에는 해당 묶음을 변경하지 않는다.

### 4.2 이동 대상

- 이동 행동을 배치하면 이동 가능한 칸을 녹색 상태로 표시한다.
- 색뿐 아니라 이동 상태와 문구를 함께 사용한다.
- 이동은 도착 칸을 직접 선택한다.
- 앞선 수에 배치된 이동을 반영한 예상 위치를 이후 행동의 출발 위치로 사용한다.

### 4.3 공격 방향

- 공격 행동을 배치하면 사거리 내 좌·우 칸을 붉은 공격 가능 상태로 표시한다.
- 플레이어는 공격할 왼쪽 또는 오른쪽 방향을 선택한다.
- 공격은 상대를 자동 추적하지 않는다.
- 선택 방향이 상대 반대쪽이면 빗나간다.
- 상대 방향이 맞아도 실제 거리가 사거리보다 멀면 빗나간다.

슬롯에는 다음처럼 선택 결과를 표시한다.

```text
이동: ← 3번 / → 5번
공격: ← 공격 / → 공격
```

## 5. 판정 순서

현재 묶음의 대응 행동을 먼저 준비한 뒤, 각 수를 순서대로 판정한다.

```text
대응 → 속공 → 이동 → 일반 공격
```

실제 처리:

1. 묶음에 배치된 막기·회피·태세 대응 결합의 비용과 방어 프로필을 준비한다.
2. 현재 수의 속공을 판정한다.
3. 현재 수의 이동을 동시에 판정한다.
4. 현재 수의 일반 공격을 판정한다.
5. 현재 수의 명상·단독 태세 같은 일반 행동을 판정한다.
6. 다음 수로 이동한다.

대응은 판정 단계상 먼저 준비되므로 같은 묶음의 공격에 적용된다. 일반 회피의 완전 회피만 배치한 같은 수로 제한된다.

## 6. 같은 판정 단계의 동시성

- 같은 수·같은 공격 단계에서 양측 공격이 모두 유효하면 양쪽 피해를 먼저 계산한 뒤 동시에 적용한다.
- 한쪽이 먼저 피해를 줬다는 이유로 같은 단계의 상대 공격을 취소하지 않는다.
- 실제 피해는 같은 수의 아직 실행 전 행동만 중단하되, 이미 성립한 동시 공격과 이후 수 행동은 유지한다. 전투 불능만 이후 행동도 취소한다.

## 7. 비용과 자원 계획

### 7.1 비용 표시

현재 카드 비용은 다음 세 항목만 사용한다.

- 행동 슬롯.
- 기력.
- 내력.

`행동력`, `내공`, 공통 `막기 경감` 필드는 사용하지 않는다.

### 7.2 배치 즉시 예상값

행동을 배치하거나 해제할 때 현재 묶음의 행동을 실행 순서대로 가상 계산한다.

- 행동 비용을 예상 기력·내력에서 차감한다.
- 명상은 해당 실행 수에서 기력·내력 회복을 예상값에 더한다.
- 행동 해제 시 예상값을 다시 계산한다.
- 실제 전투 자원은 `[진행]` 전까지 확정하지 않는다.
- 자원이 부족한 행동은 `자원 부족`으로 표시하고 진행을 막는다.

예:

```text
1수 명상 → 2수 속공 → 3수 보법
```

명상의 회복이 2수와 3수 비용보다 먼저 적용되므로, 현재 자원만으로는 부족하더라도 실행 순서상 유효하면 진행할 수 있다.

### 7.3 전투 시작 자원

현재 T0 POC 기본값:

| 자원 | 플레이어 | 상대 |
|---|---:|---:|
| 체력 | 20/20 | 20/20 |
| 기력 | 5/5 | 5/5 |
| 내력 | 4/4 | 4/4 |
| 절초 기세 | 데이터 지정 | 데이터 지정 |

- 체력·기력·내력은 전투 시작 시 최대치로 초기화한다.
- `start_penalties.health`, `start_penalties.stamina`, `start_penalties.internal`이 있으면 최대치에서 해당 값만큼 감소한다.
- 최소 시작값은 0이다.
- 절초 기세는 최대치로 자동 회복하지 않는다.
- HUD의 플레이어 4/5·상대 3/5는 기세 게이지 표시 검증용 데이터다.

## 8. 기초 행동 8종

### 8.1 이동

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 없음 |
| 이동 거리 | 좌·우 1칸 |
| 판정 단계 | 이동 |

선택한 인접 빈 칸으로 이동한다.

### 8.2 보법

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 내력 1 |
| 이동 거리 | 좌·우 1칸 또는 2칸 |
| 판정 단계 | 이동 |

플레이어가 1칸과 2칸 중 실제 도착 칸을 선택한다.

### 8.3 막기

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 없음 |
| 기본 방어도 | 4 |
| 판정 단계 | 대응 |

막기는 현재 행동 묶음의 공격 피해를 방어도만큼 줄인다.

같은 수에 들어온 공격에는 다음 두 감소량을 비교한다.

```text
A. 원래 피해의 50%
B. 현재 방어도
최종 감소량 = A와 B 중 큰 값
```

최종 피해는 0보다 작아지지 않는다.

예:

```text
피해 6, 같은 수
50% 감소량 3 / 방어도 4
→ 4 감소, 최종 피해 2

피해 12, 같은 수
50% 감소량 6 / 방어도 4
→ 6 감소, 최종 피해 6

피해 12, 같은 묶음의 다른 수
→ 방어도 4 감소, 최종 피해 8
```

막기의 묶음 방어도는 막기를 배치한 수보다 앞선 수를 포함해 현재 묶음 전체에 적용된다. 이는 모든 대응 프로필을 묶음 판정 시작 시 준비하기 때문이다.

### 8.4 회피

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 기력 1 |
| 효과 | 같은 수 공격 완전 회피 |
| 판정 단계 | 대응 |

일반 회피는 배치한 같은 수에 들어오는 공격만 피해 0으로 만든다. 같은 묶음의 다른 수는 보호하지 않는다.

### 8.5 속공

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 기력 1 |
| 사거리 | 지정 방향 거리 1 |
| 피해 | 6 |
| 판정 단계 | 속공 |

이동 전에 실행한다. 따라서 같은 수에 상대가 이동을 배치했더라도 이동 전 위치를 기준으로 공격한다.

### 8.6 강공

| 항목 | 값 |
|---|---|
| 슬롯 | 2 |
| 비용 | 기력 1·내력 1 |
| 사거리 | 지정 방향 거리 1 또는 2 |
| 피해 | 8 |
| 판정 단계 | 일반 공격 |

연속 두 슬롯을 점유하고 마지막 점유 수에서 실행한다.

### 8.7 명상

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 없음 |
| 회복 | 기력 2·내력 1 |
| 판정 단계 | 일반 |

최대치를 넘겨 회복하지 않는다. 배치 예상값과 실제 판정에 같은 회복량을 사용한다.

### 8.8 태세

| 항목 | 값 |
|---|---|
| 슬롯 | 1 |
| 비용 | 내력 1 |
| 단독 효과 | 다음 적중 가능한 공격 피해 +2 |
| 판정 단계 | 일반 또는 대응 결합 |

단독 태세는 실행 후 다음 공격에 보너스 2를 저장하고, 해당 공격이 판정될 때 사용한다.

## 9. 태세+대응 결합

태세와 막기 또는 회피를 연속 선택한 뒤 하나의 빈 슬롯에 결합 행동으로 배치할 수 있다. 선택 순서는 어느 쪽이 먼저여도 된다.

```text
태세 → 막기
막기 → 태세
태세 → 회피
회피 → 태세
```

결합 규칙:

- 하나의 행동 슬롯을 점유한다.
- 두 카드의 기력·내력 비용을 합산한다.
- 별도 방향·이동 대상은 필요하지 않다.
- 일반 태세의 다음 공격 +2는 함께 적용하지 않는다. 결합 태세는 대응 강화에 사용된다.

### 9.1 태세+막기

- 현재 행동 묶음 전체에 막기를 적용한다.
- 기본 방어도 4에 1.5배를 적용한다.
- 올림 처리하여 방어도 6이 된다.
- 같은 수 공격에서는 50% 감소량과 강화 방어도 6 중 더 큰 값을 사용한다.

### 9.2 태세+회피

- 현재 행동 묶음의 모든 공격을 완전히 회피한다.
- 회피 효과는 배치 수에 제한되지 않는다.

태세+회피는 매우 강한 효과이므로 비용, 사용 가능 조건, 적의 대응 수단은 POC 플레이테스트에서 집중 검증한다.

## 10. 공격 판정

공격 하나는 다음 순서로 판정한다.

1. 공격 비용을 지불할 수 있는지 확인한다.
2. 선택 방향이 상대 방향과 같은지 확인한다.
3. 현재 실제 거리가 카드 사거리 이내인지 확인한다.
4. 단독 태세의 다음 공격 보너스를 더한다.
5. 회피를 확인한다.
6. 막기의 감소량을 계산한다.
7. 최종 피해를 같은 단계의 대기 피해에 더한다.
8. 해당 단계의 모든 공격 계산 뒤 체력을 동시에 감소시킨다.
9. 기세는 묶음 완료, 막기 성공, 회피 성공, 합 승리의 확정 결과로만 올린다.

공격이 반대 방향이나 사거리 밖으로 실패하더라도 이미 지불한 비용은 반환하지 않는다.

## 11. 이동 판정

이동 하나는 다음 순서로 판정한다.

1. 이동 비용을 지불할 수 있는지 확인한다.
2. 지정 도착 칸이 전장 안인지 확인한다.
3. 출발 위치에서 카드 이동 거리 이내인지 확인한다.
4. 상대 현재 위치와 같은지 확인한다.
5. 양측 이동 목적지가 같은지 확인한다.
6. 유효하면 동시에 위치를 변경한다.

유효하지 않은 이동은 제자리 유지로 처리하며 비용은 반환하지 않는다.

## 12. 절초 기세

현재 POC 규칙:

- 최대치 5.
- 양측 전투 시작 기세는 `0/5`다.
- 묶음을 하나 완료하면 양측 기세가 각각 1 오른다.
- 막기 성공, 회피 성공, 합 승리는 해당 전투원 기세를 각각 1 올린다.
- 일반 적중과 절초 적중은 기세를 올리지 않는다. 최대치를 넘지 않는다.
- 절초는 기세 게이지 아래에 상시 표시되는 3종 목록에서 예약한다. 강공 선택은 목록의 표시·선택 조건이 아니다.

장기적으로는 단순 피해뿐 아니라 정확한 대응·중단·거리 예측 성공을 기세 획득과 연결하는 방향을 STEP 14에서 검증한다.

## 13. 로그와 결과 설명

전투 로그는 다음 사건을 직접 설명한다.

- 행동 배치·해제·배치 불가.
- 이동 칸·공격 방향 선택.

## 14. 현재 적 행동 계획

T0의 상대는 정식 AI가 아니다. `combat_resolution_preview.json`의 고정 행동 계획을 사용한다.

- 1묶음: 이동 → 막기 → 속공.
- 2묶음: 이동 → 속공 → 막기.
- 3묶음: 이동 → 속공 → 강공.

이 계획은 판정·UI·로그 검증을 위한 fixture다. 플레이어의 배치를 읽어 수정하지 않는다.

## 16. 2026-07-23 현행 절초·중단·밀착 규칙

이 절은 절초·중단·강건·밀착의 현행 정본이다.

- 양측 시작 체력은 `30/30`, 공격력은 `8`이다. 공격력은 기초 공격에는 적용하지 않고 절초에만 적용한다.
- 실제 피해는 피해가 발생한 같은 수에서 아직 실행하지 않은 행동만 중단한다. 같은 수 공격은 모두 성립한 뒤 합 또는 동시 피해를 계산하며, 이후 수 행동은 유지한다. 전투 불능은 이후 행동을 취소한다.
- 태세가 실제 실행되면 기존 다음 공격 `+2`와 함께 다음 공격용 `[강건]`을 준다. `[강건]`은 전투원 상태 패널의 금빛 태그·상태 설명·전투 로그에 동시에 표시되며, 보호가 소비되면 즉시 사라진다. 강건은 속공 단계의 실제 피해 중단만 한 번 막고, 피해량·체력 감소는 바꾸지 않는다. 1슬롯 공격만 보호되며 2·3슬롯 행동과 사망에는 적용되지 않는다.
- `[집중]`은 현재 규칙·UI·fixture에서 제거했다. 재도입 후보가 아니다.
- 한 타일에는 1대1 기준 최대 두 전투원이 설 수 있다. 같은 타일은 거리 0의 `[밀착]`이며 사거리 1 이상 공격은 방향과 무관하게 유효하다. 정지한 상대 타일 진입과 같은 빈 타일 공동 이동은 허용하고, 자리 교환·상대 통과는 금지한다.

### 16.1 절초 예약

전투 시작 기세는 `0/5`다. 묶음 완료마다 양측이 1, 막기 성공·회피 성공·합 승리마다 해당 전투원이 1을 얻는다. 절초 3종 목록은 플레이어 기세 게이지 **바로 아래에 planning 중 항상 표시**하고, 기세가 **정확히 5**이며 현재 묶음에 필요한 연속 빈 슬롯이 있을 때만 해당 버튼을 활성화한다. 강공 카드 선택은 절초 목록의 표시·선택 조건이 아니다. 고른 절초는 현재 묶음의 연속 빈 슬롯과 방향을 지정해 `[절초 예약]`으로 확정한다. 예약 즉시 기세 5를 소비하지만, `[진행]` 전 `planning` 상태에서는 점유한 어느 수 슬롯이든 클릭 또는 Enter로 예약 전체를 취소해 기세 5를 돌려받을 수 있다. `committed` 이후 실제 실행·중단·방향 실패·사거리 실패에는 환불하지 않는다. 절초는 기력·내력을 추가로 쓰지 않으며, 일반·절초 적중 자체는 기세를 올리지 않는다.

| 절초 | 슬롯·단계 | 사거리/규칙 | 최종 피해 (공격력 8) |
|---|---|---|---:|
| 십보 유파 | 1·속공 | 지정 방향 최대 1칸 돌진 후 사거리 1 | `8 + floor(8×0.25) = 10` |
| 단악결 | 2·일반 공격 | 지정 방향 사거리 1~2 | `14 + floor(8×0.75) = 20` |
| 파공검기 | 3·일반 공격 | 지정 방향 사거리 1~3 | `22 + floor(8×1.5) = 34` |

파공검기는 1·2묶음에서는 해당 묶음의 세 칸 전체, 3묶음에서는 7~9 또는 8~10만 점유할 수 있다.

## 17. Issue #13 합·방어·필중 규칙

- 같은 수에 양측의 유효 공격이 성립하면 `[합]`이다. `[속공]`은 이동보다 먼저 돌진·위치를 처리하지만, 속공과 일반 공격은 같은 공격군에서 합을 판정한다.
- 합은 방어·회피 전 공격력(기초 피해, 절초 계수, 태세 다음 공격 +2 포함)을 비교한다. 높은 쪽은 피해를 받지 않고 낮은 쪽은 차이 피해를 받으며, 동점은 `[합 상쇄]`로 양측 피해 0이다.
- 합 차이 피해도 방어·회피 대상이다. 승자 공격에 `[필중]`이 있으면 회피만 무시한다. 합 승자는 차이 피해가 막혀도 기세 1을 얻고, 중단·사망은 합 정산 뒤 한 번만 처리한다.
- 막기는 먼저 `max(0, 원피해 - 방어도)`를 적용한다. 막기와 공격이 같은 수면 그 결과를 `floor(×0.5)`한다. 태세 강화 막기는 해당 묶음 방어도 1.5배(기본 4→6)다.
- `[필중]`은 현재 파공검기에만 있다. 방어도, 사거리·방향 실패, 합, 중단, 전투 불능은 무시하지 않는다.
