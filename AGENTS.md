# 십보강호 협업 규칙

## 1. 규칙 우선순위

문서와 실제 파일이 충돌하면 다음 순서를 따른다.

1. 사용자의 최신 확정 지시
2. 이 `AGENTS.md`
3. `docs/BASE_RULES_VERSION.md`가 존재하면 해당 버전 정보
4. `docs/DOCUMENTATION_MAP.md`
5. 프로젝트 내부에 동기화된 Base 규칙 사본
6. 프로젝트 전용 기획서·README
7. 현재 Issue·Goal
8. 실제 구현 파일

## 2. 기획 문서 갱신

- 대화에서 확정된 기획 변경은 관련 GitHub 기준 문서에 같은 작업 흐름 안에서 반영한다.
- 전투, 아트, UI, 콘텐츠, 세계관 등 분야별 기획서는 누락 없이 서로 연결해 최신화한다.
- 새 버전 문서를 계속 늘리지 않는다. `*_v2`, `*_final`, `*_latest` 같은 활성 중복 파일을 만들지 않고 기존 기준 파일을 직접 갱신한다.
- 파일명과 경로를 유지할 수 있으면 유지한다. 경로 변경이 필요하면 연결 문서와 링크를 함께 고친다.
- 수치·용어가 여러 문서에 걸치면 단일 기준 문서를 먼저 정하고, 나머지 문서는 그 기준을 참조하게 한다.

## 3. 백업 정책

- 기준 문서를 크게 수정하기 전에는 수정 전 파일 전체를 `docs/[백업]/YYYY-MM-DD/`에 복사한다.
- `docs/[백업]/`은 읽기 전용 과거 기록이다.
- ChatGPT, Codex 및 기타 작업 에이전트는 백업 문서를 현재 요구사항, 구현 기준, 테스트 기준으로 사용하지 않는다.
- 활성 작업은 `docs/DOCUMENTATION_MAP.md`에 연결된 기준 문서만 사용한다.
- 백업 파일은 과거 결정의 경위를 확인할 때만 참고한다.

## 4. 관련 문서 동기화

- 기준 문서를 갱신하면 `docs/DOCUMENTATION_MAP.md`의 제목·설명·읽기 순서를 확인한다.
- 프로젝트 첫 화면의 링크가 바뀌면 `README.md`도 함께 갱신한다.
- 아트 시안에 표시된 임시 수치가 전투 데이터와 다르면 시안 수치를 규칙 기준으로 확정하지 않는다. 레이아웃 예시임을 명시하고 수치 기준 문서를 따른다.
- 구현 범위 또는 완료 기준이 바뀌면 테스트 체크리스트와 Codex Plan의 동기화 필요 여부를 반드시 검토한다.

## 5. 작업 결과 보고

GitHub 갱신 후 다음을 보고한다.

- 수정한 기준 파일
- 생성한 백업 파일
- 연결 영향이 있는 문서
- 후속 동기화가 필요한 항목
- 공용 Base로 승격할 규칙과 프로젝트 전용으로 남길 규칙
