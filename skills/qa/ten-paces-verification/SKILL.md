# 십보강호 검증

## 책임 원본

- 제품 체크리스트: `docs/08_TEST_CHECKLIST.md`
- 작업 게이트: `[기획서]/00_프로젝트_허브/DEVELOPMENT_GATES.md`
- 규칙·범위·아키텍처·UI·연출의 관련 본책.

## 검증 층

1. 정적 문법·경로·Registry.
2. 문서 간 수치·용어·상태 일치.
3. 순수 전투 시뮬레이션 단위 테스트.
4. 저장·재도전·결정론 회귀.
5. UI 입력·상태·접근성·자산 폴백.
6. Windows 데모와 전체판 빌드.
7. 플레이어가 규칙·상대 정보·결과 원인을 설명하는지 확인.
8. PDF 생성·전 페이지 렌더·사람 시각 검수.

## 절차

1. 검증할 주장과 실패 조건을 문장으로 적는다.
2. 필요한 입력·환경·도구·명령을 확인한다.
3. 자동 검증과 사람 검수를 분리한다.
4. 예상 결과와 실제 결과를 기록한다.
5. 실패를 숨기거나 우회 결과를 정상 완료로 처리하지 않는다.
6. 수정 후 가장 좁은 회귀부터 전체 영향 범위로 확장한다.
7. 완료 보고에서 통과·실패·미실행을 분리한다.

## 주요 제품 검증

- AI 사전 잠금과 잠금 뒤 변경 불가.
- 10칸·10타이밍·2수 묶음 결정론.
- 일반 공격·공격 대 방어·합 구분.
- 성과·행운·수련 정산과 저장 유지.
- 상대 내부 프로필과 이명·풍문·정탐 일치.
- 절초 보유·미보유 5전 완주.
- 접근성 옵션·자산 누락에서 결과 불변.

## 금지

- 체크리스트 존재를 테스트 통과로 간주.
- 파일 존재를 실제 실행 성공으로 간주.
- 정적 패턴만으로 UI 품질 결함 확정.
- 실행하지 않은 검증을 암묵적으로 통과 처리.

## 완료 기준

- 각 완료 주장에 재현 가능한 증거가 있다.
- 실패와 미검증이 다음 작업으로 연결된다.
- 변경 전후 결과를 비교할 수 있다.
- 사용자·플레이어 이해 검증과 자동 테스트가 분리된다.
