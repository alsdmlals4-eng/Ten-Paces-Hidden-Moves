# 십보강호: 숨은 수의 비무
## Ten Paces: Hidden Moves

10칸 전장에서 양측이 행동 두 개를 비공개로 잠그고 동시에 공개하며, 거리·합·행동력·기력·내공을 읽어 승부하는 1대1 무협 로그라이트입니다.

## 현재 상태

- 대상 엔진: **Godot 4.7.1 stable**.
- T0 전투 플레이테스트 프로토타입을 구성했습니다.
- 포함: 10칸 전장, AI 선잠금, 2수 동시 공개, 이동, 공격, 막기, 회피, 명상, 합, 절초 기세와 절초 발동.
- 제외: 5전 대회, 수련·성과·행운, 상대 정보·제약, 12세력 전체 데이터.
- 실제 Godot 4.7.1 런타임 검증은 해당 실행 환경에서 수행해야 합니다.

## 빠른 시작

```bash
godot --editor --path .
godot --headless --path . --script res://tests/run_all.gd
python scripts/check_planning_sync.py
python tests/static_check.py
```

- 메인 씬: `scenes/combat_playtest.tscn`
- 플레이테스트 안내: [`PLAYTEST.md`](PLAYTEST.md)
- 통합 기획서: [`[기획서]/README.md`](%5B기획서%5D/README.md)

## 기획서 구조

`[기획서]/`는 사람과 AI가 함께 확인하는 통합 설계 공간입니다.

- `00_통합`: 전체 방향과 현재 제작 상태
- `01_게임기획`: 핵심 경험과 범위
- `02_전투기획`: 2수·합·절초 기세
- `03_UI_UX`: 화면 정보 위계와 상호작용
- `04_아트디자인`: 이미지와 시각 원칙
- `05_플레이테스트`: 준비·진행·기록 템플릿
- `assets/예시이미지`: 문서 본문에서 바로 보는 참고 이미지
- `manifest.json`: AI와 자동 검사에서 사용하는 문서·이미지·구현 경로 목록

## 현재 방향

- 전장 10칸, 라운드 행동 타이밍 10개, 전체 대회 10전의 `10-10-10` 구조입니다.
- 한 라운드는 행동 두 개씩 묶어 총 5번 잠금·동시 공개·순차 해상합니다.
- 같은 타이밍에 양측 공격이 유효하면 `[합]`을 겨루고 낮은 쪽이 차이만큼 피해를 받습니다.
- 절초는 핵심무공 10성으로 해금하고 전투 중 절초 기세 6을 채워 발동합니다.
- 절초는 데모 필수 조건이 아닌 선택적 하이라이트입니다.
- 상대 내부 수치는 이명·풍문·전적·정탐으로 암시하며 개발용 태그를 직접 노출하지 않습니다.

## 다음 순서

1. Godot 4.7.1에서 자동 테스트와 수동 T0 세션을 실행합니다.
2. 입력·가독성·기세 충전·절초 소모 문제를 같은 범위에서 수정합니다.
3. T0 기준 통과 후 상대 정보→제약→전투→성과·행운→수련의 T1 5전 흐름으로 확장합니다.
4. 절초 보유·미보유 양쪽 5전 완주를 검증한 뒤 시작 6세력과 12세력으로 확장합니다.
